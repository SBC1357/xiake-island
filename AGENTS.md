# AGENTS.md

## 侠客岛仓库级最高前置规则

所有会动侠客岛仓库的 Agent，在开始任何任务前，必须先读：

1. `仓库卫生最高禁令.md`
2. `防钻空子.md`
3. `一级事故补充案例.md`
4. `防爆指令.md`
5. 当前任务自己的需求文档（如果有）

如果当前任务没有 SPEC，也照样必须先服从 `仓库卫生最高禁令.md` 与 `一级事故补充案例.md`。
如果任何任务文档与它们冲突，以 `仓库卫生最高禁令.md` 与 `一级事故补充案例.md` 为最高优先级；`防钻空子.md` 作为默认并行约束同步生效；涉及长线程续跑、接棒或可能中断的任务时，`防爆指令.md` 也默认并行生效。
上游文件写了“必须先读”，不等于当前 Agent 已执行逐份阅读。
开始执行、汇报、调度、巡检、续跑前，必须先完成当轮前置阅读回执，至少写清：已直接读取的文件、每份文件各自 `1` 条本轮约束、当前最相关红线。
没有回执，一律按未读处理；未读状态下发生的一切执行、汇报和完成声明默认无效。

## 可变内容唯一真相源纪律

所有**会变化的内容**，必须绑定**唯一真相源**。

最常见的可变内容包括：

1. 当前阶段
2. 当前阻塞
3. 当前下一步
4. 当前分数 / 当前是否达标
5. 当前执行位置
6. 当前是否完成 / 是否通过
7. 当前试验状态 / 当前验证结论

硬规则：

1. 真相源没有改完、没有冻结前，禁止在多个文件里各写一版新口径。
2. 除真相源外，其他文件只允许：
   - 明确回指真相源
   - 摘要已经冻结的稳定结论
3. 不允许把“当前进度、当前阻塞、当前下一步”同时写进规则正文、任务卡、运行状态卡、步骤拆解、总结文档等多个地方。
4. 只要同一件可变内容出现多处说法，且彼此不完全一致，按口径污染处理。

默认分工：

1. 当前阶段 / 当前主任务 / 当前阻塞 / 当前下一步：只认任务卡或状态卡中的唯一真相源
2. 运行态 / 验证态 / 原始报错 / 最近验证：只认运行状态卡与直接验证锚点
3. 长期规则 / 节点标准：只认正式规则层
4. 系统总图 / 节点关系：只认完整架构层
5. 经验当前裁决状态：只认经验裁决层

## 永久红线

- 不得在仓库内随地大小便
- 不得把任何运行、测试、构建、导出、调试垃圾倒进仓库主路径
- 一切会导致文件膨胀的操作，必须先考虑落点与清理责任
- 怎么来的，怎么清走
- 不做仓库卫生检查，不得宣称完成

## 违规处理

- 本轮完成声明无效
- 脏产物必须立刻清走或搬离仓库
- 当前 Agent 上下文视为不可信，可终止并重开

## 文档红线

- 规划、计划、报告、清单默认都是临时过程文档
- 过程文档用完就删
- 能压成 handoff / checkoff 的，不允许继续留一堆 SPEC / prompt / checkpoint
