# 强留痕记录模板_v1

状态：正式模板  
日期：2026-04-03  
用途：为 `01_强留痕重跑记录.md` 提供统一记录骨架，把“看得见推理过程”和“可审计、可比较、可升期判断”同时保住

## 一、强留痕的工作定义

强留痕不是原始聊天记录整段倾倒。  
强留痕要保留推理过程，但要压成**结构化变更链**。

它至少要回答 3 件事：

1. 进入这篇样本前，我们拿着什么旧判断
2. 是哪几个正文位置把旧判断逼到不够
3. 最后把哪个旧判断改成了什么新判断，并影响了哪些对象、规则或拓扑

如果只能看到结论，看不到“为什么改”，不算强留痕。  
如果只有原始聊天流，没有可比结构，也不算强留痕。

更重要的是：

**被推翻的旧判断，不是边角料，而是实验主产出。**  
做实验时，如果还不知道什么是对的，就先把什么是错的、为什么错、被什么击穿，系统记下来。

## 二、合格线

一份 `01_强留痕重跑记录.md` 至少满足下面 7 条，才算合格：

1. 每篇样本都有明确的入组前旧判断
2. 每篇样本都有主问题和主拓扑
3. 每个关键判断至少绑定 `3` 个正文位置或段落锚点
4. 每次对象/规则/拓扑变更，都写清“为什么改”
5. 每篇样本都有 `假设变更链`
6. 每篇样本都有 `废案保留`，而且废案不是附录凑数，要写成这篇样本最先贡献出来的负结论
7. 每篇样本都有最终处理结论：
   - 直接通过
   - 条件通过
   - 需修边界
   - 推翻旧定义

## 三、错案优先原则

强留痕默认先记“错了什么”，再记“改成什么”。

推荐顺序是：

1. 先写入组前旧判断
2. 再写哪几个正文位置击穿了它
3. 再写为什么旧判断不够
4. 最后才写新判断和影响范围

如果一篇样本只帮我们确认“旧判断不成立”，但还没给出稳定新答案，  
这篇样本仍然是有效实验结果，不能因为“还没得到正确答案”就降格处理。

## 四、推荐写法

一个中心的一份 `01_强留痕重跑记录.md`，建议按“总说明 + 分样本记录”来写。

### 总说明

- 当前轮次
- 当前中心
- 当前使用的候选方案版本
- 本轮样本范围
- 本轮重点要验证的对象/规则/拓扑

## 五、什么不算强留痕

下面这些都不算正式强留痕：

1. 只有最后结论，没有变更链
2. 只有对象名录，没有“为什么改”
3. 只有聊天原文，没有结构化判断
4. 只有证据锚点，没有旧判断和新判断之间的桥
5. 把已经推翻的猜测直接删掉，不留废案痕迹

## 六、和其他文件怎么分工

- `01_强留痕重跑记录.md`
  - 负责：记录每篇样本的判断链和变更链
- `02_对象边界修订.md`
  - 负责：把跨样本稳定成立的对象边界变化收敛出来
- `03_规则补丁表.md`
  - 负责：把跨样本稳定成立的规则变化收敛出来
- `99_证据锚点.md`
  - 负责：只存证据入口，不替代推理链
- `04/05_汇总补充结论.md`
  - 负责：互读后补充判断，不替代单样本强留痕

## 七、当前使用规则

从本模板起，正式结果里的“强留痕”统一按这份模板理解：

**保留推理过程，但不用原始聊天流做正式结果；正式结果统一保留为“旧判断 -> 触发证据 -> 新判断 -> 影响范围”的结构化变更链。**

并且默认遵守这条优先级：

**先找错，再收敛对；先保留被推翻的判断，再写最后留下来的判断。**
