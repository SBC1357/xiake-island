## 一、这一期只回答什么

`IIC` 只回答 6 个问题：

1. 在不同处理强度的闭卷素材包下，当前写作系统能否稳定写出成立的文章，而不是只会答“标准化例题”。
2. 当结构提示逐步减少时，系统能否自己完成问题框定、信息分层和主次取舍。
3. 在额外给定少量限制条件时，系统能否写出与原稿“同一家族”的作品，而不是靠记忆式模仿。
4. 当素材包中故意加入低价值噪音后，系统能否分辨主承重信息与次要信息。
5. `IIB` 遗留的两条边界，在闭卷条件下是否仍会持续分裂：
   - `问题现场群像稿`
   - `双人物共享核心轴`
6. 在不暴露原稿身份的前提下，系统能否对一篇已在 `IIA` 验证过的旧 `B` 组人物稿，写出人物核心轴、材料回钉方式和结构家族都足够接近的版本。

## 二、这一期不做什么

这一期先不做 4 件事：

1. 不直接给执行中心作者名、刊物名、原标题
2. 不直接给执行中心原文段落顺序
3. 不用原稿标志性金句做明牌提示
4. 不因为一次闭卷失败就立刻新增底层第五层

## 三、题型与难度结构

`IIC` 第一轮固定为 `4` 档主卷，加 `1` 道附加仿写对照题。

主卷仍是 `4` 篇文章、`4` 档压力测试，难度递增；附加题单列，不并入主门槛。

### A类：校准卷

目标：给高整理度素材包，先测系统是否至少会调用当前骨架，不至于连“公式怎么用”都答不出来。

- 固定样本：`养老院的春节“喘息服务”`
- 重点观察：`问题现场群像稿`
- 素材特征：允许出现 `事实卡 / 引语卡 / 关系卡 / 时间线 / 冲突点 / 任务要求 / 噪音卡`
- 判定地位：这是 `IIC` 的冒烟测试；如果大多数执行中心连 `A类` 都答不好，应直接反查 `IIA/IIB` 前面的有效性

### B类：半结构卷

目标：减少预处理强度，测试系统能否在提示变少后，自己完成基本的结构组织。

- 固定样本：匿名文章，由总协调另行冻结
- 约束：不得与 `IIA/IIB` 已跑样本重复
- 素材特征：保留事实、引语、时间信息和少量关系提示，但不再提供现成的冲突点总括

### C类：实战卷

目标：尽量逼近真实处理后素材池，测试系统能否在更脏、更杂、更接近生产环境的材料里自己找题眼。

- 固定样本：匿名文章，由总协调另行冻结
- 约束：不得与 `IIA/IIB` 已跑样本重复
- 素材特征：允许出现碎片化事实、混排引语、背景材料、轻微重复信息和明确噪音，但不替执行中心做主次排序

### D类：压限卷

目标：在接近实战卷的材料基础上，再附加有限约束，测试系统能否逼近原稿家族，而不是靠作者记忆按图索骥。

- 固定样本：`我和她，在相反的路上闪着光`
- 重点观察：`双人物共享核心轴`
- 素材特征：以 `C类` 或更接近 `C类` 的材料处理强度为底，再额外附加相似度限制条件
- 判定地位：这是第一轮最难的卷，主要测可控复现能力和抗过拟合能力

### 附加题：旧B稿仿写对照

目标：不重新证明底层骨架，而是单独测试“人物核心轴 + 人物-问题混合回钉”的控制精度上限。

- 固定样本：由总协调从 `IIA` 已验证 `B` 组人物稿中封存取样，不向执行中心公开原稿身份
- 判定地位：这是附加控制题，只作旁证，不并入 `IIC` 主通过门槛
- 素材特征：处理强度介于 `B类` 与 `D类` 之间；既给必要事实与引语，也给有限写法限制，重点看能否逼近旧 `B` 稿的结构家族，而不是写成普通企业人物稿或行业解释稿

## 四、闭卷素材包规则

每个闭卷素材包都只能从下面这些信息类型中取材：

1. `事实卡`
2. `引语卡`
3. `关系卡`
4. `时间线`
5. `冲突点`
6. `任务要求`
7. `噪音卡`

### 素材包禁止项

1. 作者名
2. 刊物名
3. 标题原文
4. 原文段落顺序
5. 标志性金句整段照搬
6. 任何能直接把执行者带回原稿身份的信息

### 各档允许的处理强度

1. `A类`
   - 可以给出完整卡片化信息
   - 可以显式写出冲突点和任务要求
   - 目的是测“会不会调用系统”
2. `B类`
   - 可以给事实、引语、时间线和少量关系提示
   - 不再给现成的冲突总表
   - 目的是测“能不能自己基本立起来”
3. `C类`
   - 只做必要清洗与去身份
   - 不做主次排序，不写现成结构提示
   - 目的是测“能不能在更像真实素材池的材料里完成判断”
4. `D类`
   - 材料复杂度不得低于 `C类`
   - 允许额外增加少量限制条件
   - 目的是测“能不能在压力条件下逼近原稿家族”

### 噪音卡要求

每个闭卷包都必须人为加入少量噪音，至少包含下面两类中的任意两类：

1. 低价值事实
2. 次要引语
3. 可删可不删的背景信息
4. 轻微重复信息

## 五、D类与附加题限制

`D类` 与附加题都可以额外给 `6` 类限制条件，但不能给作者身份：

1. `人格`
2. `主拓扑`
3. `情绪温度`
4. `证据密度`
5. `收尾方式`
6. `禁用项`

这里测的是“结构与气质是否接近原稿家族”，不是逐句模仿。

## 六、组织方式

本轮必须满足：

1. 出题方与答题方分离
2. 做素材包的人，不参与同一包的答题
3. 做素材包的人，不负责同一包的盲评

默认组织方式：

1. 总协调或独立中心负责出题与素材包制作
2. 三中心负责盲答
3. 盲评可由总协调统一收口

如果用户另行指定第四中心参与执行，可以，但必须继续满足“出题/答题/盲评分离”。

## 九、IIC 通过门槛

只有同时满足下面 5 条，才算 `IIC` 通过：

1. `A类` 至少大多数成稿达到“结构成立、信息取舍清楚、可读性强”的水平；如果 `A类` 出现系统性失守，`IIC` 直接不能写成通过
2. `B类` 与 `C类` 至少大多数成稿表明：在提示逐步减少后，系统仍能自己完成基本立意、结构组织和主次判断
3. `D类` 至少大多数成稿能在不暴露作者身份的前提下，逼近原稿的结构家族与气质家族
4. 加噪条件下，执行中心没有系统性地把次要信息误当主承重
5. `问题现场群像稿` 与 `双人物共享核心轴` 两条边界，至少有一条得到明显收束，另一条至少缩小分歧范围

附加仿写对照题不计入上述 `5` 条主门槛，只作旁证。

如果多数执行中心在附加题上都出现以下失真，应在 `IIC` 终验里单列“控制精度不足”：

1. 把人物稿写成普通行业解释稿
2. 把人物核心轴写丢
3. 把人物-问题混合回钉写成松散拼接

正式数值打分口径见：

## 十、这一期的收束判断

`IIC` 跑完后，只允许给出 3 种阶段判断：

1. `支持当前骨架，IIC 通过，进入 IID`
2. `支持当前骨架，但 IIC 需扩一轮`
3. `不支持当前骨架，建议回退 I 期`
