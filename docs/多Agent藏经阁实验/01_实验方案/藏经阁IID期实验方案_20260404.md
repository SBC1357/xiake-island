## 一、这一期只回答什么

`IID` 只回答 6 个问题：

1. 在素材相对充足时，AI 至少会不会把稿子写成立，不至于直接写崩。
2. 在素材过载、但字数被压缩时，AI 会不会主动删、敢不敢丢、能不能定主线。
3. 在素材稀缺、线索不够直接拼成完整稿时，AI 会不会提炼判断、搭桥过渡、把文章撑起来。
4. 在只给一个核心信息点和少量锚点时，AI 能不能在不胡编的前提下把文章往外推开。
5. `限制条件` 能不能真实改动文章结构、重点和表达，而不只是停留在表面服从。
6. `开放性要求` 能不能被转成有效写作动作，而不是口号式响应。

## 二、这一期不做什么

这一期先不做 5 件事：

1. 不让执行中心自行挑文章、自行定题。
2. 不复用 `IIC` 已经暴露过底层结构的旧题包。
3. 不把素材按成稿友好逻辑先分块再下发。
4. 不给作者名、刊物名、原标题和原文段落顺序。
5. 不因为某一篇写得像，就直接宣称系统已经成熟。

补充说明：

- `IID` 可以与 `IIC` 使用同一批底稿做纵向比较。
- 但只能采用“同底稿、重封装”方式重做需求包，不能直接把 `IIC` 的 `A/B/C/D` 正式包重新发给执行中心。

## 三、IID标准输入合同

`IID` 的标准题面固定由 4 部分组成：

1. `任务`
   - 例如：深度稿 / 特稿 / 人物稿 / 解释稿
2. `素材`
   - 客观事实、引语、关系、时间、背景、现场碎片与噪音混排
3. `限制条件`
   - 例如：字数、读者对象、强调点、禁写点、开头要求、去 AI 味儿
4. `开放性要求`
   - 例如：有趣一点、别太八股、把一个论点讲清楚、节奏顺一点

`IID` 测的不是“会不会答结构题”，而是：面对这类真实需求入口时，AI 基于现有规则最终会写成什么。

## 四、卷型结构

`IID` 第一轮固定为 `4` 卷主卷，不再按“脏不脏”分，而按真实生产里最容易拉开水平的`素材供给状态`来分。

### A类：充足素材基准卷

目标：保留一个最低基准，先看在素材相对充足时，系统会不会把稿子写崩。

- 核心问题：`会不会写`
- 重点观察：主线是否成立、文章是否基本完成、限制条件是否至少能落住
- 输入特征：有效素材充足但不过量，开放性要求较轻

### B类：富素材压缩卷

目标：在素材明显过载的情况下，把字数压到较低上限，逼系统做真正的删减和取舍。

- 核心问题：`会不会删`
- 重点观察：是否敢放弃次线、是否会把主线压清、是否会因舍不得删而写成拼盘
- 输入特征：有效素材明显多于成稿容量，且字数上限收紧

### C类：稀素材提炼卷

目标：在素材不够直接拼出完整稿时，测系统能不能靠提炼与搭桥把文章撑起来。

- 核心问题：`会不会撑`
- 重点观察：判断提炼、硬过渡、背景调用边界、少料成稿能力
- 输入特征：有效素材明显偏少，只够支撑一条主线，不够支撑面面俱到

### D类：单点起稿卷

目标：只给一个核心信息点与少量锚点，测系统能不能在边界内往外推开。

- 核心问题：`会不会推开`
- 重点观察：议题判断、作者积累替代能力、从一点外推到一篇文章的能力
- 输入特征：一个核心判断 + 少量锚点 + 很薄的背景许可边界

## 五、题包强制规则

1. 所有执行中心必须跑同一套统一题包。
2. 题包只允许以“需求单 + 素材池”的形式交付，不允许回到“结构卡组”。
3. 允许沿用 `IIC` 同底稿，但必须重做题面、重排素材、重混噪音，形成新的 `IID` 需求包。
4. 不允许用“事实 / 引语 / 关系 / 时间线 / 冲突点”或“起点 / 变化 / 新路子”这类标题，替执行中心先拆结构。
5. 不允许用标题性小节或排序方式偷偷提示成稿路径。
6. 素材池必须显式打乱顺序，并尽量使用单列混排。
7. `A类` 建议控制在 `12-16` 条有效素材，外加 `2-3` 条噪音。
8. `B类` 建议控制在 `18-24` 条有效素材，外加 `2-4` 条噪音，同时把成稿字数压到 `1500` 字上下。
9. `C类` 建议控制在 `6-10` 条有效素材，外加 `1-2` 条噪音，不再额外堆案例。
10. `D类` 建议控制为 `1` 个核心信息点、`4-6` 条锚点，再加极薄的背景许可边界与 `1-2` 条噪音。
11. 限制条件可以明确，但不得偷偷替执行中心完成编排判断。
12. 开放性要求可以保留模糊性，但不得反向泄漏原稿结构。
13. 出题方、答题方、盲评方必须分离。

## 六、组织方式

本轮默认按下面的方式组织：

1. 总协调或独立中心负责统一出题与需求包制作。
2. 三中心只负责盲答，不自行改题、不自行换题。
3. 盲评仍由总协调统一收口。
4. 原稿身份在盲评完成前不向执行中心公开。

## 九、IID通过门槛

只有同时满足下面 5 条，才算 `IID` 通过：

1. `A类` 至少大多数成稿能把文章写成立，不出现系统性跑偏或泛化八股文。
2. `B类` 至少大多数成稿表明：系统会主动删减、敢于放弃次线，而不是把富素材硬拼成清单。
3. `C类` 至少大多数成稿能在少料条件下完成判断提炼与结构搭桥，而不是用空话硬撑。
4. `D类` 至少大多数成稿能在不胡编的前提下，从单点与少量锚点往外推开，不直接塌成套话扩写。
5. 四卷综合后，能明确看出：当前系统的问题主要卡在“写 / 删 / 撑 / 推开”中的哪一层，而不是继续混成一团。

## 十、这一期的阶段判断

`IID` 跑完后，只允许给出 3 种阶段判断：

1. `支持当前骨架，IID 通过，进入下一阶段`
2. `支持当前骨架，但 IID 仍需扩一轮`
3. `不支持当前骨架，建议回退 I 期`
