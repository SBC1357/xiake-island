## 一、这一期只回答什么

`IIF` 只回答 5 个问题：

1. `IIE` 之前所有被正式说出来、并在实验链中被反复调用的“说法”，哪些是真正的最小单元。
2. 哪些现有说法其实是有限组合，不是最小单元，不能继续按单条规则或单条对象冻结。
3. 哪些现有说法更像：
   - `写作目的候选`
   - `条件性对象候选`
   - `稿型子型候选`
   - `旧母路线容器`
   - `观察项 / 暂不能判`
4. 经最小单元审计后，能否仍用有限单元组合解释 `IIE` 的 `32` 篇医学样本，而不再新增平行概念。
5. `IIF` 跑完后，是否支持：
   - 继续沿用并修订 `II期` 当前口径
   - 还是宣布 `II期` 临床失败并重开

## 二、这一期不做什么

这一期先不做 6 件事：

1. 不继续扩充 `IIE` 样本池，复验仍只用现有 `32` 篇工作集。
2. 不直接新增 `CO / GS` 条目，不把任何新说法推进为正式规则。
3. 不把“写作目的规则域”直接写成既成正式层级；它只能作为审计中的候选归属。
4. 不提前进入桥接层、生成层或侠客岛反推层执行。
5. 不因为 `IIF` 发现重大隐患，就在本期中途直接宣判 `II期` 整体失败。
6. 不把个别高风险样本的复杂写法，直接升格为新拓扑、新主路线或新本体层。

## 四、最小单元审计判断框架

每条前置说法都必须经过同一套判断，不允许只给结论不给拆法。

### 1. 每条说法的必填字段

1. `原始说法`
2. `来源路径`
3. `当前是否仍在正式载体中`
4. `最小单元判断`
   - 是
   - 否
   - 暂不能判
5. `若不是最小单元，它混入了什么`
6. `候选归属`
   - `写作目的候选`
   - `条件性对象候选`
   - `稿型子型候选`
   - `旧母路线容器`
   - `观察项 / 暂不能判`
7. `复验要求`
   - 用哪类样本去看它
   - 通过什么现象判断它是否还能成立

### 2. 允许的审计结论

每条说法只允许进入下面 5 档之一：

1. `最小单元，可保留`
2. `有限组合，不得再按单条冻结`
3. `旧容器，不是最小单元`
4. `观察项，暂不升格`
5. `暂不能判`

## 五、组织方式与角色划分

### 1. 总协调 / 实验中心

职责：

1. 冻结审计范围与样本边界
2. 维护阶段口径、目录治理、状态同步与强留痕
3. 不直接替各中心裁判具体说法
4. 只在 `IIF` 末尾做阶段裁决

### 2. `C0_审计整编中心`

职责：

1. 把 `IIE` 之前所有发现过的说法全部整编出来
2. 做去重、并列和来源标引，但不先行改名、不先行发明新概念
3. 在 `C1/C2` 裁判完成后，用审计后的有限单元集合回看 `32` 篇样本
4. 输出全量复验汇总，不提前写桥接方案

### 3. `C1_结构裁判中心`

职责：

1. 判断每条说法是不是最小单元，还是有限组合、旧容器或案例表达
2. 重点看：
   - 结构可拆性
   - 是否混入多个动作、多个承重判断或多个路线约束
   - 是否只是把旧路线下的多步动作包成一个新条目
3. 对复验中的高风险样本给出“还能否压回旧母路线”的结构裁决

### 4. `C2_功能裁判中心`

职责：

1. 判断每条说法的功能归属更像什么
2. 重点看它更像：
   - `写作目的候选`
   - `条件性对象候选`
   - `稿型子型候选`
   - `旧母路线容器`
   - `观察项 / 暂不能判`
3. 对复验中的高风险样本给出“它到底是对象问题、稿型问题，还是目的缝合问题”的功能裁决
