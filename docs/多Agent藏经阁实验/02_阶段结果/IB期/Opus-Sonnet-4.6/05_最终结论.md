# IB期最终结论 — Opus/Sonnet 4.6

## 本文件作用

基于 IB 期的持出拟合(03)、跨域重组(04)和对照组结果，对 IB 期四个核心问题给出判断。

**前置声明**：以下结论仅基于本轮 3 组新样本(人物向/半佛仙人/张佳伟) + I期基线(EGFR/戴老板/辰公子/空之客)的累积证据。阿基米德因 raw pool 缺口，未参与补硬，是本轮明确的证据空白。

---

## 问题一：拓扑层到底是否必须单列？

### 判断：**是，必须单列。**

### 证据链：

1. **同一作者不同文章拓扑不同**：张佳伟·天涯海角用 T06(同构叠加弧)，张佳伟·体育文则不用任何已录拓扑——如果不单列拓扑，就无法解释同一作者的这种结构差异。

2. **拓扑具有跨作者迁移性**：实验1(天涯海角拓扑×半佛动作)部分成功——说明拓扑骨架可以独立于动作/构件被提取和复用。

3. **拓扑不可还原为构件序列**：T06(同构叠加弧)不是"5个同构叠加块排在一起"那么简单——它还包含"情感递增方向""最后一段变奏许可""余韵收束点"等全篇级信息，这些信息不住在任何一个构件里。

4. **拓扑和构件之间存在亲和性约束**：实验2(编号分章×人物构件)失败——说明拓扑不是自由变量，它和构件之间有兼容性限制。这个"兼容性限制"本身就是拓扑层的属性，不是构件层能表达的。

### 限定：

- 拓扑的迁移不是无条件的——需要附加一条元规则："拓扑和构件之间存在亲和性，某些组合不兼容"
- 当前拓扑表只覆盖了"叙事型"和"论证型"两大类，对"混合分析型"文章(如张佳伟体育文)还有覆盖缺口

## 问题二：人格到底是否值得单列？

### 判断：**值得存在，但不应作为与四层平级的第五个底层对象。它应该被定位为"上层组合映射"。**

### 证据链：

1. **人格的内容可以完全用对四层的引用表达**：  
   P01(witness) = {偏好A01-A04, 偏好C01-C05, 偏好T01-T03, 激活R01-R05, 排斥论证结构}  
   它本身不携带任何不可还原到四层的底层信息。

2. **人格跨文章稳定、跨拓扑稳定**：  
   张佳伟的天涯海角(T06)和体育文(无明确已录拓扑)共享同一人格(P03/storyteller)——大白话判断、跨界嵌入、短句节奏在两篇中都出现。这证明人格确实是一个独立的、比拓扑更高层的稳定物。

3. **人格是"选什么组合"的指令，不是"有什么可选"的清单**——后者由四层负责。两者的抽象级别不同，不应并列。

### 具体建议：

藏经阁架构中，人格/persona_kernel 应该被存储为**映射配置文件**，引用四层中的对象ID——类似于 CSS 引用 HTML 元素，而非自己成为一种新的 HTML 标签。

## 问题三：人物向对象是否稳定存在？

### 判断：**人物向产生了新的、I期未见的动作和构件，确认为独立的有效入组样本。**

### 证据链：

1. **A01(矛盾画面开篇)、A02(引语阀门)、A03(讲述者交接)、C03(多声部群像块)** 均为本轮新增，I期样本(辰公子/空之客/戴老板)中未出现。

2. **T02(群像交叉弧)** 是全新拓扑——I期没有任何作者使用过多人物交替出场的全篇结构。

3. **R01(作者情绪不得超过受访者)** 是极强的硬约束，在其他所有作者中都不存在——半佛的作者情绪极强(痞气犀利)，张佳伟的作者情绪是"温柔命运感"(中等)，人物是"克制到几乎消失"。这个规则是人物向内核的核心标识。

### 限定：

- 本轮人物向的分析主要依赖灯塔蒸馏数据，原文池的 JSON 文件为 stub（仅含标题/URL），未获取到完整正文 HTML。这意味着持出拟合是"灯塔→四层"而非"原文→四层"——证据链比半佛和张佳伟弱一环。
- 如果后续要进一步补硬人物向，需要解决原文池的正文获取问题。

## 问题四：当前方案能否明显优于更简单的对照组？

### 判断：**是，四层+映射方案在核心需求维度上明显优于整包样板法和纯碎片法。**

### 对照组结果摘要（详见 04_重组复盘表）：

| 能力 | 整包样板 | 纯碎片 | 四层+映射 |
|------|----------|--------|-----------|
| 解释同作者不同文章差异 | ❌ | ❌ | ✅ |
| 精确定位"哪里像/不像" | ❌ | ⚠️ | ✅ |
| 支持跨域重组 | ❌ | ❌ | ✅ |
| 解释失败原因 | ❌ | ⚠️ | ✅ |

### 但需要注意的 trade-off：

- 四层+映射的**工程复杂度高于**其他两种方案——需要维护4个对象库 + 1个映射层 + 亲和性约束表
- 以藏经阁当前的规模和使命（长期写作知识资产库），这个复杂度是值得的
- 但当规模进一步扩大（比如50+作者、500+构件）时，**亲和性约束的组合爆炸**可能成为工程瓶颈——这是 II 期需要考虑的

## 阿基米德：明确证据缺口

- `staging/editorial/source_articles/` 下无阿基米德原文池目录
- 全局搜索 `**/ajmd*` 仅命中 persona_profiles（profile.json × 10个副本位置）
- **阿基米德在本轮 IB 期中不能被写成"已补硬"**
- 它是一个 **proxy-only 队列**——有 persona profile 但无 source articles
- 后续如需补硬阿基米德，需要先建立其 raw pool

## 总结：IB期对 I期方向的修正

| I期结论 | IB期修正 |
|---------|----------|
| 底层方向大概率是动作+构件+规则 | ✅ 确认，且**补充拓扑层为必须单列** |
| 作者风味更可能是人格 | ✅ 确认，并进一步定位为**上层组合映射而非底层对象** |
| 整包样板不能做底层主存储 | ✅ 确认，对照组验证其在4个核心维度上不如四层法 |
| 还很可能要补一层拓扑 | ✅ **确认必须补**——拓扑不可还原为构件序列，且具有独立迁移性 |

### IB期新增发现：

1. **拓扑-构件亲和性约束**：不是所有拓扑都能装所有构件，存在兼容性限制
2. **动作层存在"功能等价簇"**：不同作者的不同动作可能解决同一个功能问题（如引语阀门 ≈ 短句重锤在功能上都是"铺垫后的精准一击"）
3. **规则层存在"公约数规则"和"作者专属规则"两个子层**：前者是通用工艺，后者是人格映射的一部分
4. **当前模型对分析型/评论型文章有覆盖缺口**：动作表和拓扑表偏向修辞叙事型，需要扩展
5. **阿基米德是明确的 raw pool 证据缺口**，不能作为已验证样本
