### R4_隐喻承重触发规则

```python
Rule(
    rule_id="R4",
    family=RuleFamilyId.NARRATIVE_RULES,
    name="隐喻承重触发规则",
    description="何时用隐喻vs直接分析",
    severity=RuleSeverity.HIGH,
    checker=lambda context: (
        context.concept_abstraction == "high" and
        context.direct_explanation_memory_score < 0.6
    ),
    condition="""
    触发条件:
    1. 概念抽象度高(如"真实稀缺性""企业文化""命运")
    2. 直接解释难以产生记忆点
    3. 需要情感共鸣而非纯逻辑说服
    
    不触发条件:
    1. 概念具体可量化(如"营收增长13.3%")
    2. 直接解释已足够清晰
    3. 技术性/学术性文章
    """,
    on_match="使用承重比喻",
    on_fail="未使用承重比喻",
    suggestion="""
    建议使用承重比喻,如:
    - "玻尿酸成精的伪人感"(C1)
    - "体育克苏鲁"(C2)
    - "清越的笛声"(C4)
    """,
    failure_degradation="""
    失败降级:
    1. 比喻不恰当或产生歧义 → 改用"对比案例+数据支撑"组合
    2. 比喻过度导致理解困难 → 简化比喻,保留核心意象
    3. 领域不适配 → 放弃隐喻,改用直接分析
    """,
    enabled=True
)
```

**验证证据**:
- C1第1段:"玻尿酸成精的伪人感" - 抽象概念"AI评论的伪人感"用比喻承载
- C2第1段:"体育克苏鲁" - 抽象概念"安踏的恐怖实力"用比喻承载
- C3/C4较少使用隐喻,改用意象回归,验证"不适用边界"

### R6_级联vs平行选择规则

```python
Rule(
    rule_id="R6",
    family=RuleFamilyId.NARRATIVE_RULES,
    name="级联vs平行选择规则",
    description="何时用级联拓扑vs平行拓扑",
    severity=RuleSeverity.HIGH,
    checker=lambda context: (
        context.argumentation_type == "progressive"
    ),
    condition="""
    触发条件(级联):
    1. 论证需要递进式"层层加码"(如C1三层防线每层更难攻破)
    2. 每层论证依赖前一层结论
    3. 最终结论比中间结论更强
    
    触发条件(平行):
    1. 多个证据独立支撑同一论点
    2. 证据之间无依赖关系
    3. 可任意顺序排列
    
    不适用:
    1. 单一论点文章无需选择
    2. 混合型文章可能同时使用两种拓扑
    """,
    on_match="选择级联拓扑",
    on_fail="选择平行拓扑",
    suggestion="""
    建议根据论证类型选择:
    - 递进论证 → 级联(如C1三层防线、C2五层"猛")
    - 并列论证 → 平行(如多人物群像、多案例对比)
    """,
    failure_degradation="""
    失败降级:
    1. 级联中某层证据不足 → 合并弱层,减少层数,保证每层证据充分
    2. 平行中某证据过强或过弱 → 调整证据权重,保持平衡
    3. 混合型文章 → 分段使用不同拓扑,明确标记边界
    """,
    enabled=True
)
```

**验证证据**:
- C1第2-4章: 文本侧写→行为侧写→物理锚定,三层递进,验证"级联拓扑"
- C2第1-5章: 五层"猛"递进,验证"级联拓扑"
- C3: 对话体无需级联vs平行选择,验证"不适用场景"
- C4: 人物真相钻头=回钻弧(级联变体),验证"级联拓扑变体"

### R7_开篇类型选择规则

```python
Rule(
    rule_id="R7",
    family=RuleFamilyId.NARRATIVE_RULES,
    name="开篇类型选择规则",
    description="何时用问题锚定vs事实锚定开篇",
    severity=RuleSeverity.HIGH,
    checker=lambda context: (
        context.article_type in ["financial_report", "data_analysis"]
    ),
    condition="""
    触发条件(事实锚定):
    1. 文章主题为"财报/数据解读"(如C2安踏财报)
    2. 关键数据/事实具有强吸引力
    3. 需要快速建立权威感
    
    触发条件(问题锚定):
    1. 文章主题为"观点论证"(如C1 AI评论治理)
    2. 问题本身具有悬念或争议性
    3. 需要引发读者思考
    
    触发条件(对话体开篇):
    1. 文章主题为"揭示被遮蔽的真相"(如C3不喜欢工作)
    2. 需要通过第三方视角建立说服力
    3. 对话本身具有吸引力
    
    触发条件(人物锚定开篇):
    1. 文章主题为"人物解读"(如C4金镶玉)
    2. 人物出场具有戏剧性
    3. 需要快速建立人物印象
    """,
    on_match="根据文章类型选择开篇",
    on_fail="使用问题锚定开篇(默认)",
    suggestion="""
    建议根据文章类型选择:
    - 财报/数据解读 → 事实锚定开篇(如C2)
    - 观点论证 → 问题锚定开篇(如C1)
    - 揭示真相 → 对话体开篇(如C3)
    - 人物解读 → 人物锚定开篇(如C4)
    """,
    failure_degradation="""
    失败降级:
    1. 事实锚定缺乏悬念 → 事实锚定后立即引入问题锚定("为什么安踏能这么猛?")
    2. 问题锚定过于平庸 → 改用"场景锚定开篇"(如C3对话场景)
    3. 开篇类型不适配 → 中期调整,在第二章切换到适合的开篇类型
    """,
    enabled=True
)
```

**验证证据**:
- C1第1段: "看两条评论: A...B...你猜哪条是AI写的?" - 问题锚定开篇
- C2第1段: "安踏今天发了财报。依然和之前一样猛" - 事实锚定开篇
- C3开篇: "您好,我来应征撸猫工作的!" - 对话体开篇
- C4第1段: "金镶玉出场时,100分钟的电影已近1/4了" - 人物锚定开篇
- 四种开篇类型均验证,确认"开篇类型选择规则"必须补硬

### R9_对话体vs论述体选择规则

```python
Rule(
    rule_id="R9",
    family=RuleFamilyId.NARRATIVE_RULES,
    name="对话体vs论述体选择规则",
    description="何时用对话体vs论述体叙事",
    severity=RuleSeverity.HIGH,
    checker=lambda context: (
        context.theme in ["reveal_hidden_truth", "break_common_sense"]
    ),
    condition="""
    触发条件(对话体):
    1. 主题涉及"揭示被遮蔽的真相"(如C3不喜欢工作的真相)

    3. 需要情感共鸣而非纯逻辑说服

    触发条件(论述体):
    1. 主题涉及"数据解读""技术分析"
    2. 需要严谨的逻辑论证
    3. 数据和案例占主导
    4. 不需要情感共鸣
    
    不适用:
    1. 财报分析类文章不适用对话体
    2. 情感共鸣类文章不适用纯论述体
    """,
    on_match="选择对话体叙事",
    on_fail="选择论述体叙事",
    suggestion="""
    建议根据主题选择:
    - 揭示真相/打破常识 → 对话体(如C3人vs猫对话)
    - 数据分析/技术解读 → 论述体(如C1/C2编号分章)
    """,
    failure_degradation="""
    失败降级:
    1. 对话体难以承载复杂论证 → 对话体用于开篇和结尾,中间改用论述体+案例支撑
    2. 论述体缺乏情感共鸣 → 在关键论点插入"引语阀门块"或"感官载情块"
    3. 混合型文章 → 分段使用不同叙事体,明确标记边界
    """,
    enabled=True
)
```

**验证证据**:
- C3全文: 人vs猫对话体叙事,揭示"工作本质是枯燥"的真相
- C1/C2: 论述体叙事,数据分析和论证占主导
- C4: 人物稿,介于对话体和论述体之间,用"人物真相钻头"叙事

### R8_重复锤击强度规则

```python
Rule(
    rule_id="R8",
    family=RuleFamilyId.STYLE_RULES,
    name="重复锤击强度规则",
    description="重复多少次合适",
    severity=RuleSeverity.MEDIUM,
    checker=lambda context: (
        context.keyword_repetition_count >= 3 and
        context.keyword_repetition_count <= 5
    ),
    condition="""
    触发条件:
    关键判断词重复3-5次为宜,强化印象但不削弱新鲜感
    
    过度重复(>5次):
    - 审美疲劳,削弱新鲜感
    - 显得刻意,降低说服力
    
    重复不足(<3次):
    - 印象不够深刻
    - 失去强化效果
    """,
    on_match="重复锤击强度适中",
    on_fail="重复锤击强度不合适",
    suggestion="""
    建议:
    - 半佛仙人风格: 关键词重复3-5次(如C2"猛"字20+次为过度,建议控制在5次左右)
    - 其他风格: 关键词重复2-3次即可
    """,
    failure_degradation="""
    失败降级:
    1. 重复过度 → 减少重复次数,改用"数据支撑+对比案例"替代部分重复
    2. 重复不足 → 在关键论点增加关键词重复,强化印象
    """,
    enabled=True
)
```

**验证证据**:
- C2全文: "猛"字出现20+次,过度重复,验证"重复过度降级方案"
- C1/C3/C4: 关键词重复适中,验证"重复锤击强度规则"

### R11_人物稿vs事件稿区分规则

```python
Rule(
    rule_id="R11",
    family=RuleFamilyId.ARTICLE_TYPE_RULES,
    name="人物稿vs事件稿区分规则",
    description="何时用人物真相钻头拓扑",
    severity=RuleSeverity.MEDIUM,
    checker=lambda context: (
        context.article_type == "character_profile"
    ),
    condition="""
    触发条件(人物稿):
    1. 主题为"揭示某个人物的本质"(如C4金镶玉)
    2. 围绕人物的一个骨相反复回钻
    3. 需要展示人物复杂性
    
    触发条件(事件稿):
    1. 主题为"分析某事件/现象"(如C1 AI评论治理)
    2. 系统性拆解事件原因/影响/对策
    3. 需要逻辑论证而非人物刻画
    
    不适用:
    1. 财报分析类文章不适用人物稿拓扑
    2. 人物稿不适用事件稿拓扑
    """,
    on_match="选择人物真相钻头拓扑",
    on_fail="选择事件分析拓扑",
    suggestion="""
    建议根据主题选择:
    - 揭示人物本质 → 人物真相钻头(如C4围绕"金镶玉讨厌虚伪"回钻)
    - 分析事件/现象 → 事件分析拓扑(如C1三层防线递进)
    """,
    failure_degradation="""
    失败降级:
    1. 钻头过度导致重复 → 减少钻头次数,改用"对比案例+场景描写"替代部分钻头
    2. 人物稿缺乏深度 → 增加钻头次数,围绕骨相多角度揭示
    """,
    enabled=True
)
```

**验证证据**:
