# GPT-5.4 IIA期对象边界修订

状态：对象边界修订版  
日期：2026-04-03  
修订范围：B组人物向对象

## 证据摘要

- B1 `稚晖君` 证明人物稿可以围绕单一驱动力反复回钉，公众标签只是外壳，不是答案。
- B2 `朱易` 证明人物稿可以让第一人称长段承重，但前提是它持续推进“如何继续滑”的核心轴，而不是散点抒情。
- B3 `沈亚楠` 证明人物稿可以与问题稿叠合，行业细节照样能入稿，但必须回收到人物核心轴，否则应降级回问题稿。

## 修订结论

| 边界项 | 修订后定位 | 是否进入通用核心对象 | 修订理由 |
|---|---|---|---|
| `人物核心轴` | 条件性对象 / 稿型子型 | 否 | 三篇样本都稳定存在，但它控制的是整篇选材与回钉方向，不是可直接跨稿复用的最小底层件 |
| `公众标签壳` | 外压材料，不单列对象 | 否 | `天才少年`、`科学家父亲`、`前理想总裁` 都只是外部认知壳，正文必须穿透，不应误存为人物真相 |
| `人物-问题混合回钉` | 拓扑亲和性说明 | 否 | B3 说明人物稿可吞问题稿材料，但无需为此新立一个大拓扑类，只要补亲和性和降级规则 |

## 为什么不建议把人物向新增写成通用对象

### 1. 三篇样本的“轴值”差异过大

- B1 是 `急迫`
- B2 是 `被误解后的自我校准与继续滑`
- B3 是 `对不合理的零容忍 / 对确定性的偏执`

它们稳定的是“有一条轴”，而不是“轴的内容具有通用最小复用性”。

### 2. 人物核心轴控制的是入稿资格，不是最小内容件

- 它决定哪些材料能进、哪些材料要删、哪些段落必须回收
- 它更接近文章级控制对象，而不是底层 `硬事实 / 场景 / 引述 / 判断` 那类最小件

### 3. B3 说明它不是纯人物稿专属世界观层

- 一旦问题解释部分占比升高，文章仍能成立
- 真正决定文章没有滑成普通解释文的，是“问题链要不断回人”，不是另起一层全新底座

## 修订后的明确边界

### `人物核心轴` 是什么

- 是人物稿的主问题子型
- 是一条可被反复回钉的人物内核判断
- 可以表现为驱动力、伤口、执念、反向问题、价值冲突
- 负责约束材料筛选、段落回收、收尾拔高

### `人物核心轴` 不是什么

- 不是公众标签
- 不是人物履历总表
- 不是作者风格
- 不是单独可脱离人物稿而通吃所有稿型的通用底层对象

## 建议的存储方式

如果 IIA 期候选方案 `v1` 要把它收进去，建议只作为“条件性对象 / 稿型子型”保存，并至少带 5 个字段：

1. `axis_statement`
   - 这篇人物稿最终要回答的那条核心轴是什么
2. `axis_type`
   - 驱动力 / 伤口 / 执念 / 反向问题 / 价值冲突
3. `shell_pressure`
   - 外部公众标签或误读是什么
4. `return_anchors`
   - 文章在哪些关键段落反复回钉这条轴
5. `downgrade_condition`
   - 证据不足时，应该降级成事件稿还是问题稿

## 对现有骨架的影响

- 不推翻 `动作 / 构件 / 规则 / 拓扑`
- 不要求新增拓扑大类
- 需要在对象分级里补入 `人物核心轴` 这一类条件对象
- 需要在规则表里补入人物稿的回钉规则、标签穿透规则、混合稿降级规则

## 本中心边界结论

本轮对 B 组的独立判断是：

- `人物向对象稳定存在`
- `但它不是通用对象，而是稿型子型`
- `因此结论归类为：支持当前骨架，但需补一类对象`
