## 候选样本 1

### 标题

脑机接口门诊的一个下午

### 路径

`../../../../../../藏经阁/staging/editorial/source_articles/bingdian/2390471121-2653345690_1.json`

### 稿型判断

`现场报道 + 问题稿 + 技术/政策解释` 混合稿。

### 混合点

1. 文章不是纯技术解读，而是从协和门诊一个下午的现场切入。
2. 文章不是单一人物稿，而是用多位患者和医生构成“问题现场群像”。
3. 文章中后段不断把门诊见闻回收到脑机接口的技术边界、康复成本和政策推进。

### 预估主人格

`P01 witness / 克制的见证者` 为主，叠加 `P04 analyst / 冷静分析师`。

- `P01` 成分在于门诊现场、患者动作、医生口径和情绪克制。
- `P04` 成分在于适应症、疗效边界、医保试点、国家标准与临床进展的解释密度。

### 预估主拓扑

`问题现场推进 -> 个案并列回收 -> 技术边界解释 -> 政策抬升收束`。

它更像 `T02 群像交叉弧` 和分析型拓扑簇的叠合，但目前还缺一条更明确的“问题现场群像稿”命名。

### 预估触发的条件性对象 / 稿型子型

- `CO-02 解释桥`：脑机接口、无创/有创差异、康复边界都需要被翻译成人话。
- `CO-06 政策推演`：医保试点、国标立项、临床试验推进会直接改变稿件后半段判断重心。
- `CO-07 分析型拓扑簇`：它已经不是纯叙事弧，而是现场与解释并联推进。
- 候选新稿型：`问题现场群像稿`。

### 最可能挑战的边界

当前体系能否稳定处理“没有单一主人公、也不是纯分析”的门诊现场稿。

### 验证样本 / 挑战样本

`挑战样本`

### 如果失败，最可能暴露什么结构问题

1. 现有路由过度依赖“单一人物核心轴”或“纯分析型拓扑”，中间地带会被误分。
2. 现有拓扑表对“现场报道型问题稿”覆盖不足。
3. 条件性对象表里缺少“问题现场如何把多个个案回收到同一问题”的约束件。

## 候选样本 2

### 标题

影石十年：走进那片硬件的迷雾森林

### 路径

`../../../../../../藏经阁/staging/editorial/source_articles/dailaoban/articles/DBA-20250709-0049.json`

### 稿型判断

`公司史 + 品类史 + 创始人叙事 + 产业分析` 混合稿。

### 混合点

1. 开头先从宝丽来和“品类定义型产品”起手，不是直接讲影石公司史。
2. 中段把刘靖康的学习曲线、产品试错和公司关键转折穿起来。
3. 后段再切到供应链红利、软硬一体化、对 GoPro 的反超与下一阶段目标。

### 预估主人格

`P04 analyst / 冷静分析师` 为主，带明显的“宏大叙事深稿”倾向。

- 核心驱动仍是分析：品类、供应链、竞争位势、产品能力。
- 但文章并不满足于冷分析，还要把公司十年路径写成一条可被记住的创业长弧。

### 预估主拓扑

`品类史开口 -> 创业试错 -> 供应链分层推进 -> 竞争反超 -> 未来问题回收`。

它最接近 `TN-01 阶梯推进弧`，但比纯行业分析更强地嵌入了创始人与产品试错的叙事层。

### 预估触发的条件性对象 / 稿型子型

- `CO-02 解释桥`：把全景相机、运动相机、软硬一体化等复杂概念持续降维。
- `CO-03 数据锤击`：市场份额、收入、迭代速度、产品矩阵不断购买判断。
- `CO-04 权威锚定`：GoPro、苹果、宝丽来、供应链老师傅等参照系承担背书作用。
- `CO-07 分析型拓扑簇`：主体推进方式已经明显脱离纯人物或纯散文。
- 候选新稿型：`公司史-品类史混合稿`。

### 最可能挑战的边界

公司成长深稿到底应被路由为“公司问题稿”、还是“品类分析稿”、还是“创业人物稿”。

### 验证样本 / 挑战样本

`挑战样本`

### 如果失败，最可能暴露什么结构问题

1. 当前体系还不能稳定区分“创始人是串线器”与“创始人是真正主轴”。
2. 现有分析型拓扑对“品类史开口后再回公司”的长弧覆盖不足。
3. 人格映射上会出现“全篇都被压成 P04”，从而丢掉这类深稿真正的叙事势能。

## 候选样本 3

### 标题

乳腺癌战场上，一个医生和她的野心

### 路径

`../../../../../../藏经阁/staging/editorial/source_articles/renwu/2103095721-2653247586_2.json`

### 稿型判断

`人物稿 + 问题稿 + 演讲体科普` 混合稿。

### 混合点

1. 它有人物稿核心轴，廖宁的“野心”不是口号，而是整篇的入稿标准。
2. 它不是纯人物专访，主体材料大量落在乳腺癌治疗进步、保乳重建、临床误区和患者处境。
3. 它又不是普通医学科普，因为所有专业解释最后都要回到“为什么是她这样做、为什么她要这样推进”。

### 预估主人格

`P01 witness / 克制的见证者` 为主，叠加 `P04 analyst / 冷静分析师`。

- `P01` 成分在于患者处境、身体尊严、职业使命与长期陪伴。
- `P04` 成分在于临床路径、疗效判断和诊疗进展解释。

### 预估主拓扑

`人物使命起手 -> 临床问题分层解释 -> 患者意义回收 -> 价值宣言收束`。

它仍可落在“问题先行的人物-问题混合拓扑”附近，但写法是演讲体自证，不是常规报道体。

### 预估触发的条件性对象 / 稿型子型

- `GS-01 人物核心轴 / 人物真相钻头`：整篇反复回钉的是“医学未来 + 身体尊严”的野心。
- `CO-02 解释桥`：保乳、重建、精准治疗、分级检查都要被翻译。
- `CO-03 数据锤击`：40%/20% 等比例和诊疗进展承担说服任务。
- `CO-04 权威锚定`：法国进修、国际会诊平台、机器人辅助手术等专业资历承担背书。
- 候选新稿型：`演讲体人物-问题稿`。

### 最可能挑战的边界

人物核心轴能否稳定承载强专业、强公共说服任务，而不滑成“演讲稿整理”或“医学宣教稿”。

### 验证样本 / 挑战样本

`验证样本`

### 如果失败，最可能暴露什么结构问题

1. `GS-01` 只在采访/报道型人物稿中稳定，在演讲体里会失真。
2. 人物稿与问题稿叠合时，一旦专业解释过强，主轴就会从“人”滑到“议题”。
3. 当前规则还缺“演讲体如何防止价值宣言提前盖过正文证据”的约束。

## 本中心最终推荐顺序

1. `脑机接口门诊的一个下午`
   - 最值得优先进入 IIB。
   - 原因是它最像真正的新稿型，能直接测试当前体系是否已经准备好处理“问题现场群像稿”。

2. `影石十年：走进那片硬件的迷雾森林`
   - 作为第二优先级。
   - 原因是它同时打公司史、品类史、创始人叙事和产业分析四条线，适合测试路由与拓扑的分层能力。

3. `乳腺癌战场上，一个医生和她的野心`
   - 作为第三优先级。
   - 原因是它更偏验证样本，主要价值是确认“人物核心轴”在强专业、强公共议题下是否仍成立。
