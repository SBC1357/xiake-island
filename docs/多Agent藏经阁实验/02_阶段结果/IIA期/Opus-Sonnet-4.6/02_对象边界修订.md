# IIA期对象边界修订 — Opus/Sonnet 4.6

日期：2026-04-03
依据：A1、A2、D1-D4 强留痕重跑结果

---

## 一、IB 期已有对象回顾

IB 期 Opus/Sonnet 4.6 已确认的底层四层对象：

- **动作（Actions）**：I 期 8 个基础动作（压力起手、关键问题钉子、历史回望链、解释桥、比喻承重、细节买权、失败收编、边界后拔高）+ IB 期 14 个新增（A01-A14，覆盖人物向/半佛/张佳伟）
- **构件（Components）**：C01-C14
- **拓扑（Topology）**：T01-T08
- **人格（Persona）**：P01-P03（上层映射）

## 二、IIA 期新增候选动作

以下动作从 D 组阿基米德样本 + A 组锚点校准中提取，均为 I/IB 期叙事类样本中未出现的行业分析类动作。

| 动作 ID | 名称 | 定义 | 来源样本 | 与已有动作的关系 |
|---------|------|------|----------|-----------------|
| DA-01 | 数据锤击 | 用密集硬数据（临床终点/财务指标/市场规模）碾压式购买判断 | A1, D1, D2, D4 | 与半佛 A07"重复锤击"功能等价：前者靠数据体量，后者靠文字重复 |
| DA-02 | 专业降维 | 将专业术语/机制翻译成日常可感知的画面 | A1 | 与半佛 A06"降维类比"功能对位：降维起点的专业深度更硬 |
| DA-03 | 框架重分类 | 提出新分类替代旧分类，重新组织整个认知结构 | A1, D1 | 新动作，I/IB 期无直接对应 |
| DA-04 | 案例递进 | 多个案例按逻辑递进排列，每个代表蜕变的一个阶段 | A2 | 与张佳伟 A09"同构叠加"区别：叠加换人不换模板，递进换内容且升级深度 |
| DA-05 | 转折节点钉子 | 用关键事件/时刻钉住"从此以后不一样了" | A2 | 新动作，强于"关键问题钉子"——后者钉问题，前者钉转折 |
| DA-06 | 权威锚定 | 引述已验证权威的行为（而非言论）为判断背书 | D1 | 新动作，I/IB 期无对应 |
| DA-07 | 算账推演 | 显式列出计算过程让读者自行验证判断 | D1 | 新动作，I/IB 期无对应 |
| DA-08 | 风险解剖 | 系统拆解某靶点/赛道/策略的失败原因链和传导链 | D2 | 与"失败收编"区别：后者是在成功叙事里收编失败节点，前者是以风险/失败为主角做系统拆解 |
| DA-09 | 宏微切换 | 在宏观格局和微观个案之间来回切换推进 | D2 | 新动作，I/IB 期无对应 |
| DA-10 | 政策推演 | 从政策规则设计出发推演参与者行为和产业后果 | D3 | 新动作，I/IB 期无对应 |
| DA-11 | 双方视角对切 | 用当事双方的不同处境和选择解释同一事件 | D3 | 新动作，与人物向 A03"讲述者交接"结构上类似但功能不同 |
| DA-12 | 板块级判断 | 对一个行业/板块做整体投资判断 | D4 | 新动作，I/IB 期无对应；是"关键问题钉子"的板块版本 |
| DA-13 | 多路径枚举 | 并列展示多个可选路径/模式，各自优劣不做排名 | D4 | 新动作，I/IB 期无对应 |

## 三、新增候选动作的功能簇归类

以上 13 个候选不应当成 13 个独立底层对象。按功能归簇后，可以压缩为 **3 个功能簇**：

### 功能簇 F1：证据购买类

包含：DA-01 数据锤击 / DA-02 专业降维 / DA-06 权威锚定 / DA-07 算账推演

共同点：都是"用什么手段购买判断可信度"。与 I 期"细节买权"和"比喻承重"属于同一功能家族，只是购买手段不同。

**建议**：将"细节买权"升级为"证据购买"功能槽，下挂 4 种实现方式：叙事细节 / 硬数据 / 专业降维 / 权威行为 / 算账推演。

### 功能簇 F2：结构推进类

包含：DA-03 框架重分类 / DA-04 案例递进 / DA-05 转折节点钉子 / DA-09 宏微切换 / DA-10 政策推演 / DA-11 双方视角对切

共同点：都是"文章在逻辑层面怎么推进"。与 I 期"压力起手""历史回望链""失败收编"同属推进类，只是推进手法因稿型而异。

**建议**：不合并为一个对象，但在对象分级表中标为"结构推进类·条件性对象"——只在对应稿型中出现。

### 功能簇 F3：判断输出类

包含：DA-08 风险解剖 / DA-12 板块级判断 / DA-13 多路径枚举

共同点：都是"文章最终输出什么类型的判断"。与 I 期"边界后拔高"同属判断输出类。

**建议**：标为"判断输出类·条件性对象"。

## 四、已有对象的边界修订

| 对象 | 修订内容 | 原因 |
|------|----------|------|
| 比喻承重 | 降级为条件性对象 | D 组 4 篇阿基米德稿几乎不用比喻，证明比喻承重不是所有文章的必选项。它在叙事类稿型中仍然核心，但在分析类稿型中可以缺席 |
| 失败收编 | 扩展定义 | 原定义"失败不是污点，而是逼出系统性答案的节点"仍然成立，但 D2 证明还存在一种变体——"失败不是被收编的节点，而是被解剖的主角"（DA-08 风险解剖） |
| 细节买权 | 扩展为"证据购买"功能槽 | I/IB 期的细节买权偏叙事（场景/物件/对话），D 组证明还有数据锤击/算账推演/权威锚定等非叙事购买手段 |
| 解释桥 | 保持不变 | A1 的"亚特兰蒂斯"、D1 的"渠道即处方"都是解释桥的变体，覆盖充分 |

## 五、新增候选拓扑

| 拓扑 ID | 名称 | 结构描述 | 来源样本 |
|---------|------|----------|----------|
| TN-01 | 阶梯推进弧 | 编号章节逐层递进加码，每层在上一层基础上 | D1 |
| TN-02 | 双层递进弧 | 先铺宏观层（行业格局），再切微观层（个案拆解），微观内部再递进 | D2 |
| TN-03 | 时间线对比弧 | 在历史正面和当前负面之间做时间线对比 | D3 |
| TN-04 | 并列推进弧 | 两条或多条逻辑线并列推进，各自有数据支撑 | D4 |

与 IB 期 T01-T08 相比，TN-01 到 TN-04 的显著特征是：**它们全部是论证/分析型拓扑，而 T01-T08 主要是叙事/抒情型拓扑**。

这证明拓扑层的覆盖范围需要从"叙事型"扩展到"分析型"——但拓扑层本身的定位不变，仍然是独立于构件的全篇级骨架。

## 六、对象分级总结

按当前 IIA 的证据，对象可以分成 4 档：

### 核心稳定对象

这些在所有样本中都命中，或在绝大多数样本中命中：

- 压力起手
- 关键问题钉子
- 细节买权 → 升级为"证据购买"功能槽
- 历史回望链
- 边界后拔高

### 条件性对象

只在特定稿型中出现，但出现时承重很高：

- 比喻承重（叙事型必选，分析型可缺席）
- 失败收编（成功叙事型核心，分析型变体为"风险解剖"）
- 解释桥（复杂机制类必选，简单分析类可缺席）
- DA-01 数据锤击（分析型核心，叙事型可缺席）
- DA-06 权威锚定（行业分析型）
- DA-08 风险解剖（风险类分析核心）
- DA-10 政策推演（政策分析型）

### 作者子型 / 稿型子型

只在特定作者或特定稿型中出现：

- A01-A04 人物向动作
- A05-A08 半佛仙人动作
- A09-A14 张佳伟动作
- DA-03 框架重分类
- DA-04 案例递进
- DA-07 算账推演
- DA-12 板块级判断
- DA-13 多路径枚举

### 案例保留对象

当前只有单一样本支撑，不够升级但值得保留观察：

- DA-05 转折节点钉子（仅 A2）
- DA-09 宏微切换（仅 D2）
- DA-11 双方视角对切（仅 D3）
