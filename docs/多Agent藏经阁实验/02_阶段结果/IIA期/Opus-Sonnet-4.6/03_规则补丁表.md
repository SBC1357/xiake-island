## 二、IIA 期新增规则

以下规则从 D 组阿基米德补硬 + A 组锚点校准中提取，全部附触发条件和失败降级。

### RN-01 专业数据购买规则

| 项目 | 内容 |
|------|------|
| 规则 | 在含临床/行业数据的分析稿中，判断的购买成本不是叙事细节而是数据硬度（样本量/终点指标/安全性对比/财务指标） |
| 级别 | 硬约束（分析类稿型） |
| 触发条件 | 文章核心论证依赖专业数据时触发 |
| 失败降级 | 如果手头一手数据不够，降级为"引用二手数据 + 附来源标注" |
| 来源样本 | A1（EGFR P52-P59 临床数据对比）、D1（AP306 临床数据）、D2（CD47 ORR 对比）、D4（全文数据驱动） |
| 与已有规则关系 | 是 G01"判断由事实购买"在分析类稿型中的具化版本 |

### RN-02 先拆旧再建新

| 项目 | 内容 |
|------|------|
| 规则 | 提出新分类/新框架/新模型前，必须先系统拆解旧框架的缺陷 |
| 级别 | 硬约束（框架重分类型文章） |
| 触发条件 | 文章提出新分类或重新定义已知概念时触发 |
| 失败降级 | 如果篇幅不够，压缩为"一句话指出旧方法的核心缺陷是什么" |
| 来源样本 | A1（P12-P24 系统拆解外显子分类→P27-P54 MDACC 新分类） |
| 证据锚点 | EA-04, EA-05, EA-08 |

### RN-03 蜕变链规则

| 项目 | 内容 |
|------|------|
| 规则 | 展示"从 A 到 B"的蜕变时，必须同时出示旧状态和新状态的具体细节，不能只说"变好了" |
| 级别 | 偏好 |
| 触发条件 | 公司转型/个人成长/行业演进类文章中触发 |
| 失败降级 | 如果缺旧状态细节，降级为"一句话概括旧状态 + 详细展示新状态" |
| 来源样本 | A2（2016 低谷具体细节 → MIX 具体细节 → 芯片/AI/汽车具体细节） |
| 证据锚点 | EA-09, EA-10, EA-11 |

### RN-04 第三方背书规则

| 项目 | 内容 |
|------|------|
| 规则 | 判断越大，越需要非自我视角的证据（第三方评价/权威引述/行业数据） |
| 级别 | 偏好 |
| 触发条件 | 大判断（"改变了"/"第一次"/"最具备"等绝对化表述）前触发 |
| 失败降级 | 降级为直接引用可验证的具体数据/成果 |
| 来源样本 | A2（《连线》评价、格鲁夫引语、苹果类比）、D1（DaVita 投资行为） |
| 证据锚点 | EA-12, ED-01 |

### RN-05 权威行为优于权威言论

| 项目 | 内容 |
|------|------|
| 规则 | 引述权威做了什么（投资/押注/采购）比引述权威说了什么更有说服力 |
| 级别 | 偏好（行业分析类） |
| 触发条件 | 行业分析/公司拆解文章中，需要引述权威增强可信度时触发 |
| 失败降级 | 如果没有权威行为可引述，降级为行业数据背书 |
| 来源样本 | D1（巴菲特投资 DaVita → DaVita 投资 R1 Therapeutics → 投资行为链作为核心证据） |
| 证据锚点 | ED-01, ED-02 |

### RN-06 市场空间必须可算

| 项目 | 内容 |
|------|------|
| 规则 | 判断"市场很大"时必须给出算法和假设，让读者自行验证 |
| 级别 | 硬约束（涉及市场空间判断时） |
| 触发条件 | 文章涉及市场空间/商业前景判断时必触发 |
| 失败降级 | 至少给出量级对标（"对标 XX 领域的 YY 亿美元"） |
| 来源样本 | D1（60 万人 × 70% × 3.4 万美元 × 渗透率 = 30 亿美元显式推演） |
| 证据锚点 | ED-03 |

### RN-07 风险判断必须出示死亡名单

| 项目 | 内容 |
|------|------|
| 规则 | 说某赛道/靶点有风险时，必须列出已经失败的选手和具体失败原因 |
| 级别 | 硬约束（风险分析类） |
| 触发条件 | 分析有明确失败概率的行业/赛道/靶点时触发 |
| 失败降级 | 如果死亡名单不够长，降级为引用行业统计失败率 |
| 来源样本 | D2（罗氏/艾伯维/吉利德/BMS/新基在 CD47 靶点纷纷遇挫，逐一列出） |
| 证据锚点 | ED-05 |

## 三、已有规则的修订

### G01 判断必须由事实购买 — 扩展

原定义不变，但"事实"的具体形态需要按稿型区分：

| 稿型 | 事实的典型形态 |
|------|----------------|
| 叙事类（人物/戴老板/张佳伟） | 场景细节、物件描写、对话引述 |
| 分析类（阿基米德/EGFR） | 临床数据、财务指标、市场推演 |
| 混合类（小米 B 面） | 两者混合 |

### G03 承重比喻必须前置 — 降级为条件规则

原定义在叙事类稿型中仍是硬约束。
但 D 组 4 篇阿基米德稿证明：分析类稿型可以完全不使用比喻，靠数据和逻辑链推进。

**修订**：G03 的触发条件增加限定——"当文章使用比喻作为解释手段时触发；纯数据驱动型文章中不触发"。

### G05 失败必须出场 — 扩展

原定义"失败不是污点，而是逼出系统性答案的节点"仍然成立。
但 D2 证明存在变体："失败不是被收编的节点，而是被解剖的主角"。

**修订**：G05 增加子条目——"在风险分析类文章中，失败可以是文章主角（风险解剖），不必等到被收编。"
