## 候选样本 1：刘家琨，顽石的裂缝

### 基本信息

| 项 | 内容 |
|----|------|
| **标题** | 刘家琨，顽石的裂缝 |
| **路径** | `D:\汇度编辑部1\藏经阁\staging\editorial\source_articles\renwu\2103095721-2653246900_1.json` |
| **来源** | 人物杂志 |
| **字数估算** | content_html ≈ 12.4 万字符（正文约 1.2-1.5 万字） |

### 稿型判断

**人物 + 建筑/文化评论** 混合稿。

人物主线：建筑师刘家琨，2025 年获普利兹克奖，以"素人晋级"姿态成为一个例外。  
分析副线：中国当代建筑的现状诊断，"好建筑"的评价标准，在商业化趋势下坚持本质的困难。

两条线不是"人物引出分析"，也不是"分析套人物"——而是互相支撑：刘家琨的建筑实践本身就是对"什么是好建筑"这个问题的活答案，分析离不开人物，人物离不开分析。

### 混合点

1. **主问题是评价性的，不是叙事性的**：不是"刘家琨如何成为今天的他"而是"顽石意味着什么 / 裂缝指的是什么"——这是问题稿式的主干，但被套在人物稿框架里
2. **材料回收机制双轨**：既要回答"这些材料如何回钉刘家琨的核心轴"，又要回答"这些材料如何回答建筑评价的问题"——一个材料同时服务两个主干
3. **作者视角明显，有评判立场**：比 B 组三篇人物稿的"观察者"视角更近于"论者"，接近 D 组分析文的评判口吻

### 预估主人格

P01（witness / 见证者）为主，但压制了人物的戏剧内弧，转向了对建筑「精神」的理解和传递。可能出现人格"漂移"至 P02（deconstructionist），因为文章要解构"纯商业建筑"的合理性。

### 预估主拓扑

T5（人物真相钻头弧）变体：反复回钻的不是一个"骨相"而是一种"建筑哲学立场"。可能激活 TN-01（阶梯推进弧）作为分析副线。

### 预估触发的条件性对象

- `人物核心轴`（GPT-5.4 的 B 组结论）：触发，但轴是"对物质性的执念"，接近 D3/D4 类的"价值立场"而非 B 组的"驱动力/伤口"
- `框架重分类（DA-03）`：可能触发（文章将建筑分类为"有物质感 vs 缺乏物质感"）
- `比喻承重（A7）`：触发（顽石、裂缝、"沁出精神"是本文核心承重比喻）
- `权威锚定（DA-06）`：触发（普利兹克奖、评委引述）

### 最可能挑战的边界

**人物稿 vs 问题稿的路由判断规则**。

B 组规则 R-B01 要求人物稿"先写出人物核心轴"，但本文开篇 P1 是"当诗歌被人抛诸脑后"——这是文化评论式开篇，不是人物轴。人物在 P2 才出现。

这会触发一个路由问题：框架应该把本文分类为"人物 + 文化评论混合"，还是"以人物为入口的文化评论"？这两种分类会激活截然不同的拓扑和规则链。

### 验证样本 / 挑战样本

**验证样本**（接近边界但仍可被现有框架处理）。

本文不太可能推翻现有框架，但会测量 B 组人物核心轴和 D 组文化评论分析框架的叠合处是否定义清晰。失败时暴露的不是骨架问题，而是路由规则不够精确。

## 候选样本 2：一位老将的谋局样本：Biotech如何掌握自己的命运？

### 基本信息

| 项 | 内容 |
|----|------|
| **标题** | 一位老将的谋局样本：Biotech如何掌握自己的命运？ |
| **路径** | `D:\汇度编辑部1\藏经阁\staging\editorial\source_articles\shenlanguan\3864647914-2247499492_1.json` |
| **来源** | 深蓝观 |
| **字数估算** | content_html ≈ 18.7 万字符（正文约 1.5-2 万字） |

### 稿型判断

**人物 + 行业战略分析** 深度混合稿。

人物线：曹卫，中国细胞治疗领域的连续创业者，两次把公司卖给阿斯利康。  
分析线：Biotech 如何在宏观压力下保持主动权，连续创业的系统逻辑是什么。

关键结构：**人物是分析的载体，分析是人物存在意义的证明**。这和 B 组"人物为主，行业信息辅助回钉"不同，也和 D 组"分析为主，人物只作权威锚点"不同——这是一种真正混合的新密度。

### 混合点

1. **标题就已经混置主语**："一位老将"是人物线，"Biotech 如何掌握命运"是问题线，两个主语共存
2. **P1-P4 的开篇**：先是行业事件（收购公告）→ 发现人物红线（曹卫）→ 行业研判跟进——三段走完才完整定义"主问题是谁的问题"
3. **核心轴不可截断**：曹卫的核心轴（"寻找可控的确定性 / 系统性反脆弱思维"）既是人物内核，也直接等于行业分析的中心论点——这在 B 组任何一篇里都没有出现
4. **调色盘判断难度最高**：深蓝观有自己独特风格（专业医疗行业+偏分析），既不是阿基米德的"冷静分析师"，也不是人物杂志的"见证者"，是一个待命名的新调色盘

### 预估主人格

**无法直接路由**。这是本样本最强的挑战价值。

- P04（analyst / 冷静分析师）：行业论证部分100%命中，但人物深度已超出分析师模式
- P01（witness / 见证者）：人物内核清晰，但开篇不从人物出发，且核心是行业论点
- 实际可能是第五人格：**P05（strategist / 谋局者）**——围绕一个人物的系统决策模式来解构一个行业的生存逻辑

### 预估主拓扑

**TN-01（阶梯推进弧）** 作为宏观骨架，但内嵌了 T5（人物真相钻头弧）变体：反复从曹卫的行为回收行业判断。

这是 IIA 期所有候选中唯一一篇可能同时激活两个完整拓扑链的样本。

### 预估触发的条件性对象

- `人物核心轴`：触发，且轴值（"系统性反脆弱"）比 B 组三篇更抽象，更接近"执念 / 方法论"
- `DA-06 权威锚定`：触发（阿斯利康收购事件 + 曹卫的历史记录作为权威背书）
- `DA-07 算账推演`：触发（两次收购的商业逻辑、估值推演）
- `DA-13 多路径枚举`：可能触发（曹卫在西比曼和亘喜的不同路径对比）
- `R-B05（人物与问题叠合时每轮回人）`：核心压力规则，行业分析段落必须持续回收到曹卫视角

### 最可能挑战的边界

**三个并发的框架边界**：

1. **人格路由**：P04 和 P01 在同一篇中均能部分命中，但都无法完整覆盖——框架应该崩溃，还是新增第五人格 P05？
2. **核心轴定义的边界**：B 组的人物核心轴要求是"驱动力/伤口/执念"，但如果轴是"一套系统方法论"，是否仍算人物稿？还是应该被路由为"以人物为案例的行业方法论文"？
3. **规则冲突**：R-B01（人物稿先立轴）和 RN-01（行业稿先出数据/事件）在本文开篇同时适用——哪条规则优先？

### 验证样本 / 挑战样本

**挑战样本。**

本文可能无法被现有框架精确覆盖，而不是"勉强适配"。它最可能暴露的问题是：  
**框架目前缺乏处理"人物 = 行业分析的推演主轴"这种高密度混合的路由规则。**

## 候选样本 3：脑机接口门诊的一个下午

### 基本信息

| 项 | 内容 |
|----|------|
| **标题** | 脑机接口门诊的一个下午 |
| **路径** | `D:\汇度编辑部1\藏经阁\staging\editorial\source_articles\bingdian\2390471121-2653345690_1.json` |
| **来源** | 冰点周刊 |
| **字数估算** | content_html ≈ 3.7 万字符（正文约 4000-5000 字） |

### 稿型判断

**场景切片 + 技术行业分析** 混合稿。**第三类稿型**（既不属于人物稿，也不属于行业分析稿）。

结构核心：以一个具体的"下午"作为场景入口，通过该场景中的多个人物片段（患者葛雷、医生常健博）和技术场景，渐进带出"脑机接口临床化进展"这个更大的行业议题。

作者不在场，没有单一视角主人公，没有单一"核心问题钉子"——这是报道文学（reportage）叙事模式，A/B/C/D 四个组的 13 篇样本里没有一篇属于这种模式。

### 混合点

1. **无单一人物核心轴**：葛雷（患者）和常健博（医生）是并置的，没有一个是全文回收的主人公
2. **无标准问题钉子开篇**：P2 开篇是"3月5日，周四"——时间+场景定位，不是问题，不是压力
3. **分析内嵌于场景**：技术数据（"患病5年的患者，超过半数能有20%-30%的好转"）直接穿插在诊室场景描写里，不是单独分析段落
4. **行业纵深通过场景积累**：门诊一个下午 → 全国20家医院 → 技术现状 → 政策框架 → 研究空白，是从"小现场"向外扩散，而非从大框架向内聚焦

### 预估主人格

**无法路由到已有四个人格**。

- 既不是 P01（witness）：证人把情感投入写进了人物，这篇几乎情感中性
- 既不是 P02（deconstructionist）：没有批判框架，没有"打破共识"的姿态
- 既不是 P03（storyteller）：没有叙事弧，没有单一时间线
- 既不是 P04（analyst）：不以判断推进，以场景推进

**可能的新人格 P06（field reporter / 现场记者）**：通过现场切片积累行业真相，无强观点，以堆叠形成判断。

### 预估主拓扑

**现有拓扑表全部不匹配**。

最接近的是 T4（同构叠加弧，IB期/GLM-5 识别），但同构叠加弧要求重复同一结构，而本文是多个不同场景片段的线性组合，不是重复叠加。

可能需要新拓扑：**TN-05（场景积累弧）**——以多个具体场景片段堆叠，每个片段都让"行业真相"更饱满，最后不需要一个结论句，真相已经在积累中浮现。

### 预估触发的条件性对象

- 几乎无法明确触发已有的条件性对象
- 最接近：`细节买权 / 证据购买类（DA-01系）`：技术数据穿插在场景里——但这里数据不是"购买判断"，而是"丰富场景"，功能不同
- `失败收编 / 风险解剖（DA-08）`的变体：文章诚实地写出脑机接口的局限（下肢风险、信号准确率不足）——但这不是"收编失败"，而是"在现场中自然呈现局限"
- 几乎触发不到任何"规则层"：因为规则层的触发逻辑是"选择性激活"，但场景切片稿根本不做选择——它是"凡在场的都呈现"

### 最可能挑战的边界

**最激进的挑战样本。核心挑战有两层**：

1. **框架是否能处理"无核心人物、无核心问题"的稿型**：现有动作层、规则层、拓扑层的设计假设都预设了"要么有一个主要人物，要么有一个主要问题"。本文推翻这个假设。
2. **调色盘是否适用于无作者身份感的报道体裁**：冰点周刊的文章有集体风格，但不像阿基米德/张佳伟/半佛仙人有强个人调色盘。如果这类报道稿大量进藏经阁，调色盘的定义可能需要扩展为"稿型调色盘"而非"作者调色盘"。

### 验证样本 / 挑战样本

**挑战样本（强挑战）。**

本文最可能暴露的结构问题：  
**藏经阁的 4 层框架是否以"作者有观点"为隐性前提？如果文章没有强观点、只有精确现场，框架还能工作吗？**

## 不推荐入选的原因简述

### 排除：张雪峰（2103095721-2653247468_1.json）

本地 content_html 为微信安全页面（环境异常重定向），无正文内容，不满足"本地有可核原文入口"硬标准。

### 排除：蔡磊（2390471121-2653345507_1.json）

蔡磊稿是人物驱动极强的稿件，核心轴清晰（"渐冻症患者为他人争一口气"），属于 IIA B 组已覆盖类型的高质量样本，不是混合型。引入这篇是给 IIB 加验证量，不是打边界。

### 排除：艾滋谣言（2103095721-2653246315_1.json）

"艾滋谣一个餐饮店死亡 20 天"结构清晰：孙平案例（人物）→ 社会谣言机制（分析），但两条线是顺序关系而非真正混合——先案例、再分析。这是 IIA 轻人物＋轻分析的变体，没有达到"混合型"应有的结构张力。
