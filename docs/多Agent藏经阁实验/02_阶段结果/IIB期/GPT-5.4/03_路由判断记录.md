# GPT-5.4 IIB期路由判断记录

状态：公共主样本路由完成  
日期：2026-04-03  
中心：GPT-5.4  
目标：验证“一个样本进来后，是否能给出不超过 `5` 种可执行写法路线”

## 证据摘要

- 已直接回读 `5` 篇公共主样本原文，并逐篇压到 `1-5` 条候选路线内，没有出现“只能事后解释、无法前置分流”的情况。
- `人格` 在本轮真正参与了路由判断，但它仍是上层组合映射，不替代主问题与主拓扑。
- 本轮真正需要补的是 `2` 条路由边界：`问题现场群像稿`、`双人物共享核心轴`。

## 一、总路由口径

### 1. 先问主问题，不先看题材

题材只告诉我们“写的是什么”，  
路由要先回答“这篇真正要解决什么判断”。

### 2. 人物是否是主轴，要看材料是不是都在回钉它

不是出现创始人、医生、患者、明星，就自动路由成人物稿。  
只有当大量材料都在解释“为什么是这个人、为什么是这种驱动力”时，人物核心轴才真正参与路由。

### 3. 多人物 / 多个案不自动意味着多核心轴

如果多个个体都在购买同一个问题，  
那它更接近 `问题现场群像稿` 或 `双人物共享核心轴`，而不是多篇人物稿拼接。

### 4. 每篇样本最多保留 `5` 条路线

超过 `5` 条通常说明路由口径太散，  
需要回到主问题重新压缩。

## 二、分样本路由

## 脑机接口门诊的一个下午

- 主人格：`P01 witness / 克制的见证者` 为主，叠加 `P04 analyst / 冷静分析师`
- 主拓扑：`问题现场推进 -> 个案并列回收 -> 技术边界解释 -> 政策收束`
- 候选路线：
  1. `问题现场群像稿`
  2. `单患者康复人物稿`
  3. `脑机接口技术边界解释稿`
  4. `医保/标准推进政策稿`
  5. `新医疗门诊观察稿`
- 最终主路线：`问题现场群像稿`
- 不选其余路线的原因：
  - 不选 `单患者康复人物稿`：乔伊最完整，但 `葛雷 / 罗娟娟 / 黄子逸` 共同承担了筛选边界，单人化会丢掉门诊判断。
  - 不选 `技术边界解释稿`：解释很重，但正文入口和承重仍是门诊现场与病人处境。
  - 不选 `政策稿`：政策只在尾段增压，没有控制正文主体的推进顺序。

  2. `公司能力拆解稿`
  3. `供应链协同判断稿`
  4. `创始人创业长弧稿`
  5. `消费电子竞争格局稿`
- 最终主路线：`公司史-品类史混合深稿`
- 不选其余路线的原因：
  - 不选 `创始人创业长弧稿`：刘靖康贯穿全文，但正文真正要买的是“影石如何穿过硬件迷雾并定义新品类”。
  - 不选 `纯供应链稿`：供应链是核心论据，不是全文唯一问题。
  - 不选 `纯竞争格局稿`：竞争与反超只在后段抬高，不足以单独吃下全篇。

- 主拓扑：`并购新闻起手 -> 创业排雷回钻 -> 平台与并购论证 -> 西比曼回证 -> 书写收束`
- 候选路线：
  1. `战略人物-问题混合稿`
  2. `Biotech 生存法则分析稿`
  3. `资本-监管-并购链条稿`
  4. `创业排雷手册稿`
  5. `曹卫人物稿`
- 最终主路线：`战略人物-问题混合稿`
- 不选其余路线的原因：
  - 不选 `纯人物稿`：这篇并不主要写曹卫的性格与人生，而是用他的决策逻辑解释 Biotech 如何拿回主动权。
  - 不选 `纯行业分析稿`：如果去人，很多材料会失去选择逻辑，尤其是“拿专业的钱”“为什么卖公司”这些关键判断。
  - 不选 `纯创业手册稿`：书写壳很重要，但它仍是服务于主问题的收束，不是主承重。

## 三、跨样本路由补丁

### 1. `问题现场群像稿` 的最小规则

- 允许多个患者 / 个案并列出现
- 但每个个案都必须购买同一个问题轴
- 一旦各个个案开始各讲各的，立刻降级为散点见闻或拆成单人物稿

### 2. 创始人是否是主轴，要看材料选择权

- 如果公司史、产品史、供应链史都在回答“这个人如何决策”，人物核心轴参与路由
- 如果创始人只负责串时间线，而材料实际在解释品类 / 公司能力，主路线仍应回到问题稿或公司稿

### 3. 双人物稿先测“共享核心轴”，不先测“双核心轴”

- `当小鹿遇到宋佳` 证明双人物可以共同购买一个命题
- 当前还不需要一上来就拆成两条独立人物路线

### 4. 演讲体不自动降级

- 先看它有没有完整的证据购买链
- 只要专业材料在持续回钉人物使命，就仍可路由到人物-问题混合稿

## 四、本轮路由判断

1. `5` 篇样本都能被压进 `1-5` 条路线，没有出现“只能事后解释、不能前置调用”的情况。
2. `人物 + 问题`、`公司史 + 品类史 + 产业分析`、`无单一主角的现场报道型问题稿` 三类高风险混合稿，本轮都已经能给出可执行路由。
3. 当前最该补的是边界说明，而不是路由数量。
