### 2. 题目要求对照

**任务要求**：
- 不得编造素材之外的事实 → 已遵守
- 自主判断文章类型 → 选择个案+机制混合稿
- 文章必须既能让读者感受到一个小店被谣言击中的现实，也能让读者明白这种伤害为何反复发生 → 双层处理
- 允许删减碎片，但删减后仍要保留足够的社会解释力 → 舍弃噪音，保留机制分析

### 3. 碎片化材料的自主组织

C类素材是碎片池形态，顺序无意义，不预设主次，需要自主判断和组织。

执行策略：
- 从一家店下刀（个案现场）
- 向外扩到谣言的固定链条
- 再扩到社会机制分析（病耻感、熟人网络、污名内化）
- 收回到谣言的换壳能力

结构路径：个案现场 -> 谣言传播 -> 固定链条 -> 追不到的源头 -> 为什么能反复得手 -> 污名内化 -> 谣言换壳能力 -> 熟人网络扩音 -> 结语

### 4. 个案与机制的混合处理

题目要求：写出一种谣言如何在熟人社会、平台传播和病耻感中完成闭环。

执行结果：
- 个案层面：一家店的客流暴跌、验血辟谣、孩子的异样对待
- 机制层面：谣言的固定链条、追不到的源头、病耻感叠加、污名内化、熟人网络扩音
- 闭环逻辑：谣言传播成本为零，辟谣成本很高；污名内化让伤害持续；熟人网络让传播加速

**判断**：个案+机制混合稿形态成立，双层处理完成。

### 5. 噪音处理

**噪音识别**：
- 具体小吃种类（哪种地方小吃）——舍弃
- 个别年份月份精确时间点——舍弃
- 截图、视频、道歉文本表述细节——舍弃
- 直播平台围观者情绪化发言——舍弃

**保留的主承重信息**：
- 谣言传播路径（微信群->熟人->菜市场->街巷）
- 固定链条（被造谣、客流暴跌、报警、验血、通报、辟谣）
- 社会机制（病耻感、性道德想象、食品接触恐惧、熟人网络）
- 污名内化逻辑

**判断**：在碎片化材料中自主识别主承重信息，噪音处理能力通过。

### 6. 文章类型判断

题目要求自主判断是个案稿、社会问题稿，还是"个案+机制"混合稿。

执行选择：个案+机制混合稿。

理由：
- 纯个案稿会失去社会解释力，太浅
- 纯社会问题稿会失去人的刺痛感，变硬
- 混合稿能既保留个案现场，又带出机制分析

**判断**：文章类型自主判断正确。

### 3. 双人物共享核心轴形态

IIB遗留边界之一：双人物共享核心轴。

D类是女性友谊关系稿，涉及多组女性朋友，本质是"双人物共享核心轴"形态的测试。

执行结果：
- 第一组深写：稳定vs漂泊的两条路，彼此作为灯塔和安全网
- 第二组辅助回声：高光vs暗面的误解和重新靠近
- 第三组辅助回声：育儿焦虑vs玩心生活的午休陪伴
- 全文回到同一个关系判断：差异变成照亮

**判断**：
- 不是单一人物稿，而是多组关系串联
- 每组关系都有双人互动的核心轴
- 全文回到同一个判断，不是平均切块

**与IIB边界对照**：双人物共享核心轴这条边界，本轮有所收束，结构形态成立。

### 4. 逼近原稿家族（不暴露作者身份）

题目要求：在不暴露作者身份的前提下，逼近原稿的结构家族与气质家族。

**判断**：在不暴露作者身份的前提下，逼近"温柔、具体、彼此映照"的结构家族。

### 2. 附加限制遵守情况

| 限制 | 要求 | 执行情况 |

| 人格 | 冷静分析师，人物核心轴必须强显性 | 语言风格冷静有硬度，人物轴强显性 |
| 主拓扑 | 单人物核心轴+多轮问题回钉 | 行业/产品/环保/空间都成为人物轴反证 |
| 情绪温度 | 中低，有硬度不冷冰冰 | 温度控制在中低，未喊口号 |

| 收尾方式 | 从人物当下控制收回行业愿景，不喊口号 | 从核心轴延伸收回，自然落回愿景 |
| 禁用项 | 不写成企业宣传文/只堆产品功能/人物问题拆成两篇/空泛热血替代细节 | 已遵守 |

### 3. 人物核心轴形态

题目核心要求：一个人对"不合理"的强烈反应，以及他对"确定性"的偏执，如何一路控制了他的判断、动作和选材。

执行结果：
- 人物核心轴明确：对不满意的强烈反应、追问、确定性偏执
- 所有行业判断、产品方案、环保标准、空间逻辑都围绕这条轴展开
- 行业、产品、环保、空间等材料都成为人物轴的反证，不平铺并列

**判断**：人物核心轴形态成立，没有写成普通企业人物稿或行业解释稿。

### 4. 人物-问题混合回钉

题目要求：行业、产品、供应链、环保等解释段落，必须反复回到这个人本身，不能各写各的。

执行结果：
- 睡眠场景传感逻辑 → 回到他对健康底线的偏执
- 空间功能密度 → 回到他对结构与功能一致性的判断
- 环保实验室 → 回到他不相信运气的控制逻辑
- 一号位与时代判断 → 回到他个人核心轴的自然延伸

**判断**：人物-问题混合回钉结构成立，材料反复吸回到人物轴上。

### 5. 与旧B稿家族对照

附加题测试：能否逼近旧B稿的结构家族（人物核心轴+人物-问题混合回钉），而不是写成普通企业人物稿。

执行结果：
- 没有把人物稿写成普通行业解释稿
- 人物核心轴没有写丢
- 人物-问题混合回钉没有写成松散拼接

**判断**：附加题结果指向人物核心轴和混合回钉结构成立，逼近旧B稿家族特征。
