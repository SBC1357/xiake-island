# IID 对照组汇总结论

## 对照组定位

本组为 IID 对照组，使用旧本地写稿链路（不依赖后端 API）完成 4 类需求卷写作。与试验组的对比维度为：

- 试验组：可能使用藏经阁实验口径、依赖 API 链路、有外部资产支持
- 对照组：使用旧本地写稿链路、直接基于素材池写作、无外部资产支持

---

## 执行结论

### 完成情况

| 卷类 | 字数 | 主线 | 达成状态 |
|------|------|------|---------|
| A类 | 约1800 | 照护压力的短暂释放与难以真正接住 | ✓ |
| B类 | 约1050 | 代际接力——从闯到玩 | ✓ |
| C类 | 约1600 | 谣言的得手机制 | ✓ |
| D类 | 约1450 | 差异作为映照 | ✓ |

### 链路特征

- **后端状态**：不可用（curl 检查失败）
- **写作方式**：直接基于素材池写作，不走 API 链路
- **主线选择**：每卷独立选择主线，不依赖外部资产
- **素材使用**：严格遵守"不得编造素材池之外的事实"限制

---

## 主线选择总结

| 卷类 | 主线类型 | 选择依据 |
|------|---------|---------|
| A类 | 问题追问型 | 照护压力→短托服务→追问选择权 |
| B类 | 代际叙事型 | 父母靠闯完成积累，子女靠玩重新定义 |
| C类 | 机制追问型 | 个案切入→带出谣言为何总能得手的机制 |
| D类 | 关系映照型 | 核心判断推开→深写一组关系→其余做回声 |

### 主线选择共性

- 均从素材池中提炼主判断，而非平均铺开
- 均遵守"不得编造素材池之外的事实"限制
- 均符合各自卷类的限制条件
- 均有明确的叙事推进而非空泛抒情

---

## 旧本地写稿链路特征

### 优势

1. **独立性**：不依赖后端 API，可独立完成
2. **灵活性**：主线选择完全自主，不受外部资产约束
3. **可控性**：素材使用可直接追溯，无外部资产干扰

### 限制

1. **无质量审查**：不经过 quality/review 端点，依赖人工检查
2. **无语义审查**：不经过 semantic review 端点，依赖人工判断
3. **无返修机制**：不根据 rerun_scope 返修，依赖一次性完成

### 与 API 链路对比

| 维度 | 本地写稿链路 | API 链路 |
|------|-------------|---------|
| 依赖 | 无外部依赖 | 依赖后端 API |
| 主线选择 | 完全自主 | 可能受资产约束 |
| 质量/语义审查 | 人工检查 | API 端点审查 |
| 返修机制 | 无 | 根据 rerun_scope 返修 |

## 对照组价值

作为对照组，本组提供了一个"无外部资产支持"的基准：

- 提供与试验组横向对比的基准数据
- 验证素材池写作的基本可行性
