## 使用约束

- 下面每条都只是“边界压缩建议”。
- 下面所有候选观察都不得直接入正式表。
- 像 `GS-07`、`CO-08` 这类命名，只能保留为候选条目 / 候选边界观察，不得直接升格。

## 1. 监管分歧稿

- 边界压缩建议：先压回“分析型主路线中的本土评价坐标稿”，进入此类样本时优先判断“分歧本身是不是研究对象”；如果是，就先看本土坐标、权威冲突、政策传导，再看常规 `CO-02 / CO-03`。
- 当前保守口径：`CO-02` 可以记为降级触发，不再默认把它当主构建动作；`CO-04` 允许记成多权威冲突场景下的锚定变化；`CO-06` 只记特殊用法，不单独升格。
- 候选边界观察：C2 的“医学监管分歧型”只保留为候选观察，不入正式表。

## 2. 评价尺子先行 vs 产品导向

- 边界压缩建议：先保留“评价尺子先行”作为旧主写法，但增加一道保守检查：尺子立完之后，是否还在持续量全员；如果后段明显转为单品放大，就只记“产品导向尾段”，不立新拓扑。
- 当前保守口径：样本 2 与样本 3 先按“分析型主路线里的产品导向漂移”处理，不直接新造“产品推广型拓扑”。
- 候选边界观察：C1 的“评价框架 + 产品导向”与 C2 的“多框架发散”都先留在候选边界观察层，不直接拆成两个正式子类。

## 3. 赛道重定义 + 唯一性

- 边界压缩建议：先把这类稿压成“赛道重定义服务唯一性收束”的旧分析母路线；进入样本时先看分类 / 分型是不是主推进坐标，再看唯一性是否压过解释桥。
- 当前保守口径：`解释桥`、`权威锚定`、`唯一性主张` 之间的主从关系先只做记录，不正式改写对象表。
- 候选边界观察：C2 对“主张锚定优先于解释桥”的说法，只能保留为候选观察，不写成新主规则。

## 4. 公司画像评价标准 / 平台 vs 资产

- 边界压缩建议：先继续沿用“评价标准先行，再看资产组合”的旧口径；如果平台技术明显成为主锚点，就在旧口径内加注“平台优先”，而不是直接把公司稿升级成新三维拓扑。
- 当前保守口径：公司画像稿仍先放在分析型母路线下处理；“平台”暂视作资产组织方式里的高优先级观察项，而不是新层。
- 候选边界观察：C2 的“标准-平台-资产”只记为候选边界观察，不正式入表。

## 5. 人物-问题混合

- 边界压缩建议：先压回“人物主路线 + 问题解释从属”的旧保守写法；进入此类稿时先判标题设问和收束动作到底落在人物还是落在机制。
- 当前保守口径：只要标题与收尾仍在追问“这个人为何做成 / 为何成立”，就先保人物主路线；中段技术解释再重，也先记为从属插层。
- 候选边界观察：比例阈值、`GS-01 / CO-02` 主从识别规则都先保留为候选观察，不正式立数值阈值，不新增专访子类。

## 6. 机制稿里的比喻承重

- 边界压缩建议：先把“比喻是否承重”改成一道功能检查，而不是看到标题比喻就直接触发 `CO-01`；先问比喻是不是全文主结构，还是只在入口降门槛。
- 当前保守口径：样本 6 暂时不能据此把 `CO-01` 固化成主承重条目；更稳的记法是“机制解释主线明确，比喻角色待压实”。
- 候选边界观察：`CO-01 / CO-02` 互嵌、比喻是“外壳还是承重”都只做候选边界观察，不入正式对象表。

## 7. 阳性结果的场景边界

- 边界压缩建议：先压回“阳性结果不等于全场景放大”的旧保守写法；进入此类稿时默认加一道“获益来源还原 / 适用人群界定”检查。
- 当前保守口径：先不立新稿型，只把它记作“结果边界稿”或“条件性分流稿”的旧母路线内变体。
- 候选边界观察：C2 提出的 `GS-07` 只能写成候选条目 / 候选边界观察，不能直接入正式表。

## 8. 患者沟通型安全性比较

- 边界压缩建议：先压回“患者问答型安全性比较”的旧保守写法；进入此类稿时优先判断目标读者是否已从专业端切到患者端，一旦切换，就把“降维解释 / 选择建议 / 焦虑管理”记成特殊目的，而不是新层级。
- 当前保守口径：此类稿仍先当边界样本处理，不直接宣布“人格上收失效”，也不直接新增患者沟通主路线。
- 候选边界观察：C2 的 `CO-08` 只能保留为候选条目；C1 关于“人格上收可能遇到边界”的说法，也只保留为候选边界观察，不提前改写主框架。
