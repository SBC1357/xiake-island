### 样本 1：CGZ-YJWYJ-20240729-0022
**标题**：明朝的剑能不能斩清朝的官？论EMA和FDA对仑卡奈单抗的审评分歧  
**桶位**：审批 / 监管 / 注册博弈

#### 1. 主人格是什么

评价-解释型。全文核心动作是"建立一套本土评价坐标来解释为什么同药不同批"，不是纯叙事，也不是纯判断。通过"不可能三角"等框架来翻译监管机构决策逻辑，属于**分析型评价人格**。

#### 2. 主拓扑是什么

**阶梯推进型**（分层递进）。第一层陈述分歧事实；第二层用"二者都是本土机构"作为结构解释起点，逐层深入到"医疗体系差异 → 审批决策差异"；第三层重建关于仑卡奈单抗的本土评价框架。不是简单的双声部对比，而是逐层剥开监管逻辑的结构性拓扑。

#### 3. 触发了哪些条件性对象 / 稿型子型

| 条目ID | 是否触发 | C2判断 | 说明 |
|---|---|---|---|
| CO-04（权威锚定） | ✅ 触发 | 真实 | EMA/FDA 本身即权威行为主体；引用审批历史数据支撑"获批几率高"是典型权威锚定用法 |
| CO-06（政策推演） | ✅ 触发 | 真实 | "欧盟医疗体系 → 更关注安全性 → EMA拒绝批"是典型的规则传导链；以监管政策设计推导产业后果，属于政策推演结构 |
| CO-02（解释桥） | ⚠️ 降级触发 | 条件成立但特殊 | "不可能三角"起了部分解释桥功能，但本篇的独特性在于：监管分歧**本身即主研究对象**，而不是"用来说明某个判断的工具" |

#### 4. 哪些旧规则失效或需要降级

**旧规则失效（医学域）：**

- CO-02（解释桥）常规模式在监管分歧稿里**降级为描写性功能**，而非构建性功能。当"解释对象"本身就是文章主题时，解释桥不能用来搭结论，而只能用来搭理解过程。

**需要降级的旧判断：**

- "监管机构背书 = CO-04 权威锚定"这条规则，在多权威冲突场景下需要增加一层判断：**当多权威给出不一致结论时，权威锚定触发"本土坐标锚定"亚型，而非单一权威背书亚型**。

#### 5. 这篇样本最可能暴露哪条边界

**边界B-医学监管分歧型**
当一篇文章的"研究对象本身就是两个权威之间的分歧"时：
- 解释桥（CO-02）功能从"构建结论的工具"降格为"理解过程的辅助"
- 权威锚定（CO-04）功能从"单一背书"分裂为"多元本土坐标锚定"
- 常规"数据锤击"（CO-03）不适用，因为没有新的数据产出

这是**监管分歧类医学文章的独特边界**，在现有条件性对象表里没有对应条目，是本次复核发现的新边界候选。

#### 6. 反向调用路线（1-5条）

- R1 历史先例锚定 → 结构归因 → 本土坐标重建（历史铺垫型）
- R2 机构对比框架 → 医疗体系三角 → 决策逻辑拆解（对比分析型）
- R3 事件本身 → 分歧根源 → 评价框架重构（核心重构型）
- R4 分歧事实 → 结构性解释 → 仑卡奈单抗具体分析（具体应用型）

#### 7. 主路线

**R1（历史先例锚定 → 结构归因 → 本土坐标重建）**

### 样本 2：CGZ-YJWYJ-20250114-0049
**标题**：终于有研究把9款已上市的EGFR-TKI拉出来比一比了……  
**桶位**：临床结果 / 机制 / 指南解读

#### 1. 主人格是什么

评价-工具型。全文动作核心是"借助 ASCO-VF 和 ESMO-MCBS 两套工具，把9款TKI的临床获益客观量化"，主语是"评价工具"，作者退到工具后面。是典型的**数据评价人格**，以工具为中介做判断。

#### 2. 主拓扑是什么

**双框架并列型**。全文实质是"同一批数据在两套评价框架下的结果对照"。结构为：先介绍两个框架 → 分别展示9款TKI在两套框架下的评分 → 分析差异原因 → 对伏美替尼做优势总结。拓扑核心是"两套尺子的对照"，不是简单的并列，而是通过对照揭示两套框架各自的局限和各自的强项。

#### 3. 触发了哪些条件性对象 / 稿型子型

| 条目ID | 是否触发 | C2判断 | 说明 |
|---|---|---|---|
| CO-03（数据锤击） | ✅ 触发 | 真实，且升级使用 | 数据密度高（9款药 × 2套框架 × 多个维度），且最终服务于"哪个药获益最大"的大判断，是典型的数据锤击高密度触发 |
| CO-04（权威锚定） | ✅ 触发 | 真实 | ASCO-VF 和 ESMO-MCBS 本身就是权威机构背书的评价体系；引用 CDE 指导原则增加了权威维度 |
| GS-03（分层递进论证） | ✅ 触发 | 真实 | 两套框架的结果对照，形成自然的层间递进：框架介绍 → 数据填充 → 差异分析 → 综合判断 |

#### 4. 哪些旧规则失效或需要降级

**需要降级的旧规则：**

- "CO-03（数据锤击）触发后，结论是收敛的"——本篇数据锤击的结论是**发散的**：两套框架得出了不一致的排序，伏美替尼在 ASCO-NHB 里第一，但在 ESMO-MCBS 里不是最高。旧规则默认"数据越锤越清晰"，但医学域存在"多框架数据锤击后结论仍然分散"的真实场景，需要加注"框架收敛性判断"前置条件。

#### 5. 这篇样本最可能暴露哪条边界

**边界C-多框架评价发散型**
当文章依赖多套评价框架（ASCO-VF / ESMO-MCBS）而非单一框架时：
- 数据锤击（CO-03）触发后，结论不一定收敛
- 多框架并列拓扑的"意义"在于揭示框架局限性，而非给出唯一答案
- 这类稿型的写作目的是"让读者理解多框架并存的合理性"，而不是"选出最优解"

这是**多框架评价类医学稿**的独特边界，CO-03 的"收敛性假设"在此类样本中失效。

#### 6. 反向调用路线（1-5条）

- R1 评价工具介绍 → 数据填充 → 双框架对照 → 结论收敛（标准评价型）
- R2 单一框架深挖 → 数据全量展示 → 单一框架优势排序（深度评价型）
- R3 临床争议引入 → 多框架对照 → 揭示框架局限性 → 无唯一解（发散揭示型）
- R4 竞品全景 → 数据并排 → 伏美替尼最优锚定（产品背书型）

#### 7. 主路线

**R3（临床争议引入 → 多框架对照 → 揭示框架局限性 → 无唯一解）**

> 理由：本文最大价值不是"给伏美替尼背书"，而是揭示了两套框架的内在差异让读者理解"为什么临床上会有分歧"。R3 保留了原文的核心张力。

### 样本 3：CGZ-YJWYJ-20241210-0041
**标题**：EGFR突变被重新定义后，伏美替尼成为了TKI中的那个「唯一」  
**桶位**：疾病 / 赛道 / 治疗格局综述

#### 1. 主人格是什么

主张-论证型。全文的核心动作是"论证伏美替尼是覆盖所有EGFR突变亚型的唯一TKI"，主张明确，论证密集。是典型的**强主张人格**，特点是结论先行、数据跟进、论证环绕。

#### 2. 主拓扑是什么

**解释-主张嵌套型**。外层是"赛道重定义 → 伏美替尼唯一性"的主张链；内层是"分子结构（三氟乙氧基吡啶）→ 临床优势"的解释链。两条链交替推进，分子结构解释不是独立存在的，它被嵌套在"为什么能成为唯一"的主张论证框架里。

#### 3. 触发了哪些条件性对象 / 稿型子型

| 条目ID | 是否触发 | C2判断 | 说明 |

| CO-04（权威锚定） | ✅ 触发 | 真实 | FURLONG 研究、NMPA 批准、临床试验数据都是权威锚定；但锚定服务于"伏美替尼是唯一"这个主张，不是无立场的背景铺陈 |

**需要降级的旧规则：**

- CO-02（解释桥）常规优先级规则在"主张-论证型"医学稿里需要降级：当"解释桥"服务于"主张论证"时，解释桥是**从属动作**，不是主承重动作。主承重动作是"主张锚定"，解释桥只是填充材料。
- CO-04（权威锚定）在强主张稿里不能单独成立，必须"锚定数据化"才有论证效力。

#### 5. 这篇样本最可能暴露哪条边界

**边界D-主张论证稿的解释桥从属问题**
当一篇稿是"强主张 + 密集论证"结构时：
- 解释桥（CO-02）和权威锚定（CO-04）都在触发，但两者形成**主从竞争关系**
- 旧规则里"解释桥服务判断"的默认假设是平等的，但在主张论证稿里，解释桥被主张结构压缩为从属功能
- 这类稿型的边界在于：**主张锚定应该优先于解释桥被识别**，而不是并列处理

#### 6. 反向调用路线（1-5条）

- R1 赛道重定义 → 突变分类 → 伏美替尼唯一性 → 分子结构 → 临床数据（赛道-主张型）
- R2 分子结构深挖 → 临床前优势 → 临床验证 → 唯一性确认（分子-临床型）
- R3 赛道全景 → 竞品对照 → 伏美替尼差异化 → 唯一性锚定（竞品对照型）
- R4 临床未满足需求 → 突变分类 → 伏美替尼填补空白 → 唯一性（需求-填补型）

#### 7. 主路线

**R1（赛道重定义 → 突变分类 → 伏美替尼唯一性 → 分子结构 → 临床数据）**

### 样本 4：CGZ-YJWYJ-20250612-0058
**标题**：迪哲医药：构建多元血液瘤管线，定义新时代Biotech创新标杆  
**桶位**：公司 / 管线 / BD / 产业策略

#### 1. 主人格是什么

评价-建构型。全文核心动作是"建构一套 Biotech 创新评价体系，然后把迪哲放进去"，不是单纯的管线介绍，也不是无立场的公司画像。是**建构型评价人格**，特点是先立标准，再填充资产，标准本身是文章的重要组成部分。

#### 2. 主拓扑是什么

**标准-资产双线型**。明线是"血液瘤市场特点 → 迪哲资产组合 → 创新标杆定位"；暗线是"什么是真正的源头创新 → 为什么迪哲符合这个标准 → 创新标准本身就值得建构"。读者最后记住的不只是迪哲的产品，还有"什么样的创新才是真正的创新"这套框架。

#### 3. 触发了哪些条件性对象 / 稿型子型

| 条目ID | 是否触发 | C2判断 | 说明 |
|---|---|---|---|
| CO-03（数据锤击） | ✅ 触发 | 真实，但需加注 | 戈利昔替尼24.3个月OS、JAK1选择性200-400倍等具体数据是典型数据锤击；但数据服务于"源头创新"这个建构框架，不是无立场的罗列 |
| CO-04（权威锚定） | ✅ 触发 | 真实 | ASCO、NMPA、CDE、临床试验都是权威锚定；特别地，"JAK/STAT通路在自免领域是经典信号通路"也是权威锚定 |
| GS-01（人物核心轴） | ❌ 未触发 | 正确 | 人格上收后，GS-01 不在公司管线稿中强制触发；无人物驱动内核 |

#### 4. 哪些旧规则失效或需要降级

**需要降级的旧规则：**

- "公司管线稿 = 标准-资产购买"这条规则，当公司创新叙事聚焦于**平台技术**（JAK1高选择性平台）而非单一资产时，"标准-资产"拓扑需要升级为"标准-平台-资产"三维拓扑。旧规则把平台当成资产处理，忽略了平台的可扩展性叙事价值。

**旧规则失效（边界暴露）：**

- "公司管线稿的主流拓扑 = 资产清单型"——本篇颠覆了这个假设。公司管线稿在医学域的真实写法里，有相当比例是"创新标准建构型"，其拓扑重心不在资产清单，在建构框架本身。

#### 5. 这篇样本最可能暴露哪条边界

**边界E-平台技术型公司管线稿的拓扑升级问题**
当公司创新叙事以"平台技术"为核心时：
- 传统的"评价标准 → 资产购买"二维拓扑需要升级为"评价标准 → 平台优势 → 资产组合"三维拓扑
- 平台叙事比单个资产叙事有更强的可扩展性和更长线的创新说服力
- 这是公司管线稿里**平台型 Biotech 特有的写法**，不属于资产清单型

#### 6. 反向调用路线（1-5条）

- R1 疾病领域全景 → 临床未满足 → 迪哲填补 → 创新标准 → 资产组合（未满足-填补型）
- R2 源头创新定义 → 迪哲符合性 → 平台优势 → 资产组合（标准-验证型）
- R3 血液瘤市场 → 迪哲差异化 → 创新标杆定位（差异化-定位型）
- R4 技术平台介绍 → 平台扩展性 → 资产管线 → 创新标杆（平台-扩展型）

#### 7. 主路线

**R4（技术平台介绍 → 平台扩展性 → 资产管线 → 创新标杆）**

## C2 领域复核总结

### 最重要的发现

1. **CO-02（解释桥）是本批最不稳定触发项**：8篇中有4篇触发或降级触发，但触发后的功能角色各不相同（理解辅助、从属填充、入口包装、患者安心），说明 CO-02 是医学域最需要细化"功能亚型"的条件性对象。

2. **CO-03（数据锤击）的"收敛性默认假设"在多场景失效**：样本2（多框架发散）、样本7（边界还原）、样本8（选择工具）都展示了数据锤击在医学域的不同功能角色，IIB期条件性对象表的 CO-03 条目需要增加"收敛性前置判断"和"功能角色亚型"两个维度。

3. **GS-01（人物核心轴）在人物-问题混合稿里的主从关系**是本批最有价值的边界发现：样本5清楚展示了 GS-01 为主 CO-02 为从的结构，这对路由系统的优先级判断有直接影响。

4. **两个新边界候选值得正式入表**：边界H（GS-07"亚组边界还原型"）和边界I（CO-08"患者安心桥"）在医学域有明确的触发场景和稳定的触发样本，值得在条件性对象表中正式建立条目。

### 吃不动的地方

以下问题本轮 C2 盲复核无法独立裁决，需 C0/C1 对照后共同决定：

1. **边界B的精确边界**：样本1的"监管分歧即研究对象"和样本2的"多框架发散"，是否应归为同一类边界，还是两个独立边界？
2. **GS-01 与 CO-02 主从识别的可操作化规则**：如何让路由系统在不看全文的前提下，仅凭标题和开头段落就能识别 GS-01 为主？
3. **边界H的命名和触发条件**：GS-07"亚组边界还原型"是否准确？触发条件是否需要进一步细化？
