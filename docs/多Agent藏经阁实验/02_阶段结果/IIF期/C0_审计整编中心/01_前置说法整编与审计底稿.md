# C0_审计整编中心 / 01_前置说法整编与审计底稿

状态：Active  
日期：2026-04-05  
角色：C0_审计整编中心  
用途：从 `IIE` 之前所有正式载体中整编出全部已发现的"说法"，完成去重标注、近重复标注和来源标引，供 `C1/C2` 后续做结构裁判与功能裁判

## 编制原则

1. 只从正式载体里抽说法，不发明新概念
2. 允许去重标注、近重复标注、来源标引
3. 不准改名、合并成新词、发明新概念
4. 每条说法只记录事实，不做最终判断
5. 所有备注统一写"待 C1/C2 裁判"
6. 不准恢复 `6槽位`
7. 不准替 `C1/C2` 做结构或功能裁判

## 当前使用边界

1. 本底稿中的**主审计清单**，只指第一至第七部分：
   - 动作
   - 构件
   - 规则
   - 条件性对象
   - 稿型子型
   - 架构 / 框架
   - 案例保留 / 观察项
2. 第八至第十部分只作为 `IIE` 结果区触发出来的**参考附录**：
   - 边界类说法
   - 候选条目
   - 降级方向
3. 这些附录项保留的目的，是帮助 `C1/C2` 理解为什么会进入 `IIF`，以及哪些问题在 `IIE` 中被重新暴露；它们不与前置说法并列为本轮主裁判清单。
4. 本底稿中的“动作类 / 构件类 / 规则类 / 条件性对象类 / 稿型子型类 / 架构类 / 边界类 / 候选条目 / 降级方向”等标题，只是 `C0` 的整编视图，不代表新的正式分类体系，也不构成对 `C1/C2` 的裁判预设。
