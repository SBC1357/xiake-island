# C2_功能裁判中心 / 01_最小单元功能裁判

状态：Active  
日期：2026-04-05  
角色：C2_功能裁判中心  
用途：对 C0 整编的 `42` 条主审计清单逐条做功能裁判

## 裁判原则

1. 只做功能归属判断，不做结构可拆性裁判（C1 职责）
2. 每条说法只给 5 档结论之一：写作目的候选 / 条件性对象候选 / 稿型子型候选 / 旧母路线容器 / 暂不能判
3. 判断标准：
   - `写作目的候选`：更像"为什么要这样写"——目的、动机、约束来源
   - `条件性对象候选`：更像"调用什么去写"——写作中被召唤的工具、手段、材料单元
   - `稿型子型候选`：更像"稳定组合后会长成什么稿型"——一类文章的辨识外形
   - `旧母路线容器`：只是旧路线级的组织框架，不是单一可调用对象也不是目的
   - `暂不能判`：证据不足
4. 不准新增平行概念、不准改名、不准补新条目
5. 参考附录 17 条只在需要时引据说明，不与主审计清单并列裁判

## 第一部分：动作类说法（ACT-01 ~ ACT-08）

### ACT-01 压力起手

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么要让读者先感到旧状态不可持续"。这是一种开篇目的：先造紧张感才能让后文的解法有购买动机。它不是一个可被独立调用的材料单元，也不是一种稿型——它说的是"为什么这样开"。 |

### ACT-02 关键问题钉子

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么全文必须围着一个高价值问题打"。这是全文驱动意图层面的声明：没有它，全文就没有聚焦点。它不是具体手段，也不是稿型外形——它说的是"为什么这篇文章要存在"。 |
| 补充说明 | 与 GS-01"人物核心轴"有功能交叉。区别：ACT-02 是通用版的"为什么要钉问题"，GS-01 是人物稿语境下"钉问题后长成什么"。 |

### ACT-03 历史回望链

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么今天的问题需要溯因"。它不是一种材料（材料是具体的历史事实），而是一种目的：证明"这个问题是怎么长出来的"，让当下的判断站得住。 |

### ACT-04 解释桥

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 回答的是"调用什么手段把抽象机制翻译成可感画面"。它描述的是一种写作时被召唤的中介手段——不是在说"为什么要翻译"，而是在说"用什么翻译"。同一说法在 S2 以 CO-02 身份出现，进一步佐证其功能定位在"可调用对象"层。 |

### ACT-05 比喻承重

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 回答的是"调用什么手段让比喻承担解释功能而非装饰"。这是一种写作时被召唤的技法对象。同一说法在 S2 以 CO-01 身份出现，进一步佐证其功能定位在"可调用对象"层。 |
| 补充说明 | 属于比喻承重簇（ACT-05 / CO-01 / RUL-03 / BDR-08 / DEG-02）。该簇内 RUL-03 说的是"为什么必须前置"（目的面），ACT-05 / CO-01 说的是"调用什么去做"（对象面）——功能归属不同。 |

### ACT-06 细节买权

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么大判断必须先付出细节成本"。这是写作的目的约束：因为判断的可信度需要用细节来购买。它不是在描述一种可调用的材料单元（材料是具体的细节），而是在说明"为什么要这样安排证据和判断的前后关系"。 |
| 补充说明 | 与 RUL-01"判断必须由事实购买"同处细节买权簇。区别：ACT-06 侧重动机面（为什么要买），RUL-01 侧重规则面（为什么不买不行）。二者功能归属一致，都在目的层。 |

### ACT-07 失败收编

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么失败不是污点，而是逼出系统性答案的节点"。这是写作目的声明：在叙事中给失败分配什么角色。它不是在描述一种可调用的技法——可调用的是 CO-05"风险解剖"。 |
| 补充说明 | 与 CO-05"风险解剖"有功能交叉。区别：ACT-07 说"为什么失败必须出场"（目的），CO-05 说"调用什么去解剖风险"（对象）。 |

### ACT-08 边界后拔高

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么先承认边界、再做价值收束"。这是收尾的目的逻辑：先承认局限才能让最终的价值判断不悬空。它描述的不是一种可调用手段，而是一种结尾策略的动机。 |

## 第二部分：构件类说法（CMP-01 ~ CMP-06）

### CMP-01 一条硬事实

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。写作中被召唤的材料单元——"调用什么去充当基本论证材料"。它不说"为什么"，也不说"长成什么稿型"，只说"有这么一种可被调用的最小材料"。 |

### CMP-02 一个判断

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。判断是写作中产出的基本单元——在叙事链中被安放、被购买、被收束的对象。 |

### CMP-03 一组关系

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。关系是叙事中被展示或被拆解的材料单元——"调用什么去呈现人与事、因与果之间的关联"。 |

### CMP-04 一个场景

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。场景是叙事、解释中被召唤的具体时空容器——"调用什么去承载可感画面"。 |

### CMP-05 一段引述

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。引述是写作中被召唤的权威性 / 现场感材料——"调用什么去借力"。 |

### CMP-06 一条边界声明

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 最小调用构件。边界声明是写作中被召唤的诚实信号——"调用什么去标记'到此为止'"。 |

## 第三部分：规则类说法（RUL-01 ~ RUL-06）

### RUL-01 判断必须由事实购买

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么判断前面必须有事实"——写作约束来自目的层：没有事实买单的判断站不住。这是一条"为什么"层面的硬约束，不是在描述"调用什么"。 |

### RUL-02 判断越大，前面需要的细节成本越高

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | RUL-01 的递进版。仍然回答"为什么"——判断的重量决定了前面需要多大的事实投入。这是目的层的比例约束。 |

### RUL-03 承重比喻必须前置于被解释对象

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么承重比喻必须先于被解释对象出现"——如果后置，比喻就退化成装饰而非解释手段。这是针对 CO-01"比喻承重"这个对象的使用目的约束，不是对象本身。 |
| 补充说明 | 属于比喻承重簇。功能分工：CO-01 / ACT-05 是对象（调用什么），RUL-03 是目的约束（为什么必须这样用）。 |

### RUL-04 内容拓扑决定级联还是平行

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么文章结构不应该按'作者偏好'来选，而应该跟内容拓扑走"。这是目的层的结构选择原则，不是一种可调用对象，也不是一种稿型。 |
| 补充说明 | 与 ARCH-04 是同一判断的不同表述面。RUL-04 偏规则化表达、ARCH-04 偏判断性表述，但功能归属一致。 |

### RUL-05 失败必须出场，但处理力度由说服路径决定

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么失败不能缺席"以及"为什么处理力度要跟说服路径挂钩"。这是目的层面的失败处置原则，不是一种可调用手段。 |
| 补充说明 | 属于失败收编簇。功能分工：ACT-07 / RUL-05 说"为什么"（目的），CO-05 说"调用什么"（对象）。 |

### RUL-06 收尾不能高于正文证据支撑的最大判断

| 字段 | 内容 |
|------|------|
| 功能裁判 | **写作目的候选** |
| 判断依据 | 回答的是"为什么收尾不能拔得比正文证据高"——因为拔高了就失信。这是收尾的目的约束，不是在描述一种可调用手段或稿型。 |

## 第五部分：稿型子型类说法（GS-01 ~ GS-06）

### GS-01 人物核心轴 / 人物真相钻头

| 字段 | 内容 |
|------|------|
| 功能裁判 | **稿型子型候选** |
| 判断依据 | 回答的是"人物稿在稳定调用几组对象后，长出来的外形是什么"——全文围着一个人物核心轴打，就是人物稿 / 人物-问题混合稿的辨识外形。它不是单一可调用手段（可调用的是 ACT-02 等），而是组合稳定后的产物型。 |
| 补充说明 | 与 ACT-02"关键问题钉子"有功能交叉。区别：ACT-02 是通用版"为什么要钉问题"（目的），GS-01 是人物稿语境下"钉了之后长成什么"（稿型）。 |

### GS-02 反讽式对比

| 字段 | 内容 |
|------|------|
| 功能裁判 | **稿型子型候选** |
| 判断依据 | 回答的是"半佛式强评论在稳定调用反讽 + 对比后，长出来的外形是什么"。它定义了一种可辨识的评论体子型外形，不是单一可调用手段。 |

### GS-03 分层递进论证

| 字段 | 内容 |
|------|------|
| 功能裁判 | **稿型子型候选** |
| 判断依据 | 回答的是"分析型强评论在稳定使用层层递进论证后，长出来的外形是什么"。当一篇文章从头到尾采用分层递进策略，它就形成了一种辨识度很高的稿型外形。与 CO-07 的区别：CO-07 说"调用什么拓扑"，GS-03 说"持续调用后长成什么"。 |
| 补充说明 | 与 CO-07"分析型拓扑簇"内的阶梯推进 / 双层递进有功能交叉。区别：CO-07 是被调用的拓扑工具，GS-03 是调用后稳定呈现的文章外形。 |

### GS-04 荒诞假设推演

| 字段 | 内容 |
|------|------|
| 功能裁判 | **稿型子型候选** |
| 判断依据 | 回答的是"半佛式强评论在稳定使用荒诞假设推演后，长出来的外形是什么"。它定义了一种以"先给荒诞假设、再推演出严肃结论"为辨识特征的评论体子型。 |

### GS-05 对话体叙事

| 字段 | 内容 |
|------|------|
| 功能裁判 | **稿型子型候选** |
| 判断依据 | 回答的是"张佳伟式对话体在稳定使用对话推进后，长出来的外形是什么"。它定义了一种以多方对话推进、而非传统段落论证为辨识特征的叙事子型。 |

### GS-06 意象回归收尾

| 字段 | 内容 |
|------|------|
| 功能裁判 | **条件性对象候选** |
| 判断依据 | 它不定义整篇文章的外形，只管"调用什么去收尾"——在张佳伟式叙事 / 人物稿的末尾，回归一个贯穿意象完成闭环。这是一种收尾技法对象，不是一种稿型外形。稿型外形应描述"整篇文章长成什么"，而 GS-06 只覆盖收尾段。因此功能归属更接近"条件性对象候选"。 |
| 补充说明 | 这是 C2 功能裁判与 C0 原始分类出现偏差的唯一一条。C0 照 S2 将其列入稿型子型，C2 认为从功能问题"它更像什么"出发，它更像一个被有条件调用的收尾对象。此偏差记录在案，不构成对 C0 整编或 S2 原表的修改。 |

## 第六部分：架构 / 框架类说法（ARCH-01 ~ ARCH-06）

### ARCH-01 四层骨架（动作 + 构件 + 规则 + 拓扑）

| 字段 | 内容 |
|------|------|
| 功能裁判 | **旧母路线容器** |
| 判断依据 | 这是整个体系的组织框架声明——"底座应该长什么样"。它不是一种写作目的（不说"为什么"），不是一种可调用对象（你不会在写作时去"调用四层骨架"），也不是一种稿型外形。它是路线级的架构容器，容纳了动作、构件、规则、拓扑四个类别。 |

### ARCH-02 人格应正式定位为上层组合映射

| 字段 | 内容 |
|------|------|
| 功能裁判 | **旧母路线容器** |
