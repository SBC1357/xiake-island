## 一、写作动作（Actions）

写作动作 = 作者对读者认知施加的操作。是动词，不是名词。

### A1 框架替换（Framework Replacement）

- **定义**：先展示旧框架的不足，再引入新框架取代之。读者的认知坐标被系统性地重置。
- **最小例子**：
  - EGFR 稿：外显子分类 → MDACC 结构-功能分类。旧分类"亚型太多，无法与治疗药物建立对应关系"，新分类把跨外显子的 PACC 突变归为一类。
  - B面小米：公众认知里的"互联网营销公司" → 文章建构的"技术底层驱动的科技公司"。
- **适用范围**：一切需要让读者放弃原有认知框架并接受新框架的解释型文章。
- **不适用边界**：纯叙事型文章（没有需要替换的旧框架）、观点评论型文章（不引入系统性新框架）。

### A2 比喻承重（Metaphor Load-Bearing）

- **定义**：用延伸比喻充当文章核心解释支架，而非修辞点缀。比喻要承担解释功能——读者靠比喻才能理解机制。
- **最小例子**：
  - EGFR 稿："亚特兰蒂斯遗迹"比喻体系。αC 螺旋 = 遗迹大门，P 环 = 门口抓手，TKI = 遗迹守卫，铰链区 = 锁扣，疏水核心 = 地板。整个 EGFR 激酶机制靠这个比喻建立可视化。
  - B面小米："做出题的人 vs 解题的人"——苹果证明了题目有答案，小米要自己出题。这个比喻承载了"集成式创新 → 自主式创新 → 颠覆式创新"的三级递进。
- **适用范围**：机制复杂、专业门槛高、需要跨域翻译的文章。
- **不适用边界**：读者专业程度已足够高时，比喻可能降低信息密度。

### A3 时间堆叠（Temporal Stacking）

- **定义**：把跨越较长时段的多个关键时间节点按序排列，让读者看到趋势或问题的渐进演化。
- **最小例子**：
  - EGFR 稿：吉非替尼问世（2004 前后）→ 早期试验成功 → III 期失败 → 发现真正的预测标志物 → 200+ 种突变被发现 → 2021 年 MDACC 新分类。
  - B面小米：2014 封神 → 2016 危机 → MIX 发布 → 2017 AI 起步 → 2019 技术委员会 → 2021 造车 → 2023 MIX Fold3。
- **适用范围**：任何需要展示演化脉络的文章。
- **不适用边界**：时间线不是核心论证支撑时（如纯逻辑推演型文章）。

### A4 对比翻转（Contrast Reversal）

- **定义**：先建立读者预期（正向或反向均可），再用事实翻转预期。制造认知落差。
- **最小例子**：
  - EGFR 稿：早期试验"让临床上更有信心" → "随后几项III期临床试验的失败却再次将我们拉回了起点"。
  - B面小米：发布会"中规中矩，波澜不惊" → "今天还有一款手机要发布" → 全场欢呼。
- **适用范围**：叙事型与解释型文章均可用。
- **不适用边界**：无真实落差可构建的场景。

### A5 降维打开（Dimensional Opening）

- **定义**：把高度专业的概念转译到读者已有的认知维度上，让理解门槛骤降。
- **最小例子**：
  - EGFR 稿：206 块骨头编号 → "腰间盘突出"四个字就够。用日常医学常识比喻说明"外显子编号"对临床的无用。
  - B面小米：CNC 一体化工艺 → "将手机拆分为屏幕和外壳两个部分"，读者无需理解工艺细节即可理解意义。
- **适用范围**：跨专业写作、科普、面向非专业决策者的文章。
- **不适用边界**：读者和作者在同一专业维度时无需降维。

### A6 证据嵌入（Evidence Embedding）

- **定义**：将数据、引文、具体事实编织进叙事流中，而不是另起一段集中列表。
- **最小例子**：
  - EGFR 稿："240mg 伏美替尼治疗 PACC 突变的 ORR 达 68.2%，mPFS 达到了 16 个月，≥3 级 TRAEs 发生率仅为 20.7%"——镶嵌在论证段落内。
  - B面小米："公司估值 450 亿美元，手机全年出货超过 6000 万""小米的螺丝钉连苹果都觉得贵"——穿插在叙事里。
- **适用范围**：几乎所有需要说服力的非虚构写作。
- **不适用边界**：纯文学或不需要事实支撑的文体。

### A7 诚实收束（Honest Bracketing）

- **定义**：在文章接近结论时，明确说出当前方案不能解决什么、还有什么不确定。放在强证据之后。
- **最小例子**：
  - EGFR 稿："尽管 MDACC 分类解决了目前 EGFR 突变 NSCLC 重要的临床指导瓶颈，但其依然不能解决所有的临床问题。"
  - B面小米："小米也许会犯比前十年更多的错误，但这恰恰是一家高科技公司成熟的象征。"
- **适用范围**：任何严肃论证型文章。
- **不适用边界**：纯宣传稿（不需要也不适合暴露不确定性）。

### A8 升维收束（Ascending Close）

- **定义**：结尾从具体事件/数据上升到普遍性、哲理性表达，把单篇文章的意义拉到更高维度。
- **最小例子**：
  - EGFR 稿："我们或许永远无法战胜癌症，但我们总是在保持前进。"
  - B面小米："如果有这样一家公司会出现在中国，它为什么不能是小米？"以及"只要远方无限辽阔，总会有船只迎着激流出发。"
- **适用范围**：长文、深度稿、旗舰型内容。
- **不适用边界**：短新闻、快讯、纯数据报告。

## 二、结构构件（Components）

结构构件 = 文章中可辨认的功能区块。可组合、可替换、可跨域复用。

### C1 入口钩子（Entry Hook）

- **定义**：文章第一个段落或发起句，制造认知张力或好奇心，拉住读者。
- **最小例子**：
  - EGFR 稿："EGFR 突变 NSCLC 靶向治疗历经 20 余年后，基于外显子的 EGFR 突变分类方法对临床的指导，终于开始显得乏力……"——"终于"二字暗示累积已久的裂缝。
  - B面小米："对小米来说，2016 年的秋天漫长而寒冷。"——一句话定下低谷基调。

### C2 问题陈述块（Problem Statement Block）

- **定义**：显式说清楚"什么东西坏了 / 不够用了 / 需要被替换"。
- **最小例子**：
  - EGFR 稿："基于外显子的分类方法模糊了对 EGFR 突变结构本质的描述，导致我们无法理解其对功能的影响。"
  - B面小米："小米在 2016 年迎来了公司历史上第一次、也是迄今唯一一次负增长。"

### C3 机制解释块（Mechanism Block）

- **定义**：对某个系统"如何运作"进行详细拆解。是文章中信息密度最高的部分。
- **最小例子**：
  - EGFR 稿：P 环、αC 螺旋、疏水核心、铰链区的结构与功能分述，加上 Atlantis 比喻实现可视化。
  - B面小米：小米 MIX 的全面屏工程——屏占比从 70% 到 90%、屏幕比例实验、说服谷歌改安卓兼容文档、1500℃ 陶瓷烧制。

### C4 比喻支架（Metaphor Scaffold）

- **定义**：承载解释功能的延伸比喻体系。区别于一次性修辞比喻。
- **最小例子**：
  - EGFR 稿：亚特兰蒂斯遗迹体系（大门 / 守卫 / 锁扣 / 地板）。
  - B面小米："做出题的人"vs"解题的人"；苹果 → iPhone 不是一蹴而就，而是 DVD → iTunes → iPod → iPhone 的累积。

### C5 证据桩（Evidence Pillar）

- **定义**：用具体数据、引文、一手来源锚定论点的段落。
- **最小例子**：
  - EGFR 稿：FURTHER 研究数据（ORR 68.2%, mPFS 16月）、ACHILLES 研究数据（ORR 61.4%, mPFS 10.6月, TRAEs 43.8%）。
  - B面小米：450 亿估值、6000 万出货量、69 元充电宝、全球前十 5G 专利族。

### C6 转折铰链（Pivot Hinge）

- **定义**：连接两个逻辑板块的过渡句或段。不只是"接下来说……"，而是显示前后逻辑关系。
- **最小例子**：
  - EGFR 稿："基于这个思路，MDACC 的新分类体系就很好理解了。"——从机制过渡到分类。
  - B面小米："但和 MIX 的一鸣惊人不同，小米的澎湃 S1 走向了截然相反的结局。"——同一框架下不同结局的对比铰链。

### C7 诚实尾声（Honest Coda）

- **定义**：明确承认局限性的段落。位于强证据之后、升华之前。
- **最小例子**：见 A7。

### C8 升华收束（Elevation Close）

- **定义**：全文最后的哲理性 / 普遍性表达。
- **最小例子**：见 A8。

## 三、内容拓扑（Content Topology）

内容拓扑 = 文章各部分之间的依赖结构。不是作者风格标签。

### T1 级联（Cascade）

- **定义**：每节依赖上一节的结论，线性推进。
- **例子**：EGFR 稿。历史 → 旧分类问题 → EGFR 机制 → 新分类 → 临床证据。去掉中间任何一节，后面就读不通。

### T2 平行汇聚（Parallel Convergence）

- **定义**：多条独立线索各自展开，最终汇聚到同一结论。
- **例子**：B面小米。MIX 线 / 芯片线 / AI+汽车线 / 一团火线，各自独立叙述，最终共同指向"小米是一家科技公司"。

### T3 螺旋上升（Spiral Ascent）

- **定义**：同一主题在不同段落反复出现，每次出现时深度递增。
- **例子**：B面小米中的"势"——第一次出现是"风口论"，第二次是"产业发展规律"，第三次是"对制造业的敬畏"。同一个词的内涵在文章推进中不断升级。

### T4 嵌套拓扑（Nested Topology）

- **定义**：宏观拓扑和微观拓扑不同。文章整体是 A 型拓扑，每章内部是 B 型拓扑。
- **例子**：B面小米宏观是"平行汇聚"，但每个章节内部是"级联"（MIX 章节：立项 → 工程困难 → 突破 → 量产 → 意义）。

## 四、不该存的东西

| 不该存 | 原因 | 应放的位置 |
|--------|------|-----------|
| 具体比喻内容（亚特兰蒂斯、太祖长拳等） | 是动作的实例，不是动作本身 | 证据锚点 / 案例库 |
| 作者特色词汇（"风云突变"、"简单粗暴"等） | 属于作者风味，不属于通用结构 | L1 作者 Profile |
| 特定领域数据（ORR/mPFS/估值等） | 属于内容，不属于结构 | 稿件本身 |
| 整段话模板 | 换了内容域就失效 | 不存或存为案例参考 |
| "风格标签"（冷峻/热血/学术等） | 过于模糊，无法指导生成 | L1 风味描述 |

## 五、自查：这些对象是否只是"另一种文学分析术语"？

核心检验标准：**这些对象能否指导生成，而不只是解释分析？**

- A2"比喻承重"：如果给一个新领域新题目，能否指导 Agent "你需要在这里构建一个承重比喻"？→ 有可能，但需要规则配合（什么时候用，什么时候不用）。
- C3"机制解释块"：能否告诉 Agent "这里需要一个机制块，信息密度要高"？→ 可以，但"怎么写"还需要动作配合。
- T2"平行汇聚"：能否指导 Agent "全文用平行汇聚拓扑，四条线最终收束"？→ 可以作为结构指令。

初步判断：对象方向大体可行，但 **生成指导力** 需要在第一轮持出拟合中验证。
