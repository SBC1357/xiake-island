## 重组原则

实验要求"至少 4 组"跨域重组。本轮不写完整万字重组稿（违反仓库卫生红线），而是做 **重组结构方案 + 关键段落验证 + 复盘**：设计"用 X 的方法组织 Y 的内容"，写出 2-3 个关键段落验证可行性，然后复盘。

## 重组一：空之客"共识拆解"法 × EGFR 突变分类内容

### 设计

- **结构供体**：空之客（共识拆解 + 降维比喻 + 自嘲式反转）
- **内容供体**：EGFR 突变分类（MDACC 新分类体系的价值）
- **调用对象**：A1（框架替换，变体=共识拆解）、A2（比喻承重，空之客式历史比喻）、A5（降维打开）、A4（对比翻转）
- **调用规则**：R1（比喻必须承重）、R4（对比需要锚钉）、R5（每段必须推进）

### 关键段落验证

**入口钩子（C1，共识拆解型）：**

> 过去二十年，EGFR 突变 NSCLC 的治疗建立在一个看似坚固的分类地基上：按外显子分。19del 用吉非替尼好，L858R 用奥希替尼好，ex20ins 是难治的——诊疗指南这么写，临床路径这么走，KOL 讲课也这么说。可是当测序技术把 200 多种独立 EGFR 突变组合摊开在桌面上的时候，这种思路的问题，其实不需要分子生物学的训练就能发现：你把 20 号外显子上的一百种插入全归在同一个"ex20ins"的大口袋里，跟告诉你"这 206 块骨头里有一块出了问题"一样没有信息量。

**核心论证段（A1 共识拆解 + A5 降维）：**

> 问题出在哪？外显子是基因序列的分区方式，不是蛋白质功能的分区方式。用外显子号来预测药物响应，一个比较简单粗暴的比方是：你用邮政编码来预测一个小区的房价——朝阳区 100020 和朝阳区 100025 可能一个是三里屯一个是定福庄，但它们都是"朝阳区"。MDACC 的新分类把这个邮编换成了地段——你说的是 P 环、αC 螺旋还是疏水核心，这三个地方被突变了会怎样，直接对应到药物能不能进去。

### 复盘

| 维度 | 判断 |
|------|------|
| 顺的部分 | 共识拆解框架完全能容纳 EGFR 内容。"旧共识 = 按外显子分"天然适合被拆解。降维比喻（邮编→地段）可以替代原文的亚特兰蒂斯比喻。 |
| 假的部分 | 空之客的自嘲式反转在 EGFR 场景下不自然——EGFR 分类不存在"我当年也犯过这个错"的个人叙事。武侠隐喻在分子结构语境里也勉强。 |
| 假在哪里 | **动作层面**：自嘲式反转需要作者与内容有个人历史，这是空之客独有的 persona 适配条件，不是通用动作。**领域门槛**：EGFR 激酶结构的复杂度超过空之客通常处理的商业逻辑，比喻需要重新发明而非照搬金庸。 |

## 重组二：EGFR 稿"比喻承重 + 框架替换"法 × 医药 BD 交易分析内容

### 设计

- **结构供体**：EGFR 稿（系统承重比喻 + 老框架→新框架 + 级联拓扑）
- **内容供体**：和黄呋喹替尼 BD 交易（空之客"太祖长拳"文的内容）
- **调用对象**：A1（框架替换）、A2（系统承重比喻）、C3（机制块）、A6（证据嵌入）
- **调用规则**：R1（比喻必须承重）、R8（拓扑匹配内容依赖）

### 关键段落验证

**入口钩子 + 问题陈述（C1+C2，EGFR稿法）：**

> 中国创新药的价值评估，在过去五年里形成了一套看似精确的判断体系：靶点新不新？管线密不密？首创还是仿创？这套体系像外显子分类法一样好用了很长时间。直到和黄医药用一款十几年前起步的 me-better 小分子呋喹替尼，换来了 4 亿美元首付 + 7.3 亿里程碑——这份成绩单在"靶点-管线"框架下几乎解释不通。

**机制解释块（C3，系统承重比喻）：**

> 要理解这笔交易为什么值这么多钱，我们需要换一个评估框架。可以把一款药物从立项到变现的全链条想象成一座桥：桥墩是研发，桥面是注册，护栏是生产工艺，而桥通向的对岸是真正的商业化。过去五年，市场只在看桥墩——谁打了最多最新的桩子。但呋喹替尼的故事告诉我们：桥墩只是基础，真正稀缺的是把桥修通的能力。安罗替尼年销售额突破 40 亿、吡咯替尼将近 20 亿——三兄弟的桥已经通了，而很多看起来桥墩更先进的项目，桥面还没铺上。

### 复盘

| 维度 | 判断 |
|------|------|
| 顺的部分 | 框架替换完全适配（"靶点-管线"框架 → "全链条价值"框架）。级联拓扑把空之客原文的短评论拉成了系统性分析，信息量增大。"桥"比喻可延展（桥墩/桥面/护栏/对岸）。 |
| 假的部分 | "桥"比喻虽可延展，但不如亚特兰蒂斯比喻那样有"物理空间"——EGFR 激酶有 P 环、αC 螺旋、铰链区等可精确映射的位点；BD 交易链条的"桥墩、桥面"映射粒度较粗。 |
| 假在哪里 | **对象层面**：A2 系统承重比喻在"映射精度要求低"的场景可以工作，但比不了原域。→ 系统承重比喻的效果与"被比喻对象的结构清晰度"正相关。这可以细化为 R1 的触发条件。 |

## 重组三：B面小米"平行汇聚"拓扑 × 辰公子域 NASH 内容

### 设计

- **结构供体**：B面小米（平行汇聚拓扑 T2 + 螺旋上升 T3 + 升维收束）
- **内容供体**：NASH 药物研发（辰公子 NASH 文的内容）
- **调用对象**：T2（平行汇聚）、T3（螺旋上升）、A8（升维收束）、A3（时间堆叠）

### 结构方案

将 NASH 内容重组为四条平行线：

1. **失败线**：从 NGM Alafermin 看 NASH 药物为什么难做
2. **靶点线**：FXR / GLP-1 / THR-β / FGF 四路兵分多路
3. **玩家线**：大药企 vs Biotech vs 中国药企的布局差异
4. **未来线**：2021 下半年密集催化剂（展望）

四线最终汇聚于：NASH 是最后一个大型慢性病未开发市场。

螺旋上升主题词："难"——第一次出现是"诊断难"，第二次是"研发难"（靶点交织），第三次是"做成商业化更难"（全链条壁垒）。

### 复盘

| 维度 | 判断 |
|------|------|
| 顺的部分 | 平行汇聚拓扑可以让 NASH 景观文从"线性百科"升级为"多角度立体视图"。螺旋上升的"难"可以贯穿始终，增强文章纵深。 |
| 假的部分 | 辰公子原文的数据密度极高（靶点表、药企列表、临床表），平行汇聚拓扑会让数据分散到四条线里，信息密度下降。B面小米之所以能用平行汇聚，是因为每条线的数据量适中。 |
| 假在哪里 | **拓扑层面**：T2 平行汇聚在数据密度极高的景观文中可能不如 T1 级联有效。→ R8（拓扑匹配内容依赖）需要补充：数据密度高时，级联拓扑的认知负担更小。 |

## 重组四：辰公子"突发新闻+机制拆解"法 × B面小米内容

### 设计

- **结构供体**：辰公子（突发新闻钩子 + 级联拓扑 + 机制块 + 证据嵌入 + 诚实收束）
- **内容供体**：小米科技转型（B面小米的内容）
- **调用对象**：C1（突发新闻钩子）、T1（级联）、C3（机制块）、A6（证据嵌入）、A7（诚实收束）

### 关键段落验证

**入口钩子（C1，辰公子突发新闻型）：**

> 重磅！小米宣布自研AI大模型MiLM-1.3B在手机本地跑通，部分场景对标60亿参数级模型。这个消息在算力焦虑弥漫的2023年意味着什么？对一家曾经被嘲笑为"组装厂"的公司来说，又意味着什么？

**机制解释块（C3，辰公子拆解法）：**

> 我们需要从小米的技术研发三层体系来理解这件事。2019年技术委员会成立后，小米的研发形成了三个层次：面向当前产品的交付型研发、面向1-3年的预研、面向3-5年的颠覆式自研。AI大模型属于第三层。但问题在于第三层的成果不可能直接变成产品力——从字符串到手机上的实时推理，中间还有芯片适配、能耗优化、端侧量化等至少三个工程关卡。让我们逐一拆解。

### 复盘

| 维度 | 判断 |
|------|------|
| 顺的部分 | 突发新闻钩子天然适合小米的AI大模型新闻。级联拓扑可以把B面小米的散点叙事重组为"一条线讲清一件事"。机制块可以深挖技术细节（量化、端侧推理），这是原文选择了"宏观叙事"而没做的事。 |
| 假的部分 | 辰公子法的信息密度和专业术语密度极高，用在科技公司叙事上会让文章变成一篇"小米技术测评"而非"小米的成长故事"。B面小米的核心力量是"情感+反思+哲理"，辰公子法会丢弃这些。 |
| 假在哪里 | **文章级别**：结构供体的方法改变了文章的"交付类型"——从"公司成长深稿"变成了"技术分析深稿"。→ **新发现：动作/构件/规则的组合选择，决定了文章的交付类型。不同交付类型需要不同的对象组合方案（人格），这可能是一个新维度。** |

## 五、跨域重组总复盘

### 哪些地方顺

1. **A1（框架替换）跨域迁移很顺**：共识拆解、旧框架→新框架的逻辑在任何领域都可用
2. **A5（降维打开）跨域迁移最顺**：降维比喻可针对新域重新发明
3. **A6（证据嵌入）跨域迁移完全通用**：证据嵌入叙事 vs 证据堆成表格，这条规则与领域无关
4. **T1（级联）几乎万能**：任何内容都可以用级联组织

### 哪些地方假

1. **A2（比喻承重）的跨域迁移需要"重新发明比喻"**，不能搬原比喻。比喻的效果与被比喻对象的结构精度正相关
2. **自嘲式反转**跨域不通——需要作者与内容有个人历史
3. **T2（平行汇聚）在数据密度高的文章中效果变差**
4. **不同结构供体的组合会改变文章的交付类型**——这不一定是坏事，但需要意识到

### 是否出现"非搬不可"的对象

**没有。** 所有重组都不需要直接搬运原文的比喻、词汇或段落。需要搬运的只是"动作指令"和"规则约束"——这正是 H1 的核心主张。

### 关键新发现

**人格效应**：动作和构件的不同组合方案，决定了文章的交付类型。这可能是一个尚未命名的更高层维度——不是单个对象的问题，而是对象组合方式的问题。初步称为"人格（Persona）"。

- 辰公子人格：突发新闻钩子 + 级联 + 密集数据嵌入 + 单点承重比喻 + 诚实收束 → 交付类型 = 专业深度分析
- 空之客人格：共识拆解 + 历史比喻承重 + 降维打开 + 自嘲式反转 → 交付类型 = 冷峻反直觉评论
- 戴老板编辑人格：平行汇聚 + 系统承重比喻 + 升维收束 + 多线索时间堆叠 → 交付类型 = 宏大叙事深稿

→ **人格可能就是"作者风味"在通用对象框架下的精确定义。** 不是"冷峻""热血"这类模糊标签，而是"用了哪些动作、不用哪些动作、偏好哪种拓扑"的具体组合。
