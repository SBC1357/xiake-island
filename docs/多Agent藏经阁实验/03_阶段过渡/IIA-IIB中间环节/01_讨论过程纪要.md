## 一、讨论起点

本轮讨论的起点不是“是否继续 IIA”，而是：

1. 当前对象体系怎么落成人能理解的颗粒度
2. `调色盘` 是否继续沿用旧名
3. `条件性对象 / 稿型子型` 应放在哪种表结构里
4. `IIB` 应怎样开跑，样本由谁筛

## 二、关键收敛过程

### 1. `调色盘` 改名为 `人格`

讨论先收敛到一条明确判断：

1. `调色盘` 与“写作人格预设”足够接近
2. 为了提高人类可读性，正式命名统一改为 `人格`
3. 但其层级定位不变，仍是上层组合映射，不下沉为底层第五层

### 2. `条件性对象 / 稿型子型` 不作为新层

讨论中明确否定了两种做法：

1. 把它们当成新的独立层级
2. 把它们继续散落写在主对象表里

收敛后的判断是：

1. 它们属于“适用边界问题”
2. 应进入单独的边界表
3. 该表不是单列枚举，而是二维排布

### 3. `条件性对象表` 采用独立二维边界表

本轮讨论中，用户明确要求确认：

1. 这是单列还是二维表
2. 它是“大表的一组”，还是独立表

最终收敛为：

1. 不是单列
2. 不是新层
3. 是一张独立的二维边界表
4. 第一维写 `结构归属`
5. 第二维写 `边界归属`

### 4. `IIB` 不直接开主实验，先筛候选样本

讨论继续收敛到 IIB 的开工方式：

1. 不直接让各中心各跑各的正式样本
2. 先做 `IIA -> IIB` 中间总结
3. 先让各中心提交候选样本
4. 再统一冻结正式 IIB 样本

### 5. 样本筛选方式采用“各中心提候选，统一冻结”

这一点也在讨论中明确：

1. 不由各中心各自最终拍板
2. 不完全由总协调闭门指定

最终方式：

1. 各中心各提 `2-3 篇`
2. 总协调统一冻结正式名单
3. 主结论只看统一名单

### 6. 阶段过渡目录机制正式成立

最后一轮讨论把“这次过渡文档放哪儿”收成了规则：

1. 以后每个正式阶段结束后，都建立一个中间环节目录
2. 该目录专门收讨论过程、讨论结果、补充执行文档等
3. 当前首个目录即 `IIA-IIB中间环节`

## 三、这轮讨论产出的直接成果

3. `IIB 候选样本筛选` 进入待执行状态
