# IIB-IIC中间环节讨论过程纪要

状态：已记录  
日期：2026-04-04  
用途：简要保留 `IIB -> IIC` 过渡时的关键讨论路径

## 过程摘要

1. `IIB` 主样本跑完后，三中心都支持当前骨架，但对是否触发补充样本、是否可以直接写 `IIB 通过` 出现分歧。
2. `GPT-5.4` 明确指出：`脑机接口门诊的一个下午` 与 `当小鹿遇到宋佳` 两条边界仍未收束，因此按 `IIB` 方案应先补样，再正式互读。
3. 用户进一步指出，当前体系存在按作者风格“按图索骥”的过拟合风险，因此需要一轮去作者标签、去原文顺序、加入噪音的闭卷测试。
4. 讨论后确认：这轮闭卷测试不是中间小补丁，而是一轮足够大的独立验证任务，应单独立为 `IIC`。
5. 原本 `IIB` 的两篇补充样本 `刘家琨 / 蔡磊` 不再作为高优先级执行对象。
6. 第四中心补交了两篇更能直接裁决边界的候选样本，改为并入 `IIC` 的边界题库。
