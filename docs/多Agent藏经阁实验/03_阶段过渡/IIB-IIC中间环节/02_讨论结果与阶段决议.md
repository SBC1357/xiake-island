# IIB-IIC中间环节讨论结果与阶段决议

状态：已确认  
日期：2026-04-04  
用途：只记录 `IIB -> IIC` 过渡期间已经确认的阶段决议

## 证据摘要

- 已直接核对 `IIB` 三中心都支持“当前骨架不回退”，但对补充样本是否触发、是否可以直接写 `IIB 通过` 出现了正式分歧。
- 已直接核对第四中心已交 [第四中心_定向补样筛选.md](第四中心_定向补样筛选.md)，且两篇候选样本本地原文存在、正文非空。
- 当前线程已明确确认：闭卷生成与抗过拟合验证不是中间补丁，而是一轮独立大测试，应立为 `IIC`。

## 一、阶段判断决议

1. `IIB` 的总协调口径为：`支持当前骨架，但 IIB 需扩一轮`
2. 这条“需扩一轮”不再用原计划的 `刘家琨 / 蔡磊` 继续执行
3. `IIB` 遗留的两条边界，直接并入下一轮 `IIC`

## 二、立项决议

1. 下一轮正式阶段定为 `IIC`
2. `IIC` 的核心任务改为：`闭卷生成与抗过拟合验证`
3. 原先规划中的“开放任务与人工审核期”整体顺延为 `IID`

## 三、样本决议

1. 第四中心提交的两篇样本进入 `IIC` 边界题库：
   - `养老院的春节“喘息服务”`
   - `我和她，在相反的路上闪着光`
2. 这两篇不是 `IIB` 结果补件
3. 它们只服务于 `IIC` 的边界裁决题

## 四、执行决议

1. `IIC` 必须采用闭卷素材包
2. 出题方与答题方必须分离
3. 不再允许用作者名、刊物名、原标题、原段落顺序直接喂给执行中心
4. `IIC` 不再使用单一题型，而是改为 `A/B/C/D` 四档压力测试，难度递增
5. 当前已经做出的高整理度示例包，只能作为 `A类校准卷`
6. 如果执行中心连 `A类校准卷` 都系统性失守，应直接反查前序阶段结论是否被高估
7. `A/B/C/D` 四档试卷均应以独立正式包落盘后再下发，不允许边答边补题
8. 在 `A/B/C/D` 四档主卷之外，增设 `1` 道旧 `B` 稿仿写对照题
9. 这道附加题只作控制精度旁证，不并入 `IIC` 主通过门槛
