# IIC立项说明

状态：Active  
日期：2026-04-04  
用途：说明为什么 `IIC` 必须独立立项，而不能继续塞进 `IIB` 补样

`IIC` 独立立项的原因只有三条：

1. 它测试的是“脱离作者线索后，系统还能不能写”，这已经不只是 `IIB` 的边界补样
2. 它有能力推翻前面对骨架泛化能力的乐观判断，因此必须作为完整阶段留痕
3. 如果闭卷测试不通过，需要允许项目回到 `IIB` 甚至更早阶段做结构修正，而不是把失败埋在中间态里

因此：

- `IIB` 负责“结构扩样与边界裁决”
- `IIC` 负责“闭卷生成与抗过拟合验证”
- `IID` 再承接原定的开放任务与人工审核
