## 一、任务背景

当前阶段判断已经从“是否继续按原补样名单扩 IIB”转向“是否把闭卷测试独立立项为 IIC”。

在这个前提下，本文件只负责一件事：

为 `IIC` 的边界裁决题库挑出两篇高信息增益样本。

当前需要裁决的两条边界是：

1. **`问题现场群像稿` 稳定性边界**：无单一主角的现场报道型问题稿，到底是稳定稿型子型，还是仅是路由边界。
2. **`双人物共享核心轴` 升型边界**：两个人物稿的核心轴，到底是各自分别钉，还是可以共享一条。

`IIB` 冻结名单中的补充样本 `刘家琨，顽石的裂缝` 和 `蔡磊：为别人活，争一口气`，分别覆盖：
- 刘家琨 → 人物 + 文化评论（不是问题现场群像测试）
- 蔡磊 → 人物 + 行业/科研推进（不是双人物共享轴测试）

这两篇补充样本用于验证人物稿的**单人物核心轴稳定性**，而非上述两条边界的裁决。

因此，本文件的定位是：

**不替代 IIB 正式结果，只为 IIC 出题提供更准的边界样本。**

## 二、边界 1 裁决样本

### 样本：《养老院的春节"喘息服务"》

**本地入口**：
```
D:\汇度编辑部1\藏经阁\staging\editorial\source_articles\bingdian\2390471121-2653345603_1.json
```

**来源**：冰点周刊（bingdian），非 IIA/IIB 已跑样本

**边界测试能力**：

| 维度 | 脑机接口（已有主样本）| 养老院（本次候选）|
|------|---------------------|------------------|
| 问题类型 | 医疗技术（脑机接口）| 养老社会服务（喘息服务）|
| 群像结构 | 患者+医生多人物并置 | 多位老人+护理工+家属 |
| 核心议题 | 技术临床化边界 | 社会养老制度性困境 |
| 场景密度 | 门诊下午单一场景 | 春节养老院多场景 |

**为什么它比刘家琨更适合边界 1**：

- 刘家琨是**单人物核心轴**稿型（建筑师刘家琨为唯一主人格），其建筑实践是"好建筑"的活答案——这仍然是人物稿逻辑，不是问题现场群像。
- 养老院稿开篇即为场景切片："2月23日，早春的风还凛冽，内蒙古呼和浩特某县城养老院……老人聚在院子里，有人聊春节吃的席，有人小跑追上反映情况"——无单一主人公，多人物并行，问题在现场中自然浮现。
- 如果框架能把"养老院春节喘息服务"稳定路由为"问题现场群像稿"，说明这个子型不是医疗场景专属，而是一个通用稿型。
- 如果路由失败（例如只能把它强行压进"人物稿+问题稿混合"），则说明"问题现场群像稿"目前还只是路由边界，不是稳定子型。

**裁决价值**：
- 若是稳定子型 → 应进入条件性对象表，有独立触发规则
- 若仅是路由边界 → 应标注"仅用于路由分流，不产生独立子型"

## 三、边界 2 裁决样本

### 样本：《我和她，在相反的路上闪着光》

**本地入口**：
```
D:\汇度编辑部1\藏经阁\staging\editorial\source_articles\renwu\2103095721-2653246711_1.json
```

**来源**：人物向（renwu），非 IIA/IIB 已跑样本

**边界测试能力**：

| 维度 | 当小鹿遇到宋佳（已有主样本）| 我和她（本次候选）|
|------|--------------------------|-----------------|
| 人物关系 | 小鹿（脱口秀演员）+ 宋佳（演员）| "我" + "她" |
| 核心轴共享方式 | 双人物各自有轴，轴值不同但主题重叠 | 双人物在同一核心轴上互为镜像 |
| 轴方向 | 并行独立轴（各自"不想成为红色的花"）| 对照互补轴（"相反的路"但"都闪着光"）|
| 叙事权重 | 较为均衡的双线 | 互文对照式单核 |

**为什么它比蔡磊更适合边界 2**：

- 蔡磊稿是**单人物核心轴**稿型（蔡磊的"为别人活，争一口气"贯穿全篇），行业分析线（管线、呼吸机）是人物轴的附属——不测试双人物结构。
- "我和她，在相反的路上闪着光"标题本身就是两个人物结构，且核心轴是"相反的路上的闪光"——同一轴上两个对照镜像。
- 当小鹿遇到宋佳的共享轴是"女性成长/真实自我"，两个角色的轴值各自独立表达主题。
- 养老院文章则强制要求框架判断：两个人物的"相反路径"是在同一核心轴（"活出自己的光"）上的互补，还是两个各自独立的轴偶然并列？
- 如果是同一轴 → 双人物共享核心轴应升为稳定子型
- 如果是各自独立轴只是标题偶合 → 当前架构不需要升型

**裁决价值**：
- 若是稳定子型 → 应定义"互文对照式双人物共享轴"的触发条件和拓扑规则
- 若仅是偶合 → 应标注"仅用于识别双人物结构，不产生独立共享轴规则"
