# IIC-IID中间环节讨论过程纪要

状态：已记录  
日期：2026-04-04  
用途：记录 `IIC` 结果复盘后，为什么要降级结论并把重测改到 `IID`

## 关键讨论链

1. `IIC` 三中心交卷后，总协调先按横向同卷比较做了第一轮盲评。
2. 回读原文与题包时，发现当前 `A/B/C/D/附加题` 多数题包并非真正乱序素材池，而是已经带有较强整理和潜在编排逻辑的结构化素材包。
3. 用户进一步指出：这类按成稿友好顺序整理过的素材，天然携带素材排布逻辑，因此会高估系统的“自主选择、优先级判断、编排能力和抗过拟合能力”。
4. 总协调复核题包后确认：
   - `A类` 本来就是校准卷，不以“真实乱序”作为目标
   - `B类` 已按起点 / 鼎盛 / 变化 / 新路子分组，时间与结构关系被显著提示
   - `D类` 明显按关系组块和成稿友好顺序排布
   - 附加题已经按人物轴、行业判断、解决方案、环保、时代判断分区，显著降低了自主编排难度
   - 只有 `C类` 相对保留了更强的碎片感和自主组织压力
5. 因此，当前 `IIC` 不能再被解释成“严格闭卷、可直接检验自主编排与抗过拟合”的测试。
6. 用户进一步补了一层关键前提：平时真实给 AI 的写稿输入，并不是纯素材乱序包，而更接近 `素材 + 限制条件 + 开放性要求`。
7. 总协调接受这一点，并据此修正 `IID` 方向：`IID` 不能只测“乱序后会不会排”，还要测“在没有侠客岛编排逻辑直接兜底的情况下，面对真实需求入口会怎样成稿”。

## 收束结论

1. `IIC` 不作废，保留为一次有效的阶段结果。
2. 但 `IIC` 的阶段性质要降级，不能再当作“完整闭卷测试”使用。
3. `IIC` 更准确的定位是：
   - `结构化素材执行测试`
   - 或 `半闭卷执行测试`
4. 真正要验证“优先级选择、素材自主编排、去显性结构提示后的抗过拟合能力”，必须另起一轮更严格的测试。
5. 这轮更严格的重测不回写到 `IIC`，而是直接作为 `IID` 的核心任务。
6. `IID` 的题面要从“更脏的素材池”进一步修正为“更接近日常真实需求接口的需求卷”。
