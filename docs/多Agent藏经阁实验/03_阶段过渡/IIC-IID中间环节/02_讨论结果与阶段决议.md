# IIC-IID中间环节讨论结果与阶段决议

状态：已确认  
日期：2026-04-04  
用途：只记录 `IIC -> IID` 过渡期间已经确认的阶段决议

## 证据摘要

- 已直接回读 `IIC` 五个正式题包，确认其中多数题包并非真正乱序素材池，而是带有较强整理、分组和潜在编排逻辑的结构化素材包。
- 已直接回读 `IIC` 三中心结果，确认这轮数据仍可用于比较不同中心在结构化素材条件下的执行质量。
- 当前线程已明确确认：`IIC` 保留，但结论降级；真正重测方案改到 `IID`。
- 当前线程已明确确认：`IID` 不应只做“更乱的素材卷”，而应尽量模拟日常真实写稿入口，即 `素材 + 限制条件 + 开放性要求`。
- 当前线程已明确确认：为了保留纵向可比性，`IID` 可以沿用 `IIC` 同底稿，但只能“重封装”，不能直接复用 `IIC` 旧正式包。
- 已直接回读 `IID` 首版四卷，确认素材量仍然过厚，且题面一旦拼装就不容易写得太差，因此不足以把水平真正拉开。

## 一、IIC阶段决议

1. `IIC` 正式保留，不作废。
2. 但 `IIC` 的总协调结论降级，不再写成“严格闭卷、已验证自主编排与抗过拟合能力”。
3. `IIC` 的更准确定位改为：
   - `结构化素材执行测试`
   - 或 `半闭卷执行测试`
4. `IIC` 当前仍可保留的有效结论是：
   - 三中心在结构化素材条件下的执行质量比较
   - 人格 / 拓扑 / 禁用项 / 温度等限制条件的执行能力
   - 至少部分卷面上的结构调用能力

## 二、不能继续强说的结论

以下口径不再允许直接从 `IIC` 推出：

1. 系统已经稳定掌握优先级选择能力
2. 系统已经稳定掌握素材自主编排能力
3. `IIC` 已充分证明抗过拟合能力成立
4. `问题现场群像稿` 与 `双人物共享核心轴` 已在真正盲态下完成收束

## 三、IID阶段决议

1. 真正的重测改到 `IID`。
2. `IID` 的核心任务不再是一般性的“开放任务”，而是：
   - `真实需求形态下的写稿重测`
   - `去显性分组、去潜在编排提示的重测`
   - `优先级选择 / 素材编排 / 抗过拟合` 的正式复核
3. `IID` 的标准题面，应尽量贴近日常真实请求：
   - `素材`
   - `限制条件`
   - `开放性要求`
4. `IID` 允许沿用 `IIC` 同一批底稿，以保留纵向比较能力；但执行方式必须是“同底稿、重封装”。
5. `IID` 第二版四卷不再按“素材脏不脏”来分，而按素材供给状态来分：
   - `A类 充足素材基准卷`
   - `B类 富素材压缩卷`
   - `C类 稀素材提炼卷`
   - `D类 单点起稿卷`
6. `IID` 应把本轮 `IIC` 暴露出来的题包污染问题，当成显式设计前提，而不是隐含前提。
7. `IID` 重点要看：在不给“侠客岛编排逻辑”现成答案的情况下，AI 基于现有规则最终会怎样成稿。

## 四、当前一句话口径

`IIC` 保留，但只能证明“拿到整理过的素材包后，系统与不同中心谁更会写”；`IID` 要改成薄卷，重点测“会不会写 / 会不会删 / 会不会撑 / 会不会推开”。
