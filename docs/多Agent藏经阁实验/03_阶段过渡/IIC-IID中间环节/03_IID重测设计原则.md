# IID重测设计原则

状态：Draft  
日期：2026-04-04  
用途：给 `IID` 的真正重测提前冻结最小设计原则，避免重复犯 `IIC` 的题包编排错误

## 证据摘要

- 已直接回读 `IIC` 正式题包，确认多数题包存在不同程度的显性分组、隐性排序或成稿友好编排。
- 当前线程已明确确认：`IID` 必须承担真正重测，而不是继续沿用 `IIC` 的题包组织方式。
- 当前线程已明确确认：`IID` 不能只做“更乱的素材卷”，还应尽量模拟平时真实下给 AI 的写稿需求形态，即 `素材 + 限制条件 + 开放性要求`。
- 当前线程已明确确认：为了保留纵向可比性，`IID` 可以沿用 `IIC` 同底稿，但必须采用“同底稿、重封装”方式，不能直接复用 `IIC` 旧题包。
- 已直接回读 `IID` 首版四卷，确认素材量仍然偏大，且题面一旦拼装就不容易写得太差，因此第二版必须改成薄卷。

## 一、IID要测什么

`IID` 重点不再是“你能不能在整理过的材料上写好”，而是：

1. 在素材充足时，你会不会把文章基本写成立
2. 在素材过载时，你会不会主动删减并压出主线
3. 在素材稀缺时，你会不会提炼判断并把文章撑起来
4. 在只给一个核心点和少量锚点时，你会不会在边界内把文章推开
5. 在不给“侠客岛编排逻辑”现成答案的前提下，你拿到一份更像真实生产入口的需求，最终会写成什么样

## 二、IID的输入形态

`IID` 的题面应尽量贴近日常真实写稿请求，固定由三部分组成：

1. `素材`
   - 只提供客观事实、引语、背景、关系、时间、现场碎片和必要噪音
   - 不替执行中心做结构分层，不预先总结“冲突点 1/2/3”
2. `限制条件`
   - 例如：字数、面向读者、去 AI 味儿、想强调的重点、必须突出的信息、开头要抓人、哪些点不能写偏
   - 这是生产里常见的明确要求，允许进入 `IID`
3. `开放性要求`
   - 例如：有趣一点、不要太像八股文、把某个论点解释清楚、别太硬、节奏要顺
   - 这是生产里常见但不完全可程序化的模糊要求，`IID` 应显式纳入

## 三、IID题面强制规则

1. 对执行中心交付的，应是“需求单 + 素材池”，而不是“已经替你拆好的结构卡组”
2. 可以沿用 `IIC` 同一批底稿，但必须重做题面、重排素材、重混噪音
3. 不允许按成稿友好的逻辑分段给材料
4. 不允许再用“事实碎片 / 引语碎片 / 关系碎片 / 时间线碎片 / 冲突点碎片”这类标题
5. 不允许先分好“起点 / 变化 / 新路子”或“三组人物”这类天然结构块
6. 不允许通过标题性小节提前暴露结构路径
7. 素材池必须显式打乱顺序，并尽量用单列混排
8. 每包必须混入一定比例的低价值和弱相关噪音
9. `A类` 保持充足但克制，`B类` 刻意过载，`C类` 刻意偏薄，`D类` 刻意只给单点与锚点
10. 限制条件可以明确，但不得偷偷替执行中心完成编排判断
11. 开放性要求可以保留模糊性，但不得反向泄漏原稿结构

## 四、IID题包分层

建议固定分成四类：

1. `充足素材基准卷`
   - 测“会不会写”
2. `富素材压缩卷`
   - 测“会不会删”
3. `稀素材提炼卷`
   - 测“会不会撑”
4. `单点起稿卷`
   - 测“会不会推开”

## 五、IID允许保留的最小帮助

为了避免完全不可写，`IID` 仍可保留最小底线：

1. 不编造事实
2. 目标字数
3. 面向读者
4. 产出类型（深度稿 / 特稿 / 人物稿）
5. 用户特别强调的重点
6. 少量开放性要求

除此之外，不应再替执行中心做结构判断。

## 六、IID最小题面模板

`IID` 后续正式试卷建议统一成下面这类题面：

1. `任务`
   - 你要写什么类型的稿
2. `素材`
   - 事实、引语、时间、关系、现场碎片、背景和噪音混排
3. `限制条件`
   - 字数、强调点、禁写点、开头要求、读者对象等
4. `开放性要求`
   - 有趣一点、别太八股、把某个判断讲明白等

这样测出来的，才更接近真实生产里“不给你现成编排答案，你会怎么写”。

补充约束：

1. 不追求把素材堆满，而追求把能力压出来
2. 如果素材拼起来就天然像一篇完整稿，说明卷子还不够狠
3. `B/C/D` 三卷都应优先制造选择压力，而不是制造堆料幻觉

## 七、IID成功标准

如果 `IID` 通过，才更有资格说：

1. 当前系统不只是解释学
2. 当前系统不只是结构化素材执行器
3. 当前系统开始具备真实的优先级选择与素材编排能力
4. 当前系统开始能在“写 / 删 / 撑 / 推开”四类压力下被区分
