用途：压缩记录 `IID` 结束后围绕总审、对照组与后续推进方向的关键讨论路径

## 证据摘要

- 已直接回读 `IID` 试验组四卷成稿与对照组四卷成稿，并完成同卷横评。
- 已直接回读 `IIC` 方案与 `IID` 方案，确认 `IIC` 已被降级解释为“结构化素材执行测试”，`IID` 才是真实需求入口重测。

## 关键讨论路径

1. 先确认 `IID` 试验组是否已经把“会不会写 / 会不会删 / 会不会撑 / 会不会推开”真正拉开。
2. 再引入旧 `xiakedao` 写稿链路做对照组，比较新系统和旧系统在同题面下的差异。
3. 对照后确认：强模型在新系统下增益明显，但较弱模型在控制层仍有失稳风险。
4. 用户进一步指出：当前系统仍未完成编排规则和侠客岛反推建设，因此不能把 `IID` 通过误读成“系统成熟”。
5. 最终决定：追加一轮第三方独立审核，并为新 session 准备零时差交接文档，避免上下文继续膨胀。
