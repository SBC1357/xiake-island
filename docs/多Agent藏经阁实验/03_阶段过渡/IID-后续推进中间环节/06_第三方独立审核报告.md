## 一、独立结论

我的独立判断是：

`IID` 当前可以支持“底层骨架继续往下建设”，  
但还不足以被冻结成一条干净、强口径的“新系统已被证明通过”的总审结论。

更准确的说法应当是：

1. 可以保留：`IID` 证明新的需求包形态确实比 `IIC` 更接近真实入口，也确实把 `写 / 删 / 撑 / 推开` 这四类能力拉出来了。
2. 可以保留：强模型，尤其 `GPT-5.4` 与 `Opus/Sonnet 4.6`，在这套题面下已经能交出有约束力的正面卷面。
3. 必须降级：当前不能把 `GPT-5.4` 的优势写成“新系统相对旧系统已被干净对照证明的稳定增益”，因为对照链路并不干净，且试验组与对照组执行合同不对称。
4. 必须降级：当前不能把 `IID` 写成“系统成熟”或“侠客岛化已完成”，因为编排规则、侠客岛反推层、弱模型控制层都还没有闭合。
5. 必须降级：当前也不宜把 `IID` 直接写成“可冻结总审”，因为固定流程要求中的三中心互读补充结论，在 `IID` 结果区里并没有看到正式落盘。

## 二、当前最深的五个问题

### 1. 对照组不够干净，当前“增益归因”不能写得太满

[GPT-5.4 对照组执行记录](../../02_阶段结果/IID期/对照组_xiakedao旧系统/GPT-5.4/01_需求卷执行记录.md) 明确写出四卷都经过 `draft-with-evidence -> drafting -> quality -> semantic`，且 `writing_trace.formal_asset_ids` 都是 `["l1:m1_argument_logic_rules"]`。  
而 [writing/service.py](../../../../src/modules/writing/service.py) 又明确说明：只要 `m1_rules` 可用，就会把 `l1:m1_argument_logic_rules` 加入正式注入资产。

这说明 `GPT-5.4` 对照组并不是一个“完全不带新规则资产”的旧系统基线。  
因此，当前最多只能说：

- 卷面上能看到 `GPT-5.4` 试验组存在增益倾向；
- 但这份增益还不能被写成“干净地来自新系统编排层”的因果证明。

### 2. 试验组与对照组执行合同不对称，当前比较更像“直答 vs live 轻链路”

[GPT-5.4 试验组执行记录](../../02_阶段结果/IID期/GPT-5.4/01_需求卷执行记录.md) 明确记录：本地 `/health` 未返回响应，因此本轮不把 `draft-with-evidence / drafting / quality / semantic` 写成已实际执行，而只记为“基于 `IID` 正式包的直接盲答执行”。

但 [GPT-5.4 对照组状态卡](../../02_阶段结果/IID期/对照组_xiakedao旧系统/GPT-5.4/00_状态卡.md) 明确记录四卷已经走完整条 live 轻链路。

这意味着当前对照不是“新系统 live 链路 vs 旧系统 live 链路”，  
而更接近：

- 试验组：新题包下的直接盲答
- 对照组：旧本地轻链路下的 live 输出

在这种合同不对称前提下，任何“系统性增益归因”都必须收着写。

### 3. 弱模型控制层没有稳住，`GLM-5` 的 D 类不能按“通过”轻描淡写

[D类正式包](../IIC-IID中间环节/08_D类_单点起稿卷_正式包.md) 明确要求：

1. 不得编造素材池之外的事实
2. 只能在 `稳定与自由 / 婚育与事业 / 母职与非母职 / 差异如何成为映照` 这些边界内往外推开

但 [GLM-5 试验组 D 类成稿](../../02_阶段结果/IID期/GLM-5/13_D类_单点起稿卷.md) 直接写入了题包里没有给出的具象细节，包括人物名、大学宿舍、北京开会、大理生活、朋友圈场景、咖啡馆会面等。

这不是普通的“文笔保守”或“字数略低”，而是：

- 在单点起稿场景下，为了把文章推开，弱模型开始自行补情节；
- 说明当前控制层还不能稳定压住 `D类` 最危险的越界冲动。

因此，`GLM-5` 在 `IID` 试验组上的问题，应该被明确写成“控制层失稳”，而不只是“可再加强细节”。

### 4. 当前“通过”过度依赖强模型卷面和执行中心自评，弱模型失手会被掩盖

[GLM-5 IID期汇总结论](../../02_阶段结果/IID期/GLM-5/04_汇总结论.md) 把 `D类` 写成“能推开、无编造、符合门槛”；  
但与 [D类正式包](../IIC-IID中间环节/08_D类_单点起稿卷_正式包.md) 和 [GLM-5 试验组 D 类成稿](../../02_阶段结果/IID期/GLM-5/13_D类_单点起稿卷.md) 对读后，会发现这份自评明显高估了自己的控制质量。

这说明当前总审如果主要依赖：

- 强模型正面卷面
- 各中心自评
- 总协调人工归纳

那么弱模型最关键的失败点，很容易在“整体看起来通过”的话术里被抹平。

### 5. 当前 `IID` 证明的是“骨架开始生效”，不是“编排规则和侠客岛反推层已闭合”

[IID 方案](../../01_实验方案/藏经阁IID期实验方案_20260404.md) 测的是 `会不会写 / 会不会删 / 会不会撑 / 会不会推开`；  
[00_实验中心总表.md](../../00_实验中心/00_实验中心总表.md) 和 [新Session零时差交接](04_新Session零时差交接.md) 也都明确写了：当前未完成项仍然是编排规则、侠客岛反推层、弱模型控制层。

所以 `IID` 当前真正能证明的是：

- 新需求包确实把真实入口压力带进来了；
- 底层骨架对强模型成稿确实开始产生约束力；
- 系统问题已经开始从“混成一团”变成“可以定位到具体层”。

但它还不能证明：

- 编排规则已写完；
- 侠客岛反推层已建成；
- 运行时调用链已经被 cleanly 证明优于旧链路；
- 全模型控制力已经稳定。

## 三、哪些当前结论可以保留，哪些必须降级

### 可以保留

1. `IIC` 降级为“结构化素材执行测试”的处理是合理的，`IID` 作为真实需求入口重测是必要的。
2. `IID` 当前题面设计比 `IIC` 更接近真实生产入口，也更容易把能力差拉开。
3. `支持当前骨架` 这半句可以保留，但它的含义只能是：底层骨架值得继续建设，不是已被最终证明。
4. “下一步重心应转向编排规则与侠客岛反推建设”这条方向判断可以保留。

### 必须降级

1. `IID 通过，进入下一阶段` 不能直接冻结成强口径，应改成“带保留项的阶段性通过”。
2. `GPT-5.4` 的相对增益，当前只能写成“卷面上有增益倾向”，不能写成“已完成干净归因”。
3. `GLM-5` 当前不能写成“只是字数偏保守”，应明确写出 `D类` 控制层失稳。
4. 当前不能把 `IID` 结果解释成“系统成熟”或“侠客岛化完成”。

## 四、我对IID当前阶段判断的独立意见

如果必须在三种阶段判断里选，我不会选“回退 I 期”，也不建议现在再泛泛扩样一轮。  
我的独立口径更接近：

`支持当前骨架，但当前只宜作为带保留项的阶段性通过，不宜直接冻结成强总审口径。`

更具体一点：

1. `IID` 不需要整轮推倒重跑。
2. 但也不建议立刻把“通过”写成已经足够干净、足够闭环的终版结论。
3. 当前最该做的不是再泛泛问“会不会写”，而是把 `IID` 暴露出来的缺口直接转成建设任务。

## 五、下一步最该做什么

优先级建议如下：

1. 先进入 `编排规则建设`
   - 重点不是抽象谈“风格”，而是把 `写 / 删 / 撑 / 推开` 四类动作里真正起作用的编排规则写死
   - 至少先补：主线选择、素材取舍、回声与主承重区分、单点起稿防越界、结尾收束
2. 再进入 `侠客岛反推建设`
   - 把 `IID` 题面里已经显性的约束，反推成 runtime 可调用、可留痕、可解释的规则接口
   - 把“题包压力”翻译成“运行时怎么调规则、怎么留 trace、怎么复核”的实现清单
3. 单独补 `弱模型控制护栏`
   - 特别是 `D类`：禁止弱模型为了推开而自造姓名、地点、会面桥段
   - 如果后续要做验证，优先做弱模型的单点起稿定向复核，而不是再广撒一轮
4. 如需后续补测，只做小范围定向补测
   - 一条测“干净对照”
   - 一条测“弱模型 D 类防越界”
   - 不建议现在回到泛化扩样

## 六、建议冻结的最小口径

如果现在必须留一句可供后续沿用的话，我建议冻结成下面这句，而不是更强的话：

`IID 当前只足以支持“底层骨架继续建设”：强模型在真实需求入口下已出现正面卷面，但干净对照归因、编排规则、侠客岛反推层与弱模型控制仍未闭合。`
