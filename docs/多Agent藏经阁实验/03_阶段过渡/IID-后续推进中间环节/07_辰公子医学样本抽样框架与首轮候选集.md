## 一、这轮抽样怎么定义

这轮不是统计学意义上的“随机代表性抽样”，而是面向当前空白的“分析型补空白抽样”。

当前要补的不是题材面子，而是 4 类规则空白：

| 空白类型 | 这轮样本要帮我们看清什么 |
| --- | --- |
| 核心通用规则 | 哪些底层规则在真实医学写作里反复成立，而不是只在公号名篇里成立 |
| 医学条件规则 | 哪些规则只在特定触发条件下调用，例如监管博弈、临床数据降维、机制解释桥、产业传导 |
| 医学稿型子型 | 医学写作里是否存在稳定子型，例如人物-问题混合稿、临床结果解释稿、赛道综述稿、公司策略稿 |
| 编排规则 | 同一类素材进入正文后，主问题怎么立、先给什么、何时切机制、何时回到产业、如何收束 |

## 二、这轮抽样的边界

- 样本单位：`source_articles` 下的原始正文文章，不用 `distilled_articles`，也不复制样本出仓。
- 样本路径模板：`../../../../../藏经阁/staging/editorial/source_articles/chengongzi/{yjwyj|ysd}/articles/{article_id}.json`
- 排除项：`exceptions.json`
- 去重口径：标题完全重复的成对样本不同时纳入首轮候选集
- 初始规模：`24` 篇辰公子候选样本
- 抽样方式：按“当前规则空白”拆成 `6` 个桶，每桶 `4` 篇
- 选样偏好：优先选 `2k+` 的解释/分析稿；只保留少量边界短稿用于观察非常规写法
- 项目文稿口径：项目文稿先与 corpus 去重；重复稿直接回挂到藏经阁文章 `article_id`，非重复稿才进入补充 `json`。

这轮 `24` 篇候选集的分布为：

- 来源：`医界望远镜 13` 篇，`药时代 11` 篇

- 篇幅：`4k-6k 10` 篇，`2k-4k 8` 篇，`6k+ 5` 篇，`<2k 1` 篇

## 四、这批样本过审后，分析怎么做

如果这批样本质量通过，下一步不先泛读全库，而是先对这 `32` 篇工作集逐篇回到既有骨架口径，回答一组固定问题。

| 固定问题 | 要回答什么 |
| --- | --- |
| 主人格是什么 | 这篇的主要说话方式和承重气质是什么 |
| 主拓扑是什么 | 级联、平行或其他既有拓扑判断 |
| 触发了哪些条件性对象 / 稿型子型 | 哪些边界对象真的被触发了 |
| 哪些旧规则失效或需要降级 | 旧骨架里哪些规则在医学域需要改写、降级或暂缓 |
| 这篇最可能暴露哪条边界 | 这篇文章最能逼出哪条规则或对象边界 |
| 如果要反向调用，应给出哪 `1-5` 条写法路线 | 后续若要调用这类写法，可抽出哪些路线 |
| 最终你会把它路由到哪一条主路线 | 这篇更应进入哪一条主路由，而不是停在事后解释 |

这批样本已经够启动下一步，不需要再回去补“名篇样本”。  
原因很简单：它们更接近你真实要写、要拆、要调用的医学工作域。
