# IIE-IIF中间环节讨论过程纪要

状态：已记录  
日期：2026-04-04  
用途：记录 `IIE -> IIF` 过渡期间的关键讨论链，并作为固定承载入口吸收原先散落的独立过程文档。

## 关键讨论链

1. `IIE` 正式启动后，讨论重心已经从 `IID` 后续收束转成 `IIE` 批次执行、回退重整、三路复核和 `IIF` 前置准备。
2. 原先一批挂在 `IID-后续推进中间环节` 的活跃讨论件，虽然内容已属于 `IIE -> IIF`，却没有及时迁入新过渡目录。
3. 迁入后，又继续以 `11-18` 这种独立编号文件堆叠，导致过渡目录被过程耗材挤满，违背了固定少数载体的卫生目标。
4. 因此，本轮不再保留滚雪球式独立过程件，而是把有效内容折叠回：
   - `01_讨论过程纪要.md`
   - `02_讨论结果与阶段决议.md`
   - `03_补充执行文档.md`
   - `04_待执行事项.md`
   - `10_本轮讨论强留痕记录.md`
5. 在继续压 `IIE` 边界的过程中，用户明确指出：若干商稿样本并不是“天然单核文章”，而是在真实写作现场为了传播任务而把热点事件、产品任务、赛道判断或人物材料硬缝成一篇。
6. 这直接暴露出：当前很多既有说法，可能不是最小单元，而是“最小单元的有限组合”被误当成单一规则或单一稿型。
7. 因此，本轮最终没有继续把 `IIE` 往“全量汇总 + 桥接前压缩”推进，而是改成：`IIE` 提前终止，结果保留为证据；正式开启 `IIF`，先做最小单元审计，再拿 `IIE` 的 `32` 篇样本复验。

## 折叠前曾单列的内容

下面这些内容曾被单独成文，但现在不再单独保留：

1. 桥接层候选资产筛选标准
2. 桥接层候选资产评分卡
3. 后续实验规划整合草案
4. `IIE` 执行草案
5. 批次 A 校准批执行单
6. `C0` 撤回卡
7. `IIE` 重启交接 prompt
8. `C0` 旧口径重写版过渡卡

它们的有效信息已并回固定载体，结果信息则以结果区文档为准。

## 收束结论

1. `IIE` 的结果型文档继续留在结果区，但整体只保留为证据区，不再继续扩期。
2. `IIF` 启动后，本目录只负责承接“终止原因、阶段决议、执行指针、待办和强留痕”。
3. `IIE-IIF` 目录不再接受“一轮讨论一个新文件”的做法。
4. 后续如再出现新的过渡讨论，默认回填本文件或 `02/03/04/10`，不再单独编号。
