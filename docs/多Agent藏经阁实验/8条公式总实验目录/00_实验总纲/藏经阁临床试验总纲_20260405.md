## 一、总目标

这条临床试验链只验证一件事：

`写作系统新架构能否成立，并最终反向驱动侠客岛。`

整条链的顺序仍是：

1. 先确认 `存什么`
2. 再确认 `怎么通用地存`
3. 最后确认 `怎么被稳定调取`

## 二、I~III期大目标

### I期：储藏方案圈定期

`I期` 的目标不是证明“这套系统已经通用”，而是先把当前主链圈定住，并写清：

1. 当前主链到底是什么
2. 范围内资产如何回填到主链
3. 最小单元、规则、组合资产之间怎样逐项接线
4. 哪些东西已经冻结、哪些东西仍未冻结，并把未冻结项正式交接给 `II期`

### II期：储藏规范通用性验证期

`II期` 的目标不是再找一套新方案，而是验证：

1. 当前冻结方案能否跨更多样本稳定成立
2. 未冻结项和候选池能否通过扩样与上题被收口或淘汰
3. 规则、边界、组合方式能否扩成通用规范
4. 在生成、重组、对抗条件下，规范是否仍然稳定

### III期：调取规则确认与系统反推期

`III期` 的目标是验证：

1. 面对不同任务时，到底该调什么对象、规则、范围和大纲
2. 这些调取规则能否稳定留痕、稳定回指
3. 它们能否反向驱动侠客岛运行时与重建

## 四、方法库与验证职责

| 方法 | 主要职责 | 主要验证什么判断 | 不能单独证明什么 | 当前主要阶段 |
| --- | --- | --- | --- | --- |
| 推演 | 从既有样本中抽候选类型、候选规则、候选形态 | `某类目标 / 目的 / 规则 / 形态是否可能存在` | 不能单独证明该对象已收口、更不能单独宣称升期 | `I期` 已完成、`II期` 扩样前 |
| 反推 | 从已知好文章回拆到目标、范围、单元、规则、大纲 | `现有主链能否解释已知成立样本` | 不能单独证明系统会写 | `IIA / IIB` |
| 正推 | 从目标、范围和资产池往前推到大纲或结构解 | `当前主链是否闭合到足以推出结构结果` | 不能单独证明成文质量 | `IIA` |
| 生成式试验 | 用资产池与规则生成大纲或整稿 | `这套架构是否真的可用、会写` | 不能单独证明调取规则稳定 | `IIC / IID / IIF` |
| 对抗性试验 | 在噪声、冲突、去作者线索、缺材料条件下强行验证 | `系统是否抗过拟合、抗误路由、抗假模式` | 不能替代基础归类和规则冻结 | `IIC / IIF / III期` |

### 当前方法分工补充

1. 扩大样本量，主要负责验证 `候选最小单元 / 候选范围分支 / 候选组合方式` 是否在新样本中稳定重复出现。
2. 直接上题，主要负责验证 `已冻结资产` 是否真能被下层导出，以及 `待组合` 资产是否能被同一组下层关系稳定接住。
3. 任何升格动作都不能只靠扩样，也不能只靠上题；至少要形成“扩样证据 + 题型证据”之一，且不能与当前共享母表冲突。
4. 任何验证结果都必须落到 `II期` 正式结果文件与 [03_过程强留痕/00_总留痕.md](../03_过程强留痕/00_总留痕.md)；只存在于聊天里的判断一律按未发生处理。

## 五、材料齐整度梯度

| 材料级别 | 定义 | 何时主要变成题目 | 主要服务什么验证 |
| --- | --- | --- | --- |
| A类：全文齐整且已有高质量成文 | 原文完整，且已知这是高质量样本 | `IIA` 反推、接线、正推 | 验证主链是否能解释“已成立文章” |
| B类：全文齐整但无标准拆解 | 原文完整，但没有现成标准答案 | `IIA / IIB` | 验证主链和规范能否稳定覆盖新样本 |
| C类：素材包齐整 | 有事实、人物、证据、边界，但不给原文 | `IIA / IIB / IIC` | 验证能否从材料出结构、从结构出文 |
| D类：素材不齐整或含噪声 | 信息缺口大、线索冲突、夹带干扰项 | `IIC / IIF / III期` | 验证对抗能力、路由稳定性与诊断能力 |

## 六、题型库

| 题型 | 出题方式 | 主要验证阶段 | 主要验证成果 |
| --- | --- | --- | --- |
| 拆解题 | 给全文，要求回拆 `目标 -> 范围 -> 单元 -> 规则 -> 大纲` | `IIA` | 主链是否能解释样本 |
| 接线题 | 给定一批最小单元、规则、组合资产，要求写出逐项匹配关系 | `IIA` | 逐项接线是否真的闭合 |
| 范围筛选题 | 给定同一素材池，要求按不同人格/目的筛出有效内容和有效逻辑 | `IIA` | `写作目的 * 人格 = 写作范围` 是否可用 |
| 大纲重建题 | 给定目标、范围和资产池，只要求推出文章大纲 | `IIA` | 公式 `5~8` 是否闭合 |
| 组合重建题 | 给定素材与规则，要求只重建内容组合与逻辑组合，不写全文 | `IIA / IIB` | 规范能否稳定重组 |
| 闭卷生成题 | 去作者标签、去原文顺序、只给处理后材料包 | `IIC` | 系统是否真的会写、是否抗过拟合 |
| 加噪对抗题 | 在素材包中混入噪声、冲突和伪承重点 | `IIC / III期` | 系统能否识别主承重与误导项 |
| 调取路由题 | 给任务，不给答案，要求先判断应调哪些对象、规则、范围和大纲 | `III期` | 调取规则是否稳定 |
| 失效诊断题 | 给失败稿或错误大纲，要求定位失败落点 | `III期` | 系统是否具备回指和诊断能力 |

## 七、各阶段怎样算通过

### I期升入 II期

必须同时满足：

1. `8` 条公式主链已冻结
2. 范围内资产与原藏经阁 `L1/L2` 资产已完成归位
3. `最小单元 / 规则 / 组合资产 / 大纲形态` 的逐项匹配关系已冻结成共享母表
4. 未冻结项与候选池已经被压缩，并已明确交接到 `II期` 的验证路线
5. 当前主链进入 `II期` 后，不再需要先在 `I期` 内继续滚动上题

### II期升入 III期

必须同时满足：

1. 当前方案已扩成稳定规范
2. `B/C` 类材料上的重组、生成与对照验证已经稳定
3. `D` 类材料上的对抗验证不再高频击穿核心骨架
4. 失败时已经能稳定回指到对象、规则、范围或逻辑层

### III期结题

必须同时满足：

1. 调取规则已稳定，不再高频改口
2. 运行时 trace 已能稳定回指藏经阁对象
3. 侠客岛运行链已经能被这套调取规则反向约束
