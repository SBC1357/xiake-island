## 一句话定义

这条路由的职责只有一个：  
**判断当前任务理解是否已经稳定到足以进入冻结阶段；如果还不稳，就只允许先纠偏，不允许继续写。**

## 一、路由卡

| 项 | 内容 |
| --- | --- |
| 路由编号 | `R-01` |
| 上游节点 | `需求对齐回执层` |
| 下游节点 | `输入冻结层` |
| 主要执行者 | 当前接手 Agent；如已分工，则由冷启动 Agent 执行 |
| 触发时点 | 原始任务包刚进入系统、且任何冻结整理或写稿动作开始前 |
| 禁止动作 | 未经过本路由判定，不准进入 `A-E` 冻结文件生成，不准直接开写，不准下发写作任务 |
| 核心目标 | 先把“这次到底是什么任务”钉死，再允许把材料压成冻结输入 |

## 二、上游输入合同

本路由只吃上游已经存在的原始任务包，不要求用户先建目录或做二次整理。  
标准输入由 `6` 类字段组成：

1. `任务描述`
   - 用户在对话框里给的任务目的、需求、人话 brief
2. `原始材料`
   - 文件路径、附件、截图、PDF、会议资料、客户原文
3. `样稿/参考件`
   - 样稿、往期稿、希望靠近的文风或结构参考
4. `修改意见/禁区`
   - 用户批注、禁区、法务口径、不能写什么
5. `显式格式要求`
   - 平台、受众、篇幅、标题需求、是否带图
6. `用户纠偏记录`
   - 同一任务内用户前面已经明确说过的纠偏，不得在本路由中丢失

如果这些内容都只存在于对话框，本路由也必须直接消费；  
**“还没进目录”不构成本路由 blocker。**

## 三、本路由要判断什么

这条路由不判断“文笔好不好”，只判断任务理解是否已经可冻结。  
必须依次判断下面 `9` 个问题：

1. `真实写作场景` 是否已锁定  
   - 例如：新研究要写透、大会热点要抢判断、新获批/新中国数据要解释意义、核心机制要讲清、病种框架要重画、公司战略要讲清、患者最关心怎么选、监管/方法论争议要讲规则
2. `一级稿型` 是否已锁定  
   - 例如：研究解读类、机制解读类、领域与产业叙事类、患者沟通类
3. `主二级稿型` 是否已锁定  
   - 例如：研究/文献深度解读、领域新进展解读、监管/方法论争议解读、沸点争鸣、单产品机制深挖、双路线机制差异、领域格局分析、产业故事、药物/靶点发展史、患者向-治疗选择、患者向-AE与管理、患者向-疾病认知翻译
4. `主样稿母本` 是否已锁定  
   - 二级稿型一旦成立，后续必须绑定一种样稿母本，作为结构与节奏参照
5. `交付物` 是否已锁定  
   - 例如：首稿、改稿、补稿、重组稿、图解补充
6. `受众 / 平台 / 篇幅` 是否已锁定  
7. `主角度 / 主问题` 是否已锁定  
8. `主证据链 / 主信息链` 是否已锁定  
9. `必写点 / 禁区` 是否已锁定

只要其中任意一项仍明显漂移，这条路由就不得放行到冻结层。

## 四、路由判定规则

### 通过条件

同时满足下面 `4` 条，才允许进入冻结层：

1. 上面 `9` 个判断项都已经能用人话写成短回执
2. 用户纠偏过的关键边界已经被吃进去，没有被重置
3. 不存在直接冲突到无法拍板的任务理解
4. 当前 Agent 已经知道下一步要冻结哪 `5` 个文件，而不是继续泛泛解释任务

### 不通过条件

出现下面任意一种情况，就不得放行：

1. 真实写作场景还在两种以上入口之间摇摆
2. 一级稿型虽然锁了，但主二级稿型还在同级内反复摇摆
3. “交付什么”还没分清是写稿、改稿还是整理材料
4. 受众、平台、篇幅仍明显不稳
5. 主角度和主证据链还没被钉成一句话
6. 主二级稿型虽然锁了，但还没绑定主样稿母本
7. 用户已经明确纠偏过，但当前理解又滑回老路线
8. 当前 Agent 开始要求用户补 `G0`、表格、目录、冻结卡之类中间产物
9. 当前任务没有稳定命中已知场景入口，且当前 Agent 仍试图自造新稿型

## 五、路由输出与分流

本路由只允许分成 `3` 条路：

### 路 A：`PASS_TO_FREEZE`

适用条件：
- 任务理解已稳定

输出：
1. 一份极短需求对齐回执
2. 一份内部的“冻结前任务摘要”
3. 一份已锁定的`真实写作场景 × 一级稿型 × 主二级稿型 × 主样稿母本` 结果
4. 放行到输入冻结层

下游动作：
- 开始生成 `A_客户brief冻结.md`
- 开始生成 `B_原始事实锚点表.md`
- 开始生成 `C_参考样稿冻结清单.md`
- 开始生成 `D_禁区与必须点.md`
- 开始生成 `E_写作合同冻结卡.md`

### 路 B：`CORRECTION_LOOP`

适用条件：
- 任务理解还不稳，但可以通过一次极短纠偏解决

输出：
1. 一份极短需求对齐回执
2. 等待用户只纠偏理解

下游动作：
- 用户纠偏后，重新跑本路由
- 未重新跑通前，不准进入冻结层

### 路 C：`HARD_BLOCK_MINIMAL`

适用条件：
- 缺的是不可替代的原始材料，或者用户输入内部存在直接冲突，当前 Agent 不能自判

输出：
1. 一份最小 blocker 说明
2. 只指出缺哪一类原始材料或哪一对输入互相冲突

下游动作：
- 等用户补原始材料后，重新跑本路由

硬约束：
- 这里只能要“原始材料”，不能要整理后的中间件
- 禁止要求用户补目录、补卡片、补摘要表、补冻结表

## 六、标准回执格式

本路由对用户只允许输出下面这种短格式，不允许发长问卷：

```text
我将按以下理解执行；如有偏差你直接改：

- 真实写作场景：
- 一级稿型：
- 主二级稿型：
- 主样稿母本：
- 交付物：
- 受众 / 平台 / 篇幅：
- 主角度 / 主问题：
- 主证据链 / 主信息链：
- 必写点：
- 禁区：
```

要求：

1. 只回显理解，不要求用户做预处理
2. 不问“请确认以上信息”
3. 不把材料整理工作甩回给用户
4. 一次只对齐任务理解，不顺手开始写正文

## 六点五、一级稿型与二级稿型的切换规则

1. 一级稿型只在大方向明显错了时才允许重切；一旦重切，原二级稿型和原主样稿母本一律失效。
2. 只有主二级稿型明显装不下主任务时，才允许切到同支候选或旁支候选。
3. 一旦切到新的主二级稿型，必须同步重绑主样稿母本，并重跑一次本路由。
4. `研究/文献深度解读` 与 `单产品机制深挖 / 双路线机制差异` 的分界：
   - 主任务是把某篇研究或某组证据讲透，走 `研究/文献深度解读`
   - 主任务是把某个产品 / 路线的机制讲狠，走 `单产品机制深挖`
   - 主任务是把两条路线的关键机制差异讲清，走 `双路线机制差异`
5. `领域新进展解读` 与 `沸点争鸣` 的分界：
   - 主任务是解释新进展补了什么，走 `领域新进展解读`
   - 主任务是抢判断、拆争议、占观点位，走 `沸点争鸣`
6. `产业故事` 与 `药物/靶点发展史` 的分界：
   - 主任务是讲公司、平台、管线，走 `产业故事`
   - 主任务是讲一个药物 / 靶点沿时间线的发展，走 `药物/靶点发展史`
7. 如果当前任务未稳定命中已知场景入口，只允许：
   - 归并到现有最接近入口后重跑本路由
   - 或进入 `HARD_BLOCK_MINIMAL`
8. 如果二级稿型还不够稳，需求对齐时允许同时回显：
   - `1` 个主二级稿型
   - `1` 个同支候选二级稿型
   - `1` 个旁支候选二级稿型
9. 禁止靠自造新稿型绕过本路由。

## 八、本路由压掉的核心风险

这条路由要压掉的，不是语法问题，而是最上游的任务偏航风险：

1. `题眼跑偏`
   - 还没搞清这稿到底想讲什么，就直接开写
2. `场景误判 / 稿型误判`
   - 把“新进展”误写成“研究深解”，把“机制差异”误写成“泛科普”
3. `受众漂移`
   - 医学传播稿越写越像论文综述，或越写越像泛科普
4. `交付物误判`
   - 改稿任务被当成新写稿，或者整理任务被当成正文交付
5. `禁区失真`
   - 用户已经明确纠偏过的边界，在后续又被洗掉
6. `样稿乱借`
   - 稿型未锁定就乱借样稿，或者同时混借多个主样稿导致路线漂移

## 九、III期怎么测这条路由

这条路由进入 `III期` 后，至少要测下面 `4` 个观察项：

1. `回执后首稿的 route_alignment` 是否明显高于无回执条件
2. `audience_style` 是否更稳
3. 用户在首轮后指出“你理解错了”的频次是否下降
4. 首稿后需要退回上游重冻的比例是否下降

如果这 `4` 项没有改善，就说明这条路由还只是礼貌确认，不是高杠杆约束。

## 十、当前口径下的最小结论

`R-01` 不是额外加的一道客套工序。  
它是整个系统最上游的任务压缩器。

只有这条路由先把“这次到底是什么任务”钉住，  
后面的冻结层、资产路由层、写作合同层和 revise 层，才可能在同一个目标上接力。
