# IID A/B 对照协议

状态：已冻结（协议框架）/ 待 `IIF` 正式执行  
日期：2026-04-06  
用途：`IID` 正式执行资产；写死 A/B 对照的所有执行参数，供 `IIF` 直接消费

## 核心因果命题

> 在不看参考答案的情况下，白盒系统**因为**调用了 `8` 公式，**所以**能写出高质量稿。

## A 组定义

- 名称：`A 组（白盒 + 8公式约束）`
- 管线：白盒编排器全链路
- 约束：白盒管线 + 当前已接线的公式驱动约束进入 prompt / contract / scoring；以本轮 runtime 可直接核到的字段为准，不把未独立成键的公式位提前写成“已全量编码”
- 闭卷条件：不看参考答案（`gold/`）、不看样稿、不联网
- 输入：题目自带 `inputs/` + 仓内正式规则文件 + `raw_source_analysis.md`

## B 组定义

- 名称：`B 组（同模型同材料同闭卷，去掉公式约束）`
- 管线：同一 LLM 模型、同一材料包、同一闭卷条件
- 约束：**去掉**所有 `8` 公式相关的 prompt 约束、contract 约束和 scoring 维度
- 闭卷条件：与 A 组完全一致
- 输入：与 A 组完全一致
- 唯一差分：prompt 中去掉公式驱动部分，只保留通用写作指令

## 同题同材料同模型同闭卷

| 控制变量 | 要求 |
| --- | --- |
| 题目 | A/B 两组必须做同一题 |
| 材料 | A/B 两组必须使用完全相同的 `inputs/` 与 `raw_source_analysis.md` |
| 模型 | A/B 两组必须使用同一 LLM 模型、同一温度、同一 token 上限 |
| 闭卷条件 | A/B 两组都不看 `gold/`、不看样稿、不联网 |
| 执行身份 | A/B 两组都必须是 `0上下文` 正式轮 |

## A/B prompt 差分原则

1. A 组 prompt = 通用写作指令 + `8` 公式驱动约束
2. B 组 prompt = 通用写作指令（与 A 组相同部分）
3. 差分部分 = 仅 `8` 公式驱动约束
4. 差分模板必须在 `IIE` 冻结，不得在 `IIF` 执行时临时调整
5. 通用写作指令部分 A/B 必须逐字一致

## 最小样本量

- 最小样本量：`3` 题
- 每题产出 A 组稿件 `1` 份 + B 组稿件 `1` 份
- 共计最少 `6` 份稿件进入盲评

## 盲评规则

1. 评审者不知道哪份稿件来自 A 组、哪份来自 B 组
2. 稿件去除所有能暴露组别的标识（文件名、prompt 痕迹、公式引用标记）
3. 评分维度必须包含：
   - 必写事实命中率
   - 结构板块完整性
   - 成稿质量（可读性 + 专业准确性 + 表达张力）
   - 一票否决项检查
4. 评分完成后再揭盲，对比 A/B 差异

## 通过判据

1. A 组在 `3` 题中的平均质量评分显著高于 B 组 → 支持因果命题
2. A 组与 B 组无显著差异 → 不支持因果命题
3. B 组反而高于 A 组 → 反驳因果命题

## 硬边界

1. **不完成 A/B 对照，不得宣称 `8` 公式有效**
2. 不得用 Q1/Q2/Q3/Q4 的历史结果替代 A/B 对照
3. 不得用单次白盒高分替代 A/B 对照
4. 不得在 `IIE` 未完成管线搭建前提前执行 `IIF` A/B 对照
5. A/B 对照的执行必须在 `IIF` 中完成，不得拆散到其他阶段

## 依赖关系

- 前置：`IID` 缺位矩阵已填写 + 失败归因已完成
- 前置：`IIE` 管线搭建已完成 + A/B prompt 差分模板已冻结
- 执行：`IIF` 正式 A/B 对照
- 产出：因果裁决结论 → 决定 `II期` 能否升入 `III期`
