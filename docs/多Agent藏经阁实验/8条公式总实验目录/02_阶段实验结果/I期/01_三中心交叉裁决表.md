## 交叉裁决规则

1. 至少两中心共同稳定出现，且边界清楚的条目，优先冻结。
2. 一家过拆、一家过粗、另一家给出清晰触发/降级边界时，优先冻结有边界的压缩版，扩展项退回候选池。
3. 第 `3/4` 项只允许基于“已经深分析过的文章”，因此优先采用证据链最明确、样本边界最清楚的形态定义。
4. 第 `5` 项必须保持“全量阅读、跨篇归纳、不得逐篇输出”的硬边界；税则过细但留痕不够完整的拆分，不直接冻结成正式类型表。

## 二、第1项裁决：范围内资产归类

### 2.1 共享冻结结果

| 项 | 冻结结果 |
|----|----------|
| 主链直接槽位 | `20` |
| 拆件来源 | `14` |
| 停放观察 | `6` |
| 双入口同构关系 | `2` 组 |
| 人格回填 | `ARCH-02` 单独归入公式 `2` |

### 2.2 当前共享写法

1. `20` 个主链直接槽位：按 `C1` 的去重口径收，不把 `ACT-04/CO-02`、`ACT-05/CO-01` 重复占位。
2. `14` 条组合处理项：只保留为拆件来源，不恢复成单条主规则。
3. `6` 条停放观察：`ARCH-01`、`ARCH-03`、`ARCH-06`、`OBS-01`、`OBS-02`、`OBS-03`。
4. `2` 组双入口同构：`解释桥 = ACT-04 + CO-02`，`比喻承重 = ACT-05 + CO-01`。

### 2.3 裁决理由

1. `C1` 的“单槽 + 双入口留痕”写法，既保留来源，又不重复占位，优于 `C0/C2` 的“去重说明 + 总数易漂移”写法。
2. `ARCH-02` 进入 `人格` 已被三中心共同接受，因此不再保留在观察项余量里。
3. `停放观察` 必须按条目数写实；`ARCH-01/03/06 + OBS-01/02/03` 是 `6` 条，不是 `5` 条。

## 三、第2项裁决：组合规则

### 3.1 条数裁决

| 轴 | C0 | C1 | C2 | 共享冻结版 |
|----|----|----|----|------------|
| 内容单元组合规则 | `7` | `5` | `4` | `5` |
| 逻辑组合规则 | `17` | `5` | `13` | `5` |

### 3.2 内容单元组合规则冻结版

当前正式冻结以下 `5` 条：

1. 事实先于判断，判断必须被购买
2. 判断体量与细节成本同步抬升
3. 边界、失败、风险必须进入正文购买链
4. 抽象机制落地时，解释桥或承重比喻必须贴着对象出场
5. 收尾判断不得高于正文已买到的最大判断

### 3.3 逻辑组合规则冻结版

当前正式冻结以下 `5` 条：

1. 评价尺子或判断坐标必须先立，再进入比较或裁决
2. 主问题单轴时走级联；比较对象并列时走平行或对照推进
3. 对比、反讽、对切必须建立共享参照面后再启动
4. 阳性结果、唯一性、标杆判断必须先做来源还原或适用边界分流
5. 多步逻辑来源只能拆件嵌入主线，不能整包照搬

### 3.4 候选池处理

1. `C0` 的 `CR-05/06/07` 更像第 `1` 项拆件预处理约束，不冻结成第 `2` 项母规则。
2. `C0` 的 `LR-04~17` 与 `C2` 的 `13` 条逻辑候选，保留为模式池 / 候选池，不直接升格为共享母表。
3. `C2` 把 `RUL-03` 兼跨到内容侧的判断保留为边界提醒，但本轮仍按逻辑侧冻结。
