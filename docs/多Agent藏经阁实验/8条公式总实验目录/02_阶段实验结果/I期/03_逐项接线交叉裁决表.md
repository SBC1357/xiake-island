## 一、总体评估

1. 三中心对 `8` 条公式主链、`8` 个最小内容单元、`1` 个正式最小逻辑单元、`5 + 5` 条正式组合规则没有结构性分歧。
2. 当前主要分歧不在“有没有这套主链”，而在“哪些上层资产已经够资格写成 `已闭合`”。
3. `C0` 的长处是证据覆盖广、上层收口积极；弱点是把部分仍缺下层明确接线的资产提前写成 `已闭合`。
4. `C1` 的长处是闭合标准最硬，能把“正式存在”和“真正接全”分开写；本表以 `C1` 作为第三段闭合标准底盘。
5. `C2` 的长处是把逻辑侧薄弱、中间层未压平这两个风险点喊得最响；弱点是把已经能稳定导出的上层资产也一起压回 `待组合`。
6. 当前共享缺口已经被压缩到两类：
   - `人物真相` 线缺稳定最小逻辑单元
   - `决策辅助` 线缺稳定范围收口

## 二、第三段裁决规则

1. 只要有中心明确指出“某上层资产仍缺具体下层组合资产、规则或最小单元”，而另一个中心只是用泛化占位语把它写成闭合，本表一律按 `待组合` 裁决。
2. 上层资产允许正式存在但未闭合；未闭合时继续保留为正式资产，但状态只能写成 `待组合`。
3. `仅候选` 只保留给尚未进入共享母表的最小单元或扩展模式；不把单中心扩展池直接升格成共享正式资产。
4. 候选池只保留两类条目：
   - 直接阻塞当前正式上层资产闭合的条目
   - 至少两中心共同指向、且能解释同一缺口的条目
5. `C2` 的大扩展池与 `C0` 的大模式库继续保留为证据，但不进入当前共享候选池。

## 四、为什么这样裁

1. 不按 `C0` 的“全部上层都已闭合”冻结，是因为 `OUT-03 / E-OUT-03 / RANGE-03` 的下层仍显式依赖 `人物线 / 问题线主从`，这条最小逻辑单元当前还没有进入共享正式表。
2. 不按 `C2` 的“全部上层都待组合”冻结，是因为 `OUT-01 / 02 / 04 / 05` 及其对应的下层组合资产、规则和最小单元，已经在三中心材料里反复出现，且没有新的必需下层缺口。
3. `RANGE-05 / E-OUT-05` 不直接冻成已闭合，不是因为 `OUT-05` 失效，而是因为 `决策辅助范围` 里“结果边界澄清”和“患者选择沟通”这两支还没完成稳定收口。
4. `OUT-05` 继续保留为 `已闭合`，是因为它的内容组合、组合逻辑和最小单元在“结果边界澄清”主路线上已经稳定；当前未收口的是范围层，不是文章大纲层。
5. 当前真正的结构缺口已经不是“规则条数不够”，而是“逻辑侧最小单元过薄、两条上层线还没被更小的东西稳定接住”。

## 五、未冻结项如何进入 II期验证

### 5.1 总原则

1. 不能用单中心自说自话完成验证。
2. 不能只靠扩样，也不能只靠上题；两者各管不同问题。
3. `I期` 到此为止只负责冻结和理清资产；验证职责从本表开始整体转入 `II期`。
4. `II期` 的第一批验证对象，只准从当前共享未冻结项里取，不准把单中心大候选池整包抬进主线。

### 5.2 方法分工

| 方法 | 主要负责验证什么 | 适用对象 |
|------|------------------|----------|
| 扩大样本量 | 候选最小单元 / 候选范围分支是否在新样本中稳定重复出现 | `人物线 / 问题线主从`、`人群分流 / 选择建议落点` |
| 接线题 | 给定上层资产与下层候选，检验是否能稳定写出同一接线关系 | `OUT-03 / E-OUT-03 / RANGE-03`、`RANGE-05 / E-OUT-05` |
| 范围筛选题 | 给同一素材池，检验不同目的 / 人格下是否稳定筛出不同有效内容与逻辑 | `RANGE-05 / E-OUT-05` |
| 大纲重建题 | 给目标、范围和资产池，只要求推出文章大纲 | `OUT-03`、`E-OUT-03`、`E-OUT-05` |

### 5.3 当前共享未冻结项的 II期验证路线

| 未冻结对象 | 当前状态 | 主要风险 | II期验证方法 | 主要材料 | 进入阶段 | 通过标准 |
|------------|----------|----------|--------------|----------|----------|----------|
| `人物线 / 问题线主从` | 最小逻辑单元候选 | 若不稳定，则人物真相线整组都只是伪闭合 | 先扩样，再做接线题与大纲重建题 | 新增至少 `3` 篇 `B类` 人物 / 企业人物 / 人物策略全文样本；不得与已深分析样本重复 | `IIA` | 同一候选逻辑单元能在新增样本中重复出现，并且在同材接线题 / 大纲重建题里，不新增新规则也能稳定导出 `OUT-03` |
