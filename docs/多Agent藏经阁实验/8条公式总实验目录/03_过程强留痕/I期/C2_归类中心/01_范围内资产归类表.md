## 一、归类方法论

### 1.1 C2视角下的归类原则

C2归类中心从"写作规范层"视角出发，采用以下原则：

1. **功能优先**：每条资产首先回答"它约束什么、驱动什么、组合什么"
2. **公式链定位**：明确该资产在8条公式链中的位置（输入侧、输出侧、中间节点）
3. **层级区分**：区分"规范层资产"（归入公式主链）与"表达层资产"（归入L1）与"架构层资产"（停放观察）
4. **去重合并**：同一说法多表出现的，以最早定义为准，后续出现记为别名

### 1.2 归类判断流程

```
第1步：判定是否为架构层容器（旧框架、旧分级体系）→ 是则停放观察
第2步：判定是否为有限组合/旧容器 → 是则拆分子部分再归类
第3步：判定功能归属 → 回答"它约束什么/驱动什么/组合什么"
第4步：定位公式节点 → 输入侧（写作目标/排列规则/最小内容单元/最小逻辑单元/内容单元组合规则/逻辑组合规则/人格）或输出侧（写作目的/写作范围/最小有效内容单元/最小有效逻辑单元/内容组合/组合逻辑/文章大纲/有效大纲）
```

## 六、C2独立判断与发现

### 6.1 内容侧充实，逻辑侧偏薄

- 内容侧（最小内容单元8条 + 内容单元组合规则3条）明显充实于逻辑侧（最小逻辑单元1条）
- 这反映了医学写作实践：内容素材和内容组合规则更容易被显式表述，而逻辑单元和逻辑组合规则常隐含在"稿型"或"拓扑"中
- **后续需要特别关注：是否有遗漏的最小逻辑单元？**

### 6.2 写作目标是整个公式链的最上游缺口

- 当前暂无明确归到"写作目标"输入侧的资产
- 写作目标回答"为什么写这篇文章"
- **第5项（32篇推演）必须优先关注写作目标候选资产的提取**

### 6.3 排列规则 vs 组合规则的层次差异

- 排列规则（ACT-06、ARCH-05）约束的是"素材如何排列"
- 内容单元组合规则（RUL-01/02/06）约束的是"素材如何拼装"
- 排列规则更靠近公式1上游（写作目标内部），组合规则更靠近公式5/6（内容组合内部）
- 这两层规则的区分需要在第2项中进一步阐明

### 6.4 逻辑组合规则候选丰富但分散

- 直接归位3条 + 组合处理拆出10余条候选
- 这些候选散落在不同的旧分类中（CO-07、GS系列、RUL-05拆件等）
- 需要在第2项中统一整理，形成逻辑组合规则的清晰谱系

## 七、归类原则说明

1. `动作`不再自动充当母类。每条动作拆看它实际描述的是"意图"（归写作目的）、"手段"（归最小内容单元）还是"约束"（归规则）
2. `构件`直接回归最小内容单元候选，不再先当框架层对象
3. `规则`一律拆看属于排列规则、内容单元组合规则还是逻辑组合规则，不再混写成一个桶
4. `拓扑`回到逻辑侧（最小逻辑单元或逻辑组合规则），不再单独占旧母位
