## 一、两类组合规则的定义与定位

### 1.1 在8条公式中的位置

```
公式5: 最小有效内容单元 * 内容单元组合规则 = 内容组合
公式6: 最小有效逻辑单元 * 逻辑组合规则 = 组合逻辑
```

**内容单元组合规则**：约束"内容单元如何拼装成内容组合"
**逻辑组合规则**：约束"逻辑单元如何串联成组合逻辑"

### 1.2 两类规则的核心差异

| 维度 | 内容单元组合规则 | 逻辑组合规则 |
|------|-----------------|-------------|
| 约束对象 | 内容单元（事实、判断、数据、解释等） | 逻辑单元（因果、对比、递进、并列等） |
| 约束内容 | 拼装顺序、比例关系、上限约束 | 串联方式、拓扑选型、呼应模式 |
| 粒度 | 内容素材层面 | 文章结构层面 |
| 典型问题 | "事实和判断怎么排？""大判断需要多少细节？" | "级联还是平行？""首尾怎么呼应？" |
| 输出 | 内容组合（一段有逻辑的内容） | 组合逻辑（一个论证结构） |

## 二、内容单元组合规则谱系

### 2.1 直接归位的3条

| 编号 | 规则名 | 规则内容 | C2解读 |
|------|--------|----------|--------|
| RUL-01 | 判断必须由事实购买 | 判断不能凭空出现，必须有事实支撑 | **最核心的内容组合规则**。定义了内容单元的基本拼装顺序：先事实、后判断 |
| RUL-02 | 判断越大细节成本越高 | 判断量级越大，前面需要的细节素材越多 | RUL-01的梯度版。定义了判断与细节的比例关系 |
| RUL-06 | 收尾不能高于证据最大判断 | 收尾段的判断不能超出正文已购买的判断天花板 | 内容组合的上限约束。防止"收尾拔高"失配 |

### 2.2 从组合处理拆出的1条

| 原编号 | 拆出规则 | 规则内容 | C2解读 |
|--------|----------|----------|--------|
| RUL-05① | 失败出场约束 | 失败必须出现在正文中，不能跳过 | 约束"必须包含失败素材"的内容组合要求 |

### 2.3 内容单元组合规则完整谱系

```
【内容单元组合规则谱系】

一、顺序类规则
├── RUL-01 判断必须由事实购买
│   └── 定义：事实 → 判断 的基本顺序
│
└── RUL-03 承重比喻必须前置
    └── 定义：比喻 → 被解释对象 的顺序（注：此条更靠近逻辑组合，但影响内容排列）

二、比例类规则
└── RUL-02 判断越大细节成本越高
    └── 定义：判断量级 ∝ 细节投入

三、上限类规则
└── RUL-06 收尾不能高于证据最大判断
    └── 定义：收尾判断 ≤ 正文已购买的最大判断

四、必含类规则
└── RUL-05① 失败出场约束
    └── 定义：失败素材必须在正文中出现
```

### 2.4 内容单元组合规则的C2判断

1. **顺序类规则**（RUL-01、RUL-03）：约束内容单元的排列顺序
2. **比例类规则**（RUL-02）：约束内容单元的量级关系
3. **上限类规则**（RUL-06）：约束内容组合的判断天花板
4. **必含类规则**（RUL-05①）：约束内容组合必须包含的素材类型

当前内容单元组合规则谱系基本完整，覆盖了顺序、比例、上限、必含四个维度。

1. **时序类规则**（RUL-03）：约束逻辑单元出现的时序

3. **模式类规则**（GS-02、CO-06、GS-03/04/05/06、ACT-03②）：提供组合逻辑的框架模式

当前逻辑组合规则候选丰富，但分散在不同来源中。建议在第5项（32篇推演）中验证这些候选规则的实际使用频率和稳定性。

## 四、两类组合规则的交叉与边界

### 4.1 规则的层级关系

```
【规则层级图】

写作目标（最上游，约束"为什么写"）
    ↓
排列规则（约束"素材如何排列"）
    ↓
内容单元组合规则（约束"内容单元如何拼装"）
    ↓
逻辑组合规则（约束"逻辑单元如何串联"）
    ↓
有效大纲（最下游，约束"文章结构如何呈现"）
```

### 4.2 边界案例分析

| 案例 | 原分类 | C2判断 | 理由 |
|------|--------|--------|------|
| RUL-03 承重比喻必须前置 | 逻辑组合规则 | **兼跨两层** | 时序约束属于逻辑组合，但"比喻"本身是内容单元，因此也影响内容排列 |
| GS-02 反讽式对比 | 旧稿型子型 | **归入逻辑组合规则** | 它提供的是"如何组合多个逻辑单元"的模式，不是内容排列问题 |
| RUL-05 失败必须出场 | 有限组合 | **拆分处理** | ①失败出场（内容必含规则）+ ②力度调节（逻辑调节规则） |

### 4.3 判断规则类型的C2方法

```
【判断规则类型的方法】

问题1：这条规则约束的是"内容素材"还是"逻辑结构"？
├── 内容素材 → 内容单元组合规则
└── 逻辑结构 → 逻辑组合规则

问题2：这条规则是"排列顺序""拼装比例"还是"串联方式"？
├── 排列顺序 → 靠近排列规则/内容单元组合规则
├── 拼装比例 → 内容单元组合规则
└── 串联方式 → 逻辑组合规则

问题3：这条规则是"一条独立规则"还是"一个模式/框架"？
├── 独立规则 → 检查是否可单条测试
└── 模式/框架 → 可能是组合处理项，需拆分
```

## 五、C2独立判断与发现

### 5.1 内容单元组合规则谱系基本完整

- 顺序类、比例类、上限类、必含类四个维度均已覆盖
- 但当前规则主要来自"判断-事实"关系的约束，可能遗漏其他类型的内容组合规则
- **第5项需要观察：是否有关于数据组合、案例组合、解释组合的其他规则？**

### 5.2 逻辑组合规则候选丰富但需验证

- 直接归位3条 + 组合处理拆出10条候选，共13条候选
- 这些候选散落在不同来源（CO系列、GS系列、ACT拆件、RUL拆件）
- 建议在第5项中验证：
  1. 哪些候选规则在32篇中高频出现且稳定？
  2. 哪些候选规则只在特定稿型中出现？
  3. 是否有遗漏的逻辑组合规则？

### 5.3 排列规则与组合规则的层次需进一步澄清

- 排列规则（ACT-06、ARCH-05）约束"素材如何排列"
- 内容单元组合规则约束"内容单元如何拼装"
- 这两层规则的边界在什么位置？需要后续讨论澄清

### 5.4 规则的稳定性与稿型相关性

- 部分逻辑组合规则（如GS-01人物核心轴）明显是稿型专用
- 部分逻辑组合规则（如GS-02反讽式对比）可能跨稿型使用
- 建议在第5项中统计候选规则的出现频率和稿型分布

## 六、对后项的影响

### 6.1 对第3/4项的影响

- 内容单元组合规则和逻辑组合规则是定义"文章大纲"和"有效大纲"的基础
- 公式7：内容组合 * 组合逻辑 = 文章大纲
- 需要先明确这两类规则的谱系，才能定义大纲的形态

### 6.2 对第5项的影响

- 第5项需要特别关注：
  1. 写作目标候选资产
  2. 最小逻辑单元候选资产
  3. 逻辑组合规则候选的验证
