# 三系统能力天花板评估

状态：首版  
日期：2026-04-05  
用途：逐系统盘点能力天花板，给出"能不能冲到高质量成稿"的正式判断

## 核心结论

**三系统各自的天花板都不低，组合天花板足以覆盖高质量成稿。但当前组合接线率约 0%——没有任何一条从 8 公式到 L 层到管线的自动路由。III 期的任务就是搭这套桥。**

---

## 一、8 公式体系天花板

### 1.1 当前已冻结能力

| 能力维度 | 已冻结资产 | 天花板判断 |
| --- | --- | --- |
| 写作目标覆盖 | 8 种目标（OBJ-01~08） | 覆盖了还原、赋予、重建、解码、翻译、转译、照亮 7 大写作意图，已足够支撑医学内容写作的绝大多数任务类型 |
| 排列规则 | 7 种排列（ARR-01~07）+ 3 条映射规律（MAP-01~03） | 逐级翻译、框架先行、层层剥开、旧→新对位、行为反推、数据管道、偏差引入——覆盖了严肃内容写作的主要推进方式 |
| 写作目的 | 7 种目的（PUR-01~07） | 裁决权转移、坐标系重置、决策必然性还原等——每种目的都能对应到具体的文章类型 |
| 写作范围 | 5 种范围（RANGE-01~05），含 RANGE-05 双路 | 评价线、主张线、人物真相线、工程解释线、决策辅助线——覆盖了核心业务场景 |
| 最小内容单元 | 8 个正式单元 | 硬事实、判断、边界声明、数据锤击、权威锚定、解释桥、比喻承重、风险解剖——粒度已细到可操作级别 |
| 最小逻辑单元 | 3 个正式判断（+ 2 个范围专属） | ARCH-04 + 人物线/问题线主从 + 人群分流/选择建议落点 |
| 组合规则 | 5 条内容组合规则 + 5 条逻辑组合规则 | 这是纪律层——约束怎么组合而不是自由发挥 |
| 文章大纲 | 5 种（OUT-01~05） | 尺子前置型、重定义型、人物谜题型、需求解法型、结果前置型 |
| 有效大纲 | 5 种（E-OUT-01~05） | 范围 × 大纲 = 有效大纲，已全部冻结 |

### 1.2 天花板判断

- **结构精度上限：高。** 15 个公式位已全部显式落表，逐项接线已冻结。这是当前系统最严密的结构框架。
- **路由覆盖上限：中高。** 5 种范围覆盖了大部分业务场景，但新场景（如纯数据解读、行业趋势分析）可能需要扩展。
- **成稿质量贡献：约 30%。** 结构只能贡献成稿质量的一部分——骨架对了但肉不好看，外部评审仍会判失败。
- **最大短板：不可执行。** 8 公式当前只存在于人类可读的 markdown 文档中，没有被编码为机器可读的 JSON/config 资产。任何管线都无法直接消费它。

## 三、侠客岛管线天花板

### 3.1 当前六段链能力

| 阶段 | 当前能力 | 上限判断 |
| --- | --- | --- |
| Evidence | 从 L4 查证据事实，支持别名匹配、领域过滤、补充上传 | 已可用，需加 L3 上下文注入 |
| Planning | 靠 play_id + arc_id + thesis 路由，降级策略完备 | 路由精度不足（SPEC 已诊断），需改成 8 公式路由 |
| Writing | 编译 system_prompt + user_prompt，注入证据上下文 | 结构约束是弱提示（SPEC 已诊断），需升级为硬约束 |
| Drafting | 支持 Fake/OpenAI 双模式，字数自适应重试 | 基础能力已足，不需要大改 |
| Quality | 5 道门禁（basic, inference, boundary, length, compliance） | 缺结构门禁（SPEC 已诊断），需加 8 公式验收 |
| Delivery | DOCX + Markdown，字数最终验收 | 已可用 |

### 3.2 AssetBridge 集成

当前 AssetBridge 已经实现了从 `publish/current/consumers/xiakedao/` 读取藏经阁资产的能力，且有严格/降级两种模式。这意味着管线层面的资产消费通道已经存在，不需要从零搭建。

### 3.3 管线天花板判断

- **工程完整度：高。** 六段链 + 审计日志 + 降级机制 + 异步房间 = 生产级架构。
- **写作质量上限：受限于 Planning 精度。** 当前 Planning 仍靠粗粒度 play/arc 路由，无法利用 8 公式的精细结构映射。
- **改造成本：中等。** 不需要重写管线，需要：① 加一级范围检测，② 改写 Planning 输入输出，③ 升级 Writing 的约束注入，④ 升级 Quality 的门禁。

## 五、与写作 DNA 重构 SPEC 的关系

2026-04-02 的写作 DNA 重构 SPEC 已经独立诊断出了几乎相同的问题：

| SPEC 诊断 | 本次审计对应 |
| --- | --- |
| L3 被写成了历史稿型模板层 | L3 需要清模板残留 |
| Planning 路由维度过粗 | Planning 需要改成 8 公式路由 |
| Writing 规则执行是弱提示 | Writing 需要把 8 公式规则升级为硬约束 |
| Quality 拦不住结构失真 | Quality 需要加 8 公式结构门禁 |

**区别在于：** SPEC 当时提出的方案是引入 `task_intents + evidence_archetypes + reasoning_contract_operators` 作为新 L2 构件。而现在 8 公式已经提供了更具体的替代品——5 种范围、5 种内容组合、5 种逻辑组合、5 种文章大纲、5 条内容规则、5 条逻辑规则。

两者不矛盾，可以融合：

- SPEC 的 `task_intents` ↔ 8 公式的 `OBJ-01~08`（写作目标）
- SPEC 的 `evidence_archetypes` ↔ 8 公式当前未显式覆盖，但可作为 L3 层资产接入
- SPEC 的 `reasoning_contract` ↔ 8 公式的 `内容组合规则 + 逻辑组合规则`
- SPEC 的 `coverage_contracts` ↔ 8 公式的 `有效大纲` 展开项
- SPEC 的 `RenderPlan` ↔ 8 公式的 `有效大纲 + L1 风格叠加`

## 六、给信心的部分

1. **结构已经冻结。** 这是历史上第一次有 15/15 显式落表的写作结构框架。之前的系统靠 6 个 play 和 6 步 arc 粗路由，现在有精确到范围 × 单元级别的映射表。
2. **资产已经存在。** L1 有 16 份可用 JSON，L2 有 10 份，L3 和 L4 有结构化目录。不是从零建。
3. **管线已经跑通。** 侠客岛六段链已经可以端到端出 DOCX。改的是路由和约束注入，不是重写管线。
4. **SPEC 已经诊断。** 写作 DNA 重构 SPEC 的诊断与 8 公式体系高度吻合，两条线可以对接而不是互相推翻。
5. **IIC 的失败恰好证明了 8 公式的结构判断力。** 两道题都"接住了路线"——说明公式 1~8 的路由链是成立的。失分全在表达力、策略执行力和领域上下文——这些恰好是 L1~L4 的责任区。
