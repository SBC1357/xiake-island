# III期临床总入口

状态：工作版 / 冷启动入口
日期：2026-04-11
用途：给新 Agent 的零帧接手入口；先认清哪些文件只负责导航，哪些文件才是接手时必须直读的作品与证据

## 接手铁律

1. 入口页不是起手审稿入口。
2. 接手先看最新相关 `单题全貌.md` 的全文、材料全文、prompt 全文，再看对应 runtime 审稿包。
3. 看完作品和 runtime，再回 `03_当前阶段任务卡.md` 确认当前 blocker。
4. `05_试运行结果总览.md`、`07_主执行session裁决结论.md`、`08_回写清单.md` 都是控制件，不是第一读物。

## 这个目录解决什么

1. 让新 Agent 先知道 III 期临床现在有哪些正式载体。
2. 让新 Agent 先知道正式真相源放在哪里，不靠聊天记忆接手。
3. 让新 Agent 先知道哪些内容只是工作版、待补证，不能直接当最终结论。

## 这个目录不解决什么

1. 不直接替代 `02_节点模块规则明细层` 的规则正文。
2. 不直接替代 `01_散落经验裁决层` 的经验裁决。
3. 不直接替代 `03_完整架构层` 的系统总图。
4. 不负责把所有散落经验一次性写完。

## 当前读序

1. 先读上游四份仓库级规则。
2. 再读最新相关 `05_阶段实验包/IIIA/06_单题全貌档案/*.md`，先看生成稿全文、材料全文、prompt 全文。
3. 再读对应 runtime case 目录里的 `generated.txt`、`materials_full.json`、`writing_system_prompt.txt`、`writing_user_prompt.txt`、`review_bundle.json`、`task_detail.json`。
4. 再读 `00_入口索引/03_当前阶段任务卡.md`，确认当前唯一主任务是什么。
5. 如当前任务明确需要低频历史参考，再读 `06_内循环执行资产包/00_说明与分层清单.md` 及对应冻结副本。
6. 只有当这轮确实要继续执行 IIIA，才补读 `05_阶段实验包/IIIA/04_试运行计划.md`。
7. 如当前任务涉及执行、预检、shell、`.env`、`venv`、评分链或后端链，再读 `04_III期总实验方案层/04_III期执行环境契约.md`。
8. 最后才回看本入口页、资产地图、节点规则、经验裁决和系统总图。

## 冷启动入口边界

`00_入口索引` 只负责：

1. 导航
2. 背景
3. 真相源分工说明

`00_入口索引` 不负责：

1. 播报当前阶段
2. 播报当前 blocker
3. 播报当前下一步
4. 播报当前分数或当前 smoke 结果
5. 代替单题全貌和 runtime 审稿包

这些活状态统一回到：

1. `00_入口索引/03_当前阶段任务卡.md`
2. 活跃阶段自己的 `05_阶段实验包/.../04_试运行计划.md`
3. 对应 runtime 目录下的直接验证锚点
4. 如问题属于执行环境边界，再回 `04_III期总实验方案层/04_III期执行环境契约.md`

人工审稿统一先回到：

1. `05_阶段实验包/IIIA/06_单题全貌档案/`
2. 对应 runtime case 目录里的审稿包产物

## 工作版 / 待补证

1. `02_节点模块规则明细层` 里的标准件大多是工作版标准，能执行，但仍允许后续补证收紧。
2. `01_散落经验裁决层` 里仍有待验证经验，不要把待验证内容当正式定论。
3. `03_完整架构层` 是总图，不要把总图里的概览句当成节点级标准件。
4. 过程材料和 runtime 结果只算证据，不算最终正式口径。
5. `00_入口索引/03_当前阶段任务卡.md` 只负责说"当前先做什么"，不替代节点正文。
6. `00_入口索引/04_资产与真相源地图.md` 只负责指路，不替代外部真相源本体。
7. 结果总览、裁决结论、回写清单只负责控制和汇报，不得抢单题全貌与 runtime 的主舞台。

## 读到冲突时怎么做

1. 先回查 `02_节点模块规则明细层` 里的对应节点正文。
2. 再回查 `01_散落经验裁决层` 的裁决总表，看这条经验是否已被收紧、合并或保留。
3. 再回查 `03_完整架构层`，只确认系统级关系，不拿它替代节点标准。
4. 如果仍然对不上，停下，回查上游四份仓库级规则和 `D:\汇度编辑部1\藏经阁` 里的现有口径，不要自己补新口径。
