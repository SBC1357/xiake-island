# III期临床冷启动阅读顺序

状态：收缩版 / 冷启动读序
日期：2026-04-10
用途：让新 session / 新 Agent 先看到作品、真实喂料和 runtime，再决定要不要继续执行；不再鼓励先掉进治理目录和完成感文件

## 冷启动硬顺序

### 第 1 步：先读仓库级四份规则

1. `仓库卫生最高禁令.md`
2. `防钻空子.md`
3. `一级事故补充案例.md`
4. `防爆指令.md`

没有前置阅读回执，后续执行、汇报和完成声明一律按未读处理。

### 第 2 步：先看最新相关单题全貌

优先读：

1. `05_阶段实验包/IIIA/06_单题全貌档案/`
2. 与当前 case 最相关的那一份 `单题全貌.md`

读这一步时，顺序固定：

1. 先看生成稿全文
2. 再看材料全文
3. 再看写作 prompt 全文
4. 再看本题真实错点和局部判断

如果还没读到全文、材料、prompt，就不算真正进入任务。

### 第 3 步：再看同题 runtime 审稿包

对应 case 的 runtime 目录里，先看下面这些一等产物：

1. `generated.txt`
2. `materials_full.json`
3. `writing_system_prompt.txt`
4. `writing_user_prompt.txt`
5. `review_bundle.json`
6. `task_detail.json`
7. `score.json` 或 `summary.json`

目的只有一个：确认这篇稿子为什么会写成这样，不靠摘要猜。

### 第 4 步：再看当前阶段任务卡

只在这里确认三件事：

1. 当前阶段
2. 当前 blocker
3. 当前下一步

当前活状态只认：

1. `00_入口索引/03_当前阶段任务卡.md`

### 第 5 步：如果这轮要继续执行 IIIA，再读执行壳

只在这时补读：

1. `05_阶段实验包/IIIA/04_试运行计划.md`
2. `scripts/iiia/iiia_preflight.ps1`

这里解决的是入口、预检、输出根和有效 run 判定，不负责定义当前主任务。

### 第 6 步：只有遇到环境问题，再读环境契约

只在遇到 shell / `.env` / `venv` / 评分链 / 后端链问题时补读：

1. `04_III期总实验方案层/04_III期执行环境契约.md`

### 第 7 步：最后才回入口、地图、规则、经验、总图

只有在前 6 步都完成后，才允许回看：

1. `00_III期临床总入口.md`
2. `04_资产与真相源地图.md`
3. `02_节点模块规则明细层`
4. `01_散落经验裁决层`
5. `03_完整架构层`

这些文件负责导航、规则和系统解释，不是起手审稿入口。

## 禁止的起手动作

1. 不要先读 `00_阶段入口.md`、`01_阶段目标.md`、`05_试运行结果总览.md`、`07_主执行session裁决结论.md`、`08_回写清单.md`。
2. 不要还没看到全文和 runtime，就先写总结、裁决或汇报。
3. 不要用结果总览、裁决快照、回写清单代替单题全貌和 runtime。
4. 不要先看总图、入口、方法论，再倒回来猜作品问题。

## 最小停下规则

遇到下面任一情况，先停下，不要继续往下猜：

1. 单题全貌里没有全文、材料或 prompt。
2. runtime 目录里缺 `generated.txt`、`task_detail.json`、`review_bundle.json` 中的关键产物。
3. 文档口径和 runtime 直证冲突。
4. 你已经开始依赖摘要或裁决，而不是全文和原始产物。

停下后先回查：

1. 对应 `单题全貌.md`
2. 对应 runtime case 目录
3. `00_入口索引/03_当前阶段任务卡.md`
4. 如问题属于执行环境，再看 `04_III期总实验方案层/04_III期执行环境契约.md`
