# III期临床冷启动阅读顺序

状态：冻结回写版 / 冷启动读序
日期：2026-04-15
用途：让新 session / 新 Agent 先看到当前活状态、当前施工真相源和真实验证锚点，再决定要不要继续执行；不再鼓励从旧 `IIIA` 包、历史方案壳或治理目录起手。

## 冷启动硬顺序

### 第 1 步：先读仓库级四份规则

1. `仓库卫生最高禁令.md`
2. `防钻空子.md`
3. `一级事故补充案例.md`
4. `防爆指令.md`

没有前置阅读回执，后续执行、汇报和完成声明一律按未读处理。

### 第 2 步：先看当前活状态卡

只在这里确认三件事：

1. 当前阶段
2. 当前阻塞
3. 当前下一步

当前活状态只认：

1. [03_当前阶段任务卡.md](./03_当前阶段任务卡.md)

### 第 3 步：再读当前施工真相源

固定先读：

1. [白盒系统第四轮收口长线程包.md](../07_白盒施工包/04_第四轮白盒测试+建设/白盒系统第四轮收口长线程包.md)
2. [架构双图_真实_vs_理想.md](../07_白盒施工包/04_第四轮白盒测试+建设/架构双图_真实_vs_理想.md)
3. [全貌.md](../07_白盒施工包/04_第四轮白盒测试+建设/全貌.md)

这一步的目的只有一个：先认清当前执行面、当前架构面和当前 run 真相源，不靠聊天猜。

### 第 4 步：如果涉及 Persona / prompt stack，再补读 Persona 适配裁决

需要时再读：

1. [03_Persona注入适配分析.md](../01_散落经验裁决层/03_Persona注入适配分析.md)

### 第 5 步：如果这轮要继续施工，再读当前代码与协议工作面

只在这时补读：

1. 长线程包里列出的 `run_whitebox_*`、`whitebox_common.ps1`、`node_io_protocol.json`、`whitebox_llm_profiles.json`
2. 当前要动到的 prompt 文件

这里解决的是“怎么改当前白盒系统”，不是“回顾旧 IIIA 是怎么跑的”。

### 第 6 步：只有需要历史背景时，才回看旧层

只在需要历史背景、旧方法来源或长期规则上下文时补读：

1. `01_散落经验裁决层`
2. `02_节点模块规则明细层`
3. `03_完整架构层`
4. `04_III期总实验方案层`
5. 旧 `IIIA` 执行包与历史阶段包

这些层负责背景、方法、长期规则和历史记录，不再是当前活入口。

## 禁止的起手动作

1. 不要先读旧 `IIIA` 执行包、旧阶段实验包或 `04_III期总实验方案层`，再倒回来猜当前任务。
2. 不要还没读 `03` 与 `07/04` 真相源，就先写总结、裁决或回写。
3. 不要用 runtime 原始目录代替 [白盒系统第四轮收口长线程包.md](../07_白盒施工包/04_第四轮白盒测试+建设/白盒系统第四轮收口长线程包.md) 的集中结论。
4. 不要让入口页、资产地图或历史总图并行承载当前 blocker 和当前下一步。

## 最小停下规则

遇到下面任一情况，先停下，不要继续往下猜：

1. `03_当前阶段任务卡.md` 和 `07/04` 真相源说法对不上。
2. 长线程包引用的关键锚点在 runtime 中缺失。
3. 文档口径和 runtime 直证冲突。
4. 你已经开始依赖摘要，而不是当前真相源和直接产物。

停下后先回查：

1. [03_当前阶段任务卡.md](./03_当前阶段任务卡.md)
2. [白盒系统第四轮收口长线程包.md](../07_白盒施工包/04_第四轮白盒测试+建设/白盒系统第四轮收口长线程包.md)
3. [全貌.md](../07_白盒施工包/04_第四轮白盒测试+建设/全貌.md)
4. 对应 runtime 直接验证锚点
