# III 期系统 Persona 注入适配性分析

## 核心判断框架

Persona 能增强的是**需要"认知深度"的节点**——即模型需要"钻进文本/证据里看"的环节。
Persona 帮不了的是**需要"流程执行"的节点**——即主要靠字段检查、状态机流转、格式校验的环节。

判断标准：**这个节点的核心动作是"理解判断"还是"规则执行"？**

---

## 逐节点分析

### 强烈推荐注入 Persona 的节点

| 节点 | 当前核心动作 | 推荐 Persona 类型 | 为什么 |
|---|---|---|---|
| **03A 主题探索与扩证层** | 形成候选主题、组织逻辑链/故事链、给推荐主线与备选主线 | **调查记者 / 选题编辑** | 本质是"从一堆材料里找到最值得讲的故事"。高度依赖认知深度，模型需要理解证据间因果、识别信息差和悖论、判断叙事张力。Persona 可显著提升"看见更好主线"的能力 |
| **06 审稿与门禁层** | 逐段逐句诊断问题 | **资深编辑** | 已验证。Persona 能给出字句级修改意见，检出率远超纯 Protocol |
| **05 首稿生成层** | 把过闸大纲展开成首稿 | **目标稿型的标杆作者**（动态切换） | 最依赖"写作能力"的节点。Persona 应根据稿型路由动态绑定 |
| **07 返修 whole revise** | 整稿重构 | **继承首稿 Persona** | 本质是"带着问题单重写"，认知需求同首稿 |

### 可选注入、需实验验证的节点

| 节点 | 当前核心动作 | 可选 Persona | 风险 |
|---|---|---|---|
| **04 大纲过闸（构思段）** | 锁故事线、锁逻辑大纲 | **内容架构师** | 好大纲需要叙事直觉（persona 帮），合同编译是纯字段填写（persona 无用）。**只在构思阶段注入，编译阶段不注入** |
| **03B 正式主题确认** | 判断候选主题是否成立 | **学术期刊主编** | persona 可能干扰 `theme_lock_policy` 三选一的机械判断。**只在成文闭环判断时注入** |

### 不推荐注入 Persona 的节点

| 节点 | 核心动作 | 为什么不适合 |
|---|---|---|
| **00 需求对齐回执** | 锁场景、反向确认、填 10 字段 | Persona 会干扰反向确认的中立性——编辑 persona 倾向于"我理解了"而不是"我先问清楚" |
| **01 输入标准化冻结** | 不丢失 + 降噪 + 不多不少 | 纯信息管道。Persona 可能引入主观降噪偏好 |
| **02 稿型识别路由** | 场景 → 稿型 → 样稿母本 | 分类任务，需中立。Persona 可能偏好某种稿型 |
| **03 八公式资产编排** | 公式位映射、资产选择 | 纯规则匹配 |
| **07 返修 local revise** | 1 句 / 1 信息点 / 1 空缺位 | 核心是 execution_precision，Persona 会导致越界改动 |
| **08 模块层** | 模块挂载、边界检查 | 纯约束执行 |
| **09 经验回流** | 来源 → 抽象 → 裁决 | 元认知任务，需中立裁决，Persona 引入立场偏好 |

## Persona 注入的架构建议

### 1. 两阶段模式（已验证有效）

所有注入 Persona 的节点都应采用：

```
第一阶段：Persona 驱动自由诊断/思考（无格式约束）
第二阶段：Protocol 驱动结构化输出（按标准件格式）
```

### 2. 首稿动态 Persona 绑定

根据 `02 稿型路由` 输出动态切换：

| Route | Persona 认知框架 |
|---|---|
| A 研究解读 | 资深医学编辑，擅长把单个临床研究拆解成清晰的故事 |
| B 患者沟通 | 健康传播专家，擅长把复杂医学概念翻译成患者语言 |
| C 机制解读 | 科学作家，擅长用因果链条讲清楚分子机制 |
| D 产业叙事 | 医药行业资深记者，擅长串联多个研究做趋势判断 |

### 3. Persona 不是人名，是认知框架

A/B 实验教训：
- 人名 → 模型模仿**语域**（白话漂移）
- 认知框架 → 模型采用**思维方式**

```
差：你是叶圣陶
好：你是一位资深编辑，审稿时先看全篇站不站得住，再逐段看证据接不接得上
```

### 4. Persona 与 Protocol 的冲突防护

> [!WARNING]
> Persona 和 Protocol 在同一阶段出现时，Protocol 会吞掉 Persona。
> 必须**物理隔离**——先用 Persona 做诊断，再用 Protocol 做格式化。
