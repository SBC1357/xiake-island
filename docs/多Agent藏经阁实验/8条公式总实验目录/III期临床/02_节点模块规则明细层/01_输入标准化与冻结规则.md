# III期输入标准化与冻结规则

状态：整合稿 / 待继续补证  
日期：2026-04-11  
用途：收束 III 期最上游输入标准化与冻结口径，回答“材料怎么进系统，什么算冻结完成”  
边界：本文件只写输入层规则，不扩写稿型、八公式、返修 loop

## 证据摘要

1. 已直接核对接口路由文档、冷启动配置和稿型总表，确认当前生产接入默认就是“对话框直投 + 系统内部路由”，不要求用户先做中间整理件。
2. 已直接核对近期逐字稿，确认输入标准化的目标已被收紧为：`不丢失 + 降噪 + 不多不少`。
3. 已直接核对冷启动配置与实战经验，确认样稿或目标写法没全文直读前，不得宣称“已对齐”。
4. 已直接核对本轮白盒上游状态机裁定，确认输入冻结层应只冻结“可稳定复用的输入与探索前置条件”，不应提前编译正式写作合同。

## 一、输入标准化的目标

III 期输入层的职责不是“帮用户整理材料”，而是把散乱输入压成下游可稳定消费的正式输入包。

当前工作版先按下面三项目标执行：

1. `不丢失`
2. `降噪`
3. `不多不少`

这里必须明确：

1. “不丢失”不是全量抄录，而是主问题、主证据链、禁区、纠偏记录不能丢。
2. “降噪”不是删掉上下文，而是剔除不服务当前主任务的噪声。
3. “不多不少”是指只把真正会影响下游写作和判断的材料带下去。

## 二、默认输入渠道

用户默认只通过下面这些入口喂系统：

1. 对话框文字
2. 文件路径
3. 附件
4. 样稿
5. 修改意见
6. 图表 / 配图

硬规则：

1. 不要求用户先建目录。
2. 不要求用户先做摘要卡、G0 卡、事实表、写作合同。
3. 原始材料之间没有明显冲突时，不得把预处理工作退回给用户。

## 三、输入层必须抽出来的东西

在进入写作主链前，输入层至少要抽出下面 `6` 类：

1. 真实任务描述
2. 原始事实源
3. 样稿与参考件
4. 用户明确纠偏
5. 必写点
6. 禁区

## 四、冻结层的正式输出

当前先按现有口径收束为 `A-E` 五个冻结结果：

1. `客户 brief 冻结`
2. `原始事实锚点`
3. `参考样稿冻结`
4. `禁区与必须点`
5. `主题探索前置卡`（替代旧 `写作合同冻结卡`）

这 `5` 个结果一旦冻结，下游首稿、门禁和返修默认都以它们为准。

这里收紧一条：

1. 输入冻结层可以产出探索前置卡。
2. 输入冻结层不得提前产出正式写作合同。
3. 正式写作合同只能在正式主题确认后，由 `04` 层编译。

## 五、输入层最容易犯的错

1. 没有不丢失，直接漏掉用户纠偏。
2. 没有降噪，把所有附件一股脑丢给下游。
3. 没有“不多不少”，把无关背景也冻结进去。
4. 还没做需求对齐，就直接进入冻结。

## 六、样稿与目标写法的冻结纪律

1. 样稿或目标写法没全文直读前，不得宣称“已对齐”。
2. 没全文时，只能写成“当前借用了哪些表层特征”，不能冒充已经吃透结构。
3. 样稿一旦补齐全文，应优先回到样稿本身再做冻结，不继续沿用旧猜测。

## 七、与下游的关系

输入标准化层只回答：`材料怎么进入系统。`

它不直接回答：

1. 稿型怎么判
2. 八公式怎么用
3. 首稿怎么写
4. 返修怎么 loop

这些都在后续节点处理。

## 八、标准件

这一节包含已升级为工作版的正式标准件，以及仍在预留位的标准件。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。

### 标准件 S-01-01：冻结完成标准

| 字段 | 内容 |
| --- | --- |
| **标准件名称** | 冻结完成标准 |
| **所属节点** | 输入标准化与冻结层 |
| **要解决什么问题** | 明确"什么算冻结完成、可以放行到下游"，防止材料没冻全就直接进入稿型路由和写作 |
| **固定起点** | 需求对齐回执卡已通过（S-00-01 通过） |
| **固定终点** | `A-E` 五个冻结结果全部产出，且逐件点检通过 |
| **输入** | 需求对齐回执卡 + 用户原始材料（对话框文字 / 文件 / 附件 / 样稿 / 修改意见） |
| **输出** | 冻结包，包含以下 `5` 件，每件必须有实质内容（不允许写"暂无"或留空）：**A.** 客户 brief 冻结（含真实任务描述、交付物类型、受众/平台/篇幅）**B.** 原始事实锚点（主问题源、主证据链源，每条可回指到原始材料具体位置）**C.** 参考样稿冻结（主样稿母本路径 + 直读状态标注：已全文直读 / 仅借用表层特征）**D.** 禁区与必写点（逐条列出，每条标注来源：用户明确说 / 回执推导）**E.** 主题探索前置卡（真实写作场景 + 一级稿型 + 主二级稿型 + 当前篇幅级别 + 当前证据状态 + 下一轮最小扩证方向） |
| **通过标准** | 同时满足以下全部：(1) `A-E` 五件无一缺失或留空；(2) B 中每条事实锚点可回指到原始材料（写明出处路径或对话位置）；(3) C 中样稿直读状态标注与实际一致（已读标"已全文直读"，未读标"仅借用表层特征"）；(4) `A-E` 五件之间无内部矛盾；(5) E 只冻结到探索前置条件，不冒充正式写作合同；(6) 冻结包生成后，输入层不再接受新材料，除非用户主动追加并触发重冻结 |
| **不通过标准** | 命中以下任一即不通过：(1) 五件中任一缺失或留空；(2) B 中事实锚点无法回指原始材料；(3) C 中样稿标注与实际不一致；(4) 五件之间存在矛盾；(5) 需求对齐回执卡（S-00-01）未通过就直接进入本层 |
| **失败后回流到哪一层** | 缺原始材料 → 回需求对齐回执层，发最小阻塞说明；五件间矛盾 → 留在本层修正矛盾项后重新点检；需求理解本身有误 → 回需求对齐回执层 |
| **自由度压缩规则** | (1) `A-E` 五件逐件点检，不允许打包宣称"冻结完成"；(2) B 的事实锚点必须写到具体材料路径或对话位置，不允许写"来自用户提供的材料"；(3) D 的禁区和必写点逐条列，每条必须标注来源，不允许笼统写"注意事项若干"；(4) 冻结后不允许静默追加材料，追加必须走重冻结 |
| **是否可自动化** | 半自动 |
| **自动化信号或判定来源** | 可自动化：五件齐全度检查（非空检查）、B 中锚点是否含路径或位置标记、C 中直读状态标注是否存在。人工：五件间矛盾检查、事实锚点回指正确性验证 |
| **正式来源** | (1) 本文件第四节"冻结层的正式输出"；(2) 本文件第六节样稿冻结纪律；(3) 经验裁决总表 E-01（输入标准化目标不丢失+降噪+不多不少）；(4) 经验裁决总表 E-22（样稿未直读不宣称对齐）；(5) 经验裁决总表 E-02（不要求用户做中间整理件） |
| **当前状态** | **工作版** |
| **边界备注** | 本标准只管冻结包的结构完整性和内部一致性。不管冻结内容的业务正确性（业务正确性由门禁和返修兜底）。E 项当前只冻结“主题探索前置条件”，不再提前替代正式写作合同；完整写作合同必须在正式主题确认后由 `04` 层编译。 |
