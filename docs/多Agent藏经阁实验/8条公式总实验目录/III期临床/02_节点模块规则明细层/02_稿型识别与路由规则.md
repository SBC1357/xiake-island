# III期稿型识别与路由规则

状态：整合稿 / 待继续补证  
日期：2026-04-11  
用途：收束 III 期上游路由规则，回答“用户给真实场景后，系统怎么锁定稿型和样稿母本”  
边界：本文件只写路由，不展开八公式和模块层微操

## 证据摘要

1. 已直接核对稿型总表、冷启动配置和 III 期接口路由文档，确认当前稳定路由链已形成：`真实写作场景 -> 一级稿型 -> 主二级稿型 -> 主样稿母本`。
2. 已直接核对现有 III 期母图，确认需求对齐回执层是整个系统的高杠杆闸门。
3. 已直接核对本轮 IIIA 样稿 / prompt / 成稿偏航事实，确认稿型路由不能再直接越权决定正式主题、must-cover 和正文主轴。

## 一、路由链

用户不需要按系统分类说话。  
系统内部必须自己完成下面这条路由：

`真实写作场景 -> 一级稿型 -> 主二级稿型 -> 主样稿母本`

这条链的作用：

1. 真实写作场景负责锁任务入口
2. 一级稿型负责稳大方向
3. 主二级稿型负责锁具体写法
4. 主样稿母本负责提供结构与节奏参照

这里再收紧两条：

1. 稿型决定的是“做什么菜”，不等于正式主题已经成立。
2. 主样稿母本只表示“当前最该学哪种写法”，不等于正文可以直接照着样稿的主题和结论写。

这里再补两条硬纪律：

1. 先判稿型，不判不写。
2. 很多错误不是句子差，而是不同稿型被混着写了。

## 二、回执层输出

在任何冻结和写稿动作前，回执层最少要给出：

1. 真实写作场景
2. 一级稿型
3. 主二级稿型
4. 主样稿母本
5. 交付物
6. 受众 / 平台 / 篇幅
7. 主角度 / 主问题
8. 主证据链
9. 必写点 / 禁区

## 三、切换规则

1. 一级稿型只在大方向明显错了时才允许重切。
2. 主二级稿型装不下主任务时，才允许切换。
3. 一旦切到新的主二级稿型，必须同步重绑主样稿母本。
4. 如果主二级稿型还不够稳，允许短暂保留：
   - `1` 个主二级稿型
   - `1` 个同支候选
   - `1` 个旁支候选
5. 用户纠偏后，只能保留 `1` 个主二级稿型继续往下走。

## 四、最小 blocker 规则

下面这些情况不得直接放行：

1. 真实写作场景还在多个入口之间摇摆。
2. 一级稿型虽然锁了，但主二级稿型仍明显摇摆。
3. 交付物还没分清是新写、改稿还是整理。
4. 用户已经纠偏过，但系统理解又滑回旧路线。

## 五、路由层最容易犯的错

1. 让用户学系统内部稿型分类。
2. 还没锁场景，就乱借样稿。
3. 自造新稿型。
4. 把改稿任务误当成新写稿。
5. 竞争性医学评论本来该靠事实悖论和硬数据立批判力，却误走成修辞猛冲。

## 六、与下游的关系

本层输出一旦成立，下游默认按此执行：

1. 输入冻结层按这条路由冻结材料
2. 八公式与资产编排按此路由选规则和样稿
3. 路由结果先进入 `主题探索与扩证层`
4. 只有正式主题确认后，才允许进入大纲和写作合同

## 七、稿型 × 篇幅级别联动规则

当前工作版先收紧为：

1. 闭环判断不是只看字数，也不是只看主题，而是 `稿型要求 × 篇幅级别`。
2. 每种稿型都有自己的自然篇幅跨度。
3. 当目标字数明显超出当前稿型自然跨度时，不得默认继续在原稿型里扩写，而要回到任务定界层重判稿型。
4. 主样稿母本绑定后，还必须进一步编译成 `样稿合同`，明确：
   - 学什么
   - 不学什么
   - 借结构还是借语气
   - 哪些结论不得照搬
5. 路由层只负责给出“当前更像哪一路”，不直接越权决定正式主题、must-cover 和正文篇幅重心。

### 8.2 路由入口条件

| 判定信号 | 走向 |
| --- | --- |
| 输入为1-2个临床研究数据，任务是拆解数据 | Route-A |
| 输入面向患者/家属，口语化表达，科普安全性/疗效 | Route-B |
| 输入核心是分子/通路/结构层面的因果解释 | Route-C |
| 输入串联3+研究/事件做趋势判断，或企业管线/产业格局分析 | Route-D |

### 8.3 稳定判定点

以下情况路由高置信（≥90%）：
1. 用户明确说"解读某研究" + 给了该研究PDF → Route-A
2. 用户明确说"患者向/科普" + 给了通俗化要求 → Route-B
3. 用户明确说"讲清楚机制/原理" + 给了机制相关文献 → Route-C
4. 用户给了企业管线/多研究综合需求 + 要求格局分析 → Route-D

### 8.4 摇摆点与边界规则

10题+20样稿共识别3种摇摆边界：

**A↔C 边界（研究解读 vs 机制解读）**
- 若主线论证依赖特定研究的临床数据（OS/PFS/ORR）→ Route-A
- 若主线论证依赖分子/通路/结构层面的因果解释 → Route-C
- 两者并重：机制 > 50%篇幅 → Route-C；数据 > 50%篇幅 → Route-A

**A↔D 边界（研究解读 vs 领域叙事）**
- 聚焦1-2个研究的数据拆解 → Route-A
- 串联3+研究/事件做趋势判断或格局分析 → Route-D
- 核心结论关于"这个研究说明了什么" → Route-A；关于"这个领域走向何方" → Route-D

**D内新闻体变体**（来自T07）
- "新闻报道" 写法在稿型总表无独立二级稿型
- 暂归入 Route-D "领域新进展解读"，但 Register 从 R3 调降到 R2，结构从"分析论证"改为"事实先行"

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
