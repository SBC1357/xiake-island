## 一、本层解决什么问题

本层只解决下面四件事：

1. 候选主题怎么合法存在
2. 推荐主线和备选主线怎么形成
3. 扩证什么时候继续、什么时候停止
4. 什么情况下才允许送去正式主题确认

## 二、本层的输入

进入本层前，上游至少要给出：

1. 已锁定的真实写作场景
2. 已锁定的一级稿型 / 主二级稿型
3. 主样稿母本与样稿借用边界
4. 输入冻结层产出的 `主题探索前置卡`

没这四件，不得直接进入本层。

## 三、本层的合法输出

本层只允许两类输出：

1. `候选主题包`
   - 当前推荐主线
   - 当前备选主线（可空，但优先保留）
   - 关键支撑点
   - 当前大致结论 / 落点
   - 当前缺口
   - 下一轮最小扩证方向
2. `扩证未完成说明`
   - 当前为什么还不能成链
   - 还缺什么证据
   - 下一步最小补证动作是什么

## 四、成链的最小门槛

一条候选主线要送去正式主题确认，至少要同时具备：

1. 一条明确主线
2. 至少一组关键支撑点
3. 一个大致可成立的结论 / 落点
4. 能说清当前为什么是“推荐主线”，而不是随机挑中的方向

如果同一题同时长出多条可讲通的链：

1. 必须给 `推荐主线`
2. 同时保留 `备选主线`
3. 不得在本层直接拍死唯一正式主题

## 五、扩证继续 / 停止规则

1. 如果当前连一条成链主线都没有，继续扩证。
2. 如果已经有一条成链主线，允许再做一轮有限扩证，看有没有更优主线或备选主线。
3. 只要已经有：
   - 推荐主线
   - 关键支撑点
   - 大致结论 / 落点
   就应停止本层无限扩证，送去正式主题确认。
4. 本层不得以“也许还能再找到更好的主线”为由无限停留。

## 六、与上下游的边界

### 1. 对上游

本层不重判：

1. 真实写作场景
2. 一级稿型
3. 主二级稿型
4. 主样稿母本

### 2. 对下游

本层不得直接产出：

1. 正式主题卡
2. `theme_lock_policy`
3. 正式写作合同
4. 最终 must-cover facts

这些都属于下游层。

## 七、最容易犯的错

1. 稿型刚锁就把候选主题当正式主题。
2. 只有一条模糊方向，没有关键支撑点，就自称“已成链”。
3. 发现两条都说得通的主线，却不保留备选主线。
4. 无限扩证，不肯进入正式主题确认。

## 八、标准件

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。

### 标准件 S-03A-01：候选主题成链标准

| 字段 | 内容 |
| --- | --- |
| **标准件名称** | 候选主题成链标准 |
| **所属节点** | 主题探索与扩证层 |
| **要解决什么问题** | 防止系统在没有成链时提前锁正式主题，也防止系统在已有成链主线后无限扩证不收口 |
| **固定起点** | 稿型识别与资产路由已通过，且输入冻结层已产出 `主题探索前置卡` |
| **固定终点** | 进入下面两种结果之一：**A. 成链可送确认**（产出 `候选主题包`）或 **B. 扩证未完成**（产出缺口与下一轮最小扩证动作） |
| **输入** | 路由结果 + 样稿借用边界 + `主题探索前置卡` + 当前可用主证据链 |
| **输出** | **A. 候选主题包**：至少包含① 推荐主线 ② 备选主线（可空，但需显式说明是否不存在） ③ 关键支撑点 ④ 大致结论 / 落点 ⑤ 当前缺口 ⑥ 下一轮最小扩证方向；**B. 扩证未完成说明**：至少包含① 当前未成链原因 ② 还缺什么证据 ③ 下一步最小补证动作 |
| **通过标准** | 命中下面任一即可通过本层放行：(1) `候选主题包` 中 ①③④⑥ 全部非空，且推荐主线能解释为什么优于当前备选；(2) 输出 `扩证未完成说明`，且三项都非空，没有伪装成“已成链” |
| **不通过标准** | 命中以下任一即不通过：(1) 只有模糊主题，没有关键支撑点；(2) 推荐主线存在，但没有结论 / 落点；(3) 已有两条可讲通主线，却不保留备选；(4) 说“还要扩证”但不给最小补证动作；(5) 直接把候选主题写成正式主题卡 |
| **失败后回流到哪一层** | 缺主证据或样稿借用边界不清 → 回输入冻结层或稿型路由层；只是主线未成链 → 留在本层继续扩证 |
| **自由度压缩规则** | (1) 本层必须显式区分“推荐主线”和“备选主线”；(2) 推荐主线最少要有一组关键支撑点，不允许只给标题级方向；(3) 扩证未完成时必须写“还缺什么 + 下一步最小补证动作”，不允许只写“继续看材料” |
| **是否可自动化** | 半自动 |
| **自动化信号或判定来源** | 可自动化：字段非空检查、是否同时输出推荐/备选标签、是否写明最小补证动作。人工：主线是否真的成链、推荐/备选的强弱判断是否合理 |
| **正式来源** | (1) `03_完整架构层/01_III期完整架构.md` 中“主题探索与扩证层”职责；(2) `03_完整架构层/02_关键节点路由表.md` 中本层的输入 / 输出 / 回流定义；(3) 本文件第四节“成链的最小门槛”；(4) 本文件第五节“扩证继续 / 停止规则” |
| **当前状态** | **工作版** |
| **边界备注** | 本标准只回答“候选主题是否已成链”，不回答正式主题是否已经成立；正式主题成立要到下一层确认。 |
