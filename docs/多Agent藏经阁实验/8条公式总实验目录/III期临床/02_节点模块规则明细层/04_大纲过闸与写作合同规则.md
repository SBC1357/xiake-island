# III期大纲过闸与写作合同规则

状态：整合稿 / 待继续补证  
日期：2026-04-11  
用途：收束“大纲是独立过闸点”和“写作合同怎么承重”的规则  
边界：本文件不展开句子级写法，只回答大纲何时成立、合同至少包含什么

## 证据摘要

1. 已直接核对近期逐字稿，确认大纲不应再被当成附属产物，而应成为独立确认点。
2. 已直接核对现有 III 期母图和模块补丁文档，确认写作合同层当前已经承担结构、篇幅、顺序、禁止动作等正向编译任务。
3. 已直接核对实战经验归档，确认大量高价值约束真正落点在写作合同，不在句子级临场发挥。
4. 已直接核对本轮白盒状态机裁定，确认正式写作合同前必须先出现：`正式主题卡`、`theme_lock_policy` 与 `样稿合同`。

## 一、大纲是独立过闸点

当前最小规则：

1. 大纲不是首稿前的附属物，而是独立过闸点。
2. 大纲要先成型，再快速确认。
3. 没有过闸的大纲，不该直接进入首稿生成。

## 二、大纲层负责什么

1. 锁故事线
2. 锁逻辑大纲
3. 锁主承重段
4. 锁各段任务分配

## 三、正式主题确认是本层前置

进入本层前，上游必须先给出：

1. 正式主题卡
2. `theme_lock_policy`
3. 样稿合同

没这三件，不得直接编译正式写作合同。

## 四、写作合同至少包含什么

1. 正式主题卡
2. `theme_lock_policy`
3. 真实写作场景
4. 一级稿型 / 主二级稿型
5. 主样稿母本 + 样稿合同
6. 主角度 / 主问题
7. 主证据链
8. 段落任务分配
9. 模块顺序与篇幅上限
10. 禁止动作

## 五、没完全对齐前，先卡片化

如果下面这些东西还没锁稳：

1. 主比较关系
2. 第一旁证层到底比什么
3. 哪些点能写，哪些点不能写

就先出：

1. 内容卡
2. 对比卡
3. 证据卡

不要急着直接试 prose。

## 六、这层当前最重要的几个边界

### 1. 第一旁证层必须任务化

不能默认套横向、纵向、机制、反驳四件套。  
先回答：这篇稿第一层最该比什么。

### 2. 开头允许有钩子，但必须尽快入题

1. 可以有一小句可读性引子。
2. 前两段内必须进入研究对象、论题对象或主问题。
3. 不准长背景后入题。

### 3. 解释阈值与去元话术

1. 学术判断后面可以补一小句人话解释。
2. 解释只能点到关键转折位。
3. 禁止解释自己为什么解释、为什么这样写。

### 4. 研究段的最小动作链

1. 不能只剩结果名词。
2. 最少要交代干预、分组或动作、指标、要回答的问题。
3. 指标到结论之间必须留证据桥。

### 5. 价值判断必须从证据链长出来

1. 先证据，再病理意义，再研究或产品价值。
2. 不能整篇只剩防守性边界。
3. 也不能最后空口拔高。

## 七、最容易犯的错

1. 还没过大纲，就直接首稿。
2. 只给逻辑词，不给段落任务分配。
3. 大纲没问题，但写作合同没把禁止动作写死。
4. 该先出卡片时直接写正文。
5. 第一旁证层没任务化，导致整篇比散了。
6. 研究段把动作链和证据桥压没。

## 八、与下游的关系

1. 首稿生成层默认吃过闸后的大纲和写作合同。
2. 如果审稿发现故事线、逻辑大纲、主架构没立住，应先回到本层。

## 九、标准件

这一节包含已升级为工作版的正式标准件，以及仍在预留位的标准件。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。

### 标准件 S-04-01：写作合同最小字段标准

| 字段 | 内容 |
| --- | --- |
| **标准件名称** | 写作合同最小字段标准 |
| **所属节点** | 大纲过闸与写作合同层 |
| **要解决什么问题** | 防止写作合同只给逻辑词不给具体段落任务，防止首稿在没有明确约束下空写 |
| **固定起点** | 正式主题卡、`theme_lock_policy`、样稿合同已成立，且大纲已过闸（大纲含故事线 + 逻辑大纲 + 主承重段 + 各段任务分配） |
| **固定终点** | 写作合同卡 `10` 个必填字段全部填满 + 段落任务覆盖大纲所有段 + 禁止动作至少 `1` 条 |
| **输入** | 过闸大纲 + 冻结包（S-01-01 产出） + 正式主题卡 + `theme_lock_policy` + 样稿合同 |
| **输出** | 写作合同卡，包含以下 `10` 个字段，每个必须有明确值：① 正式主题卡 ② `theme_lock_policy` ③ 真实写作场景 ④ 一级稿型 / 主二级稿型 ⑤ 主样稿母本 + 样稿合同 ⑥ 主角度 / 主问题 ⑦ 主证据链 ⑧ 段落任务分配（逐段写明：该段的任务是什么、承重什么、引用什么证据，必须一一对应大纲段落） ⑨ 模块顺序与篇幅上限（每个模块标明位置和字数上限） ⑩ 禁止动作（至少 `1` 条，写明禁止做什么、为什么禁止） |
| **通过标准** | 同时满足以下全部：(1) `10` 个字段无一为空；(2) ⑧ 段落任务分配条目数 ≥ 大纲段落数，且一一对应，无遗漏段落；(3) ⑨ 每个模块有明确的篇幅上限数字（字数），不允许写"适量""酌情"；(4) ⑩ 禁止动作 ≥ `1` 条，每条写明具体禁止内容；(5) ①②④⑤ 与上游对象一致，无矛盾 |
| **不通过标准** | 命中以下任一即不通过：(1) 任何字段为空；(2) 段落任务分配条目数 < 大纲段落数；(3) 有大纲段落在合同里找不到对应任务；(4) 篇幅上限用模糊词代替数字；(5) 无禁止动作；(6) 正式主题卡、`theme_lock_policy` 或样稿合同未成立就直接编写合同；(7) 大纲未过闸就直接编写合同 |
| **失败后回流到哪一层** | 字段缺失或覆盖不全 → 留在本层补全后重新点检；大纲本身有结构问题 → 回大纲过闸重来；与冻结包矛盾 → 回冻结层核对后重冻结或修正合同；正式主题不稳 → 回主题确认层 |
| **自由度压缩规则** | (1) `10` 个字段逐个填，不允许合并或跳过；(2) ⑧ 段落任务分配必须逐段写，不允许"详见大纲"一句带过；(3) ⑨ 篇幅上限必须给具体字数，不允许百分比或模糊词；(4) ⑩ 禁止动作必须写成"禁止X，因为Y"格式，不允许只写"注意不要..." |
| **是否可自动化** | 半自动 |
| **自动化信号或判定来源** | 可自动化：字段非空检查、段落任务条目数与大纲段落数比对、篇幅上限是否为数字、禁止动作条目数检查。人工：段落任务是否真正一一对应大纲内容、禁止动作是否合理 |
| **正式来源** | (1) 本文件第四节"写作合同至少包含什么"；(2) 经验裁决总表 E-06（大纲独立过闸）；(3) 经验裁决总表 E-21（先判稿型不判不写）；(4) 经验裁决总表 E-23（第一旁证层必须任务化）；(5) 经验裁决总表 E-31（未对齐先卡片化） |
| **当前状态** | **工作版** |
| **边界备注** | 本标准只管写作合同的结构完整性。不管合同内容的写作质量（写作质量由门禁兜底）。段落任务分配的"好坏"在首稿生成和门禁阶段才能真正验证。本标准确保的是"合同不缺项、不漏段、不留模糊口子"。 |
