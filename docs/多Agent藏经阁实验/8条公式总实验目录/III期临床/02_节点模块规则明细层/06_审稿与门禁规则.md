# III期审稿与门禁规则

状态：整合稿 / 待继续补证  
日期：2026-04-08  
用途：收束审稿、门禁、问题单产出的统一口径  
边界：本文件不展开 session 纪律，session 纪律放到返修 loop

## 证据摘要

1. 已直接核对改稿复盘、模块补丁和模块卡规则，确认当前门禁已不只看事实和合规，还要抓重点不清、逻辑断裂、证据脱节、骨架被压坏。
2. 已直接核对近期逐字稿，确认情绪、逻辑、语气等维度有量化潜力，但现阶段仍应先作为“可审维度”，不是硬评分体系。
3. 已直接核对实战经验归档，确认门禁还必须抓横向比较失真、学科定义被擅改和价值判断空拔高。

## 一、门禁的基本职责

门禁不是“看着改一下”的口头判断，而是要把问题拆成下游可执行的问题单。

至少要能判断：

1. 题眼有没有偏
2. 故事线 / 逻辑大纲有没有立住
3. 证据桥有没有断
4. 动作链有没有缺
5. 开头入口有没有换轨
6. 结尾回收有没有失效
7. 模块有没有抢主线

## 二、门禁输出的最小格式

每张问题单至少要包含：

1. 问题位置
2. 问题类型
3. 严重度
4. 不允许动的骨架
5. 本轮只改什么

## 三、问题级别

当前先分 `3` 级：

1. `局部问题`
2. `结构性问题`
3. `任务理解问题`

这 `3` 级的差别，不是语气轻重，而是决定后面该回哪一层。

## 四、当前重点审的维度

1. 主线是否清楚
2. 重点是否明确
3. 逻辑是否连着
4. 证据是否接得上
5. 表达是否克制

## 五、附加硬检查项

### 1. 横向比较纪律

1. 先标证据类型，再谈强弱。
2. 不能跨维度硬比发生率、信号强度和风险等级。

### 2. 学科定义纪律

1. 局部论点不得擅自改写学科级定义。
2. 不能为了成文顺滑，就把多机制争议强行压成单因闭环。

### 3. 价值判断纪律

1. 价值判断必须从证据链自然长出。
2. 只讲边界、不讲意义，是问题。
3. 只有拔高、没有证据，也是问题。

## 六、最容易犯的错

1. 审稿只给感受，不给问题单。
2. 只说“这里不对”，不说骨架能不能动。
3. 把结构性问题误判成局部问题。
4. 横向比较只看表面数字，不看证据类型。
5. 为了让稿子顺，把复杂定义硬改成简单口号。

## 七、与下游的关系

门禁层不直接改稿。  
门禁层负责决定：

1. 这是不是局部回修
2. 要不要回合同层重开
3. 要不要回需求层重开

## 八、标准件

这一节包含已升级为工作版的正式标准件，以及仍在预留位的标准件。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。

### 标准件 S-06-01：门禁问题单标准

| 字段 | 内容 |
| --- | --- |
| **标准件名称** | 门禁问题单标准 |
| **所属节点** | 审稿与门禁层 |
| **要解决什么问题** | 防止审稿只给笼统感受（"读着不顺""整体还行"），强制把审稿结果拆成下游可执行的问题单 |
| **固定起点** | 首稿或返修稿提交到门禁层 |
| **固定终点** | 问题单生成，每条问题 `5` 个字段全部填满，且每条问题已标注严重度级别 |
| **输入** | 待审稿件 + 写作合同卡（S-04-01 产出） + 冻结包（S-01-01 产出） |
| **输出** | 门禁问题单，由 `0~N` 条问题组成。**如果 `0` 条问题**：输出"门禁通过"放行声明。**如果 ≥ `1` 条问题**：每条问题必须包含以下 `5` 个字段：① 问题位置（精确到段落编号 + 句子范围，如"第3段第2-3句"） ② 问题类型（从固定类型表中选：题眼偏移 / 故事线断裂 / 证据桥缺失 / 动作链缺失 / 开头换轨 / 结尾回收失效 / 模块抢主线 / 横向比较失真 / 学科定义被擅改 / 价值判断空拔高 / 表达不克制 / 其他） ③ 严重度（`局部问题` / `结构性问题` / `任务理解问题`，三选一） ④ 不允许动的骨架（写明该问题周围哪些句子、段落或逻辑结构在本轮修改中不允许动） ⑤ 本轮只改什么（写明本条问题的修改范围，精确到句或信息点） |
| **通过标准** | 同时满足以下全部：(1) 每条问题 `5` 个字段无一为空；(2) ① 问题位置精确到段落+句子，不允许写"中间部分""后半段"；(3) ② 问题类型从固定类型表中选取，如选"其他"必须附一句说明；(4) ③ 严重度三选一，不允许写"中等""比较严重"等模糊词；(5) ④ 不允许动的骨架写明具体引用上下文；(6) ⑤ 修改范围 ≤ `3` 句话或 `1` 个信息点（局部问题级别），结构性以上级别标明回流层 |
| **不通过标准** | 命中以下任一即不通过：(1) 任何一条问题的 `5` 个字段有空项；(2) 问题位置只给模糊描述；(3) 严重度用非标准词汇；(4) 只给感受不给问题单（如"整体读感不好"）；(5) 有问题但输出"门禁通过" |
| **失败后回流到哪一层** | 问题单本身不合格 → 留在门禁层重审，补全字段后重新输出；审稿发现无法判断某问题（缺原始材料或合同不全） → 回冻结层或合同层补齐后再审 |
| **自由度压缩规则** | (1) 问题类型从固定 `12` 类中选，不允许自创类型名称；(2) 严重度只能三选一，不允许造中间级别；(3) 每条问题必须同时给"不允许动的骨架"和"本轮只改什么"，二者缺一不可；(4) 如果审稿人认为"没问题"，必须明确输出"门禁通过"，不允许只说"大致可以" |
| **是否可自动化** | 半自动 |
| **自动化信号或判定来源** | 可自动化：每条问题 `5` 字段非空检查、问题类型是否在固定表内、严重度是否为三个标准词之一、问题位置是否含段落+句子编号。人工：问题内容的准确性、严重度判断、骨架边界划定 |
| **正式来源** | (1) 本文件第一节"门禁的基本职责"；(2) 本文件第二节"门禁输出的最小格式"；(3) 本文件第三节"问题级别"；(4) 经验裁决总表 E-10（好稿共用约束）；(5) 经验裁决总表 E-29（横向比较先标证据类型）；(6) 经验裁决总表 E-30（局部论点不得擅改学科定义） |
| **当前状态** | **工作版** |
| **边界备注** | 本标准只管问题单的格式完整性和分级合规性。不管审稿人对问题的判断是否业务正确（业务正确性由后续返修验证和经验回流兜底）。问题类型固定表如需扩展，走经验回流升层流程，不直接在本标准上临场加类型。 |
