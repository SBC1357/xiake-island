# III期返修 Loop 与 Session 纪律

状态：整合稿 / 待继续补证  
日期：2026-04-08  
用途：收束返修 loop、session 纪律、局部修改粒度和串改规则  
边界：本文件只回答“返修怎么做”，不重写首稿生成规则

## 证据摘要

1. 已直接核对模块补丁、模块卡规则和近期逐字稿，确认当前返修最大的风险不是“不会改”，而是大上下文导致的反复横跳。
2. 已直接核对最新补丁口径，确认“每一轮修改都新开全新 session”“串改优先，并发例外”“局部粒度以一句话 / 一个信息点 / 一个空缺位为主”已经成为当前最稳定的返修认识。

## 一、返修 loop 是主链，不是补丁

当前最小判断：

1. 返修 loop 不是可选动作，而是主链。
2. 审稿后先分级，再决定回哪一层。
3. 不再允许“整篇带着旧上下文继续聊着改”。

## 二、每一轮修改都必须新开 session

这是当前最重要的纪律之一：

1. 每一轮修改都必须新开 `1` 个全新 session。
2. 新 session 只吃本轮标准上下文和干净约束。
3. 旧 session 只负责提供原始证据，不继续执行修改。

## 三、本轮标准上下文至少包含什么

1. 当前待改的原文范围
2. 定死的上下文前后句 / 前后段
3. 本轮只改什么
4. 不允许动的骨架
5. 门禁点名问题
6. 本轮验收标准

## 四、问题级别与回流方向

### 1. 局部问题

典型样子：

1. 一句逻辑断裂
2. 一条证据桥缺失
3. 一个信息点表达失真

动作：
- 留在返修层处理
- 但也必须新开干净 session

### 2. 结构性问题

典型样子：

1. 故事线错了
2. 逻辑大纲错了
3. 主架构没立住
4. 多个局部问题彼此牵连

动作：
- 回写作合同与结构编译层
- 重开一个整稿改任务
- 新开 session

### 3. 任务理解问题

典型样子：

1. 受众错了
2. 稿型错了
3. 主问题错了
4. 交付目标错了

动作：
- 回需求对齐回执层

## 五、局部修改的最优粒度

局部修改默认不是：

1. 改一个字
2. 一大段整段重写

而是更接近：

1. `1` 句话
2. `1` 个信息点
3. `1` 个空缺位

## 六、串改优先，并发例外

当前默认纪律：

1. 默认串改，不默认并改。
2. 相邻句、同一段、共享同一承重逻辑的问题，禁止并改。
3. 只有完全独立、互不牵连的空缺位，才允许极少量并发。

## 七、用户已确认的正确句，默认进入保留区

如果用户明确说某个开头句、判断句或语气句“这个就是对的”，后续返修默认这样处理：

1. 先把它当成语气锚点和保留区。
2. 后续 Agent 只能接，不应顺手重写。
3. 只有用户再次点名要改，才允许动这一句。

## 八、为什么不能沿用旧会话硬改

因为旧会话会把下面这些东西一起带进来：

1. 前几轮的误判
2. 历史噪声
3. 已经过时的局部假设
4. 互相打架的修改目标

最终表现就是反复横跳：

1. 改逻辑，修辞炸
2. 改修辞，逻辑断
3. 压字数，信息掉
4. 补信息，字数又水

## 九、这层最容易犯的错

1. 旧 session 里反复补。
2. 一次把所有修改点丢给同一个 Agent。
3. 相邻句并改，最后串味。
4. 把结构问题按局部句子修。

## 十、标准件

这一节包含已升级为工作版的正式标准件，以及仍在预留位的标准件。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。

### 标准件 S-07-01：返修问题分级与回流标准

| 字段 | 内容 |
| --- | --- |
| **标准件名称** | 返修问题分级与回流标准 |
| **所属节点** | 返修 Loop 与 Session 纪律层 |
| **要解决什么问题** | 防止所有问题不分级别一律在旧会话里修，防止结构性问题被按局部问题处理，防止任务理解错误被按句子级修 |
| **固定起点** | 门禁问题单已产出（S-06-01 通过） |
| **固定终点** | 每条问题完成分级 + 回流目标层确定 + 新 session 标准上下文 `6` 件齐全 |
| **输入** | 门禁问题单（S-06-01 产出） |
| **输出** | 返修任务单，由 `1~N` 条返修任务组成。每条返修任务必须包含以下 `8` 个字段：① 问题来源（回指 S-06-01 问题单的具体条目编号） ② 问题级别（`局部问题` / `结构性问题` / `任务理解问题`，三选一，直接继承 S-06-01 ③ 的严重度） ③ 回流目标层（`局部问题` → 留在返修层；`结构性问题` → 回写作合同与大纲过闸层；`任务理解问题` → 回需求对齐回执层。此映射关系固定，不允许临场改路由） ④ 新 session 标准上下文（必须包含且仅包含以下 `6` 件：a.当前待改的原文范围 b.定死的上下文前后句/前后段 c.本轮只改什么 d.不允许动的骨架 e.门禁点名问题 f.本轮验收标准） ⑤ 修改粒度（`局部问题`：`≤ 1` 句话 / `1` 个信息点 / `1` 个空缺位；`结构性问题`：整段或整稿级，标明改动边界；`任务理解问题`：不在本层处理，直接回上游） ⑥ 串改/并改判定（默认串改；只有当待修问题与其他问题完全独立、不相邻、不共享承重逻辑时，才标注"允许并改"；相邻句、同段、共享逻辑的问题一律标注"必须串改"） ⑦ 执行顺序（如有多条返修任务，标明执行先后顺序，串改任务按顺序编号） ⑧ 验收标准（写明本条返修完成后，用什么标准判断改好了：如"第3段第2句补上证据桥，连接前句X和后句Y"） |
| **通过标准** | 同时满足以下全部：(1) 每条返修任务 `8` 个字段无一为空；(2) ② 问题级别与 S-06-01 ③ 严重度一致；(3) ③ 回流目标层与问题级别的固定映射一致（局部→返修层，结构性→合同层，任务理解→回执层）；(4) ④ 新 session 标准上下文 `6` 件齐全；(5) ⑥ 相邻句或同段问题标注为"必须串改"；(6) 每条返修任务都有明确的 ⑧ 验收标准 |
| **不通过标准** | 命中以下任一即不通过：(1) 任何字段为空；(2) 问题级别与门禁问题单严重度不一致且未说明原因；(3) 回流目标层与问题级别映射不一致；(4) 新 session 标准上下文 `6` 件不全；(5) 相邻句问题标注为"允许并改"；(6) 无验收标准；(7) 继续在旧 session 中执行修改而未新开 session |
| **失败后回流到哪一层** | 返修任务单本身不合格 → 留在本层重新编制；门禁问题单不够清晰导致无法分级 → 回门禁层补全后重新产出问题单 |
| **自由度压缩规则** | (1) 问题级别-回流目标层映射是固定的三对三，不允许造新级别或新回流路径；(2) 新 session 标准上下文固定 `6` 件，不多不少；(3) 局部问题修改粒度上限固定为 `1` 句/`1` 信息点/`1` 空缺位，不允许扩大；(4) 默认串改，并改必须逐条论证独立性；(5) 每条返修任务必须有独立验收标准，不允许"改好了就行" |
| **是否可自动化** | 半自动 |
| **自动化信号或判定来源** | 可自动化：字段非空检查、问题级别与回流目标层映射一致性检查、session 上下文 `6` 件齐全度检查、相邻句并改检测（基于位置编号）。人工：问题级别本身的判断准确性、验收标准的合理性、串改/并改独立性论证 |
| **正式来源** | (1) 本文件第二节"每一轮修改都必须新开 session"；(2) 本文件第三节"本轮标准上下文至少包含什么"；(3) 本文件第四节"问题级别与回流方向"；(4) 本文件第六节"串改优先，并发例外"；(5) 经验裁决总表 E-11（返修 loop 是主链）；(6) 经验裁决总表 E-12（结构性问题回上游重开）；(7) 经验裁决总表 E-13（每轮新开 session）；(8) 经验裁决总表 E-14（局部粒度 1句/1信息点/1空缺位） |
| **当前状态** | **工作版** |
| **边界备注** | 本标准管的是返修任务单的结构完整性和路由合规性。不管返修执行结果的质量（执行质量由下一轮门禁兜底）。如后续实战暴露新的问题级别（如介于局部和结构性之间的"跨段但非全稿"问题），走经验回流升层流程决定是否拆分级别，不直接在本标准上临场加级别。 |
