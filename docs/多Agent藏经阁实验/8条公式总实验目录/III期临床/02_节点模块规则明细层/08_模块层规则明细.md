日期：2026-04-08  
用途：收束 III 期模块层的最小规则，回答模块是什么、怎么挂、怎么防抢主线  
边界：本文件是收束版，只吸收当前稳定边界；模块库、模块卡字段和稿型-模块映射表继续以旧模块卡硬规格为真相源，这里不重抄

## 证据摘要

1. 已直接核对模块补丁与模块卡规则文档，确认当前模块层已经有较完整的模块库、模块卡字段和门禁检查项。
2. 已直接核对改稿复盘与最新补丁，确认模块层不只管“挂什么”，也要管“怎么写、怎么审、怎么改”。

## 一、模块层的定位

模块不是稿型，也不是全文主线。  
模块是围绕主线插入的内容块。

它至少回答：

1. 还要补什么内容块
2. 每个内容块承担什么功能
3. 出现在哪
4. 最多占多大篇幅

## 二、模块层的最小对象

当前最小正式对象仍然是：

1. 主线模块
2. 挂载模块
3. 模块卡

这里再钉死一条：

1. 模块层只回答“怎么挂、怎么写边界、怎么审边界”。
2. 模块层不重新发明稿型，也不单独再写一套完整架构。

## 三、模块层最核心的硬规则

1. 一篇稿只能有 `1` 条主线。
2. 挂载模块可以有多个，但必须有主次。
3. 模块不能抢主线。
4. 模块挂载必须受主二级稿型约束。

## 四、模块写法边界

当前最重要的 `3` 类模块家族边界：

### 1. 开头位模块

1. 只准建立入口
2. 不准改写全文进入方式

### 2. 研究 / 证据型模块

1. 必须保留最小动作链
2. 必须保留证据桥
3. 不得只剩结果名词

### 3. 回收模块

1. 必须回收前文主证据链
2. 不得只剩抽象总结或广告话

## 五、模块层与返修的关系

模块层不直接改稿，但它决定返修时哪些边界不能动：

1. 入口不能换轨
2. 动作链不能压没
3. 证据桥不能删空
4. 回收不能失效

## 六、模块层最容易犯的错

1. 把模块当新稿型。
2. 把模块补充写成抢主线。
3. 只写“影响哪些模块”，不写“在哪个节点生效”。

## 七、标准件预留位

这一节只预留标准件，不冻结标准本身。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。
