# III期经验回流规则

状态：整合稿 / 待继续补证  
日期：2026-04-08  
用途：收束 III 期里“案例怎么回流成规则”的最小机制，回答哪些经验能升层、哪些只能继续留在经验池  
边界：本文件不重复案例全文，也不替代 `01_散落经验裁决层`

## 证据摘要

1. 已直接核对仓库级硬约束，确认经验、AI 总结和想法都不能直接升格为正式规则。
2. 已直接核对现有母图与模块补丁，确认当前稳定接入口径已经是：先看原始证据，再做最小抽象，再映射节点，最后做裁决。
3. 已直接核对实战经验归档，确认单案例最大的价值不在“单次结论”，而在正反样式和回流边界。

## 一、回流顺序

固定顺序只有这一条：

`原始证据 -> 最小抽象 -> 节点落点 -> 裁决 -> 升层`

不允许跳步：

1. 不能从案例直接跳完整架构
2. 不能从 AI 总结直接跳正式规则
3. 不能因为某条想法说起来顺，就绕过裁决层

## 二、裁决状态

当前只允许下面 `5` 类状态：

1. `保留`
2. `收紧`
3. `合并`
4. `删除`
5. `待验证`

## 三、什么样的经验可以升到 02 层

至少同时满足下面 `3` 条：

1. 有直接来源，能回指到原始证据
2. 最小抽象已经压成一句不容易跑样的话
3. 节点落点明确，不会出现“到处都顺手写一版”

## 四、什么样的经验可以再升到 03 层

必须再多满足下面 `2` 条：

1. 它影响的是节点职责、主链路由、上下游输入输出或冻结边界
2. 它不是句位级微操、稿型专用技巧或模块库细目

## 五、案例先积累，再反抽清单

当前硬规则：

1. 单案例不直接扩规则。
2. 先积累正例和反例，再反抽清单动作。
3. 如果旧规则能接住，就优先补旧规则，不平地起新规则。

## 六、单一真相源纪律

1. `01_散落经验裁决层` 负责来源、抽象和裁决。
2. `02_节点模块规则明细层` 负责节点级规则落地。
3. `03_完整架构层` 只收会影响系统主链的结果。

除这三层之外，不再额外造一份平行总账。

## 七、标准件预留位

这一节只预留标准件，不冻结标准本身。

### 标准件标准

1. 标准件必须能挂到一个节点。
2. 标准件必须说清楚：解决什么问题、未来从哪来、当前是什么状态、现在能不能用、边界在哪里。
3. 标准来源优先级按 `00_总则与需求对齐回执规则` 的统一口径走。
4. 只有来源可回指、节点落点明确，才允许从预留位升级成正式标准。
