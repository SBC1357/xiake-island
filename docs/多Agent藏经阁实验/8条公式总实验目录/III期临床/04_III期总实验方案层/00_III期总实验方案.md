# III期总实验方案

状态：工作版 / 总方法论层
日期：2026-04-10
用途：定义 III 期如何分阶段生成，而不是预设固定字母阶段；保留 `IIIA` 既有锚点，后续阶段由证据反推。

## 这份文件解决什么

1. 把 III 期的阶段划分从“默认 ABC”改成“按闸口生成阶段”。
2. 只保留 `IIIA` 作为既有锚点，不把后续阶段先写死。
3. 说明阶段如何由主闸口、主 blocker、主验收对象、主回写层级、资产接入深度决定。

## 这份文件不解决什么

1. 不预设后续阶段数。
2. 不替代 `02_节点模块规则明细层`、`03_完整架构层`、`05_阶段实验包`。
3. 不播报当前执行状态，不记录单题过程。
4. 不把工作版阶段写成冻结结论。

## III期总目标

1. 把 III 期做成一套可逐段生成、可逐段验证、可逐段回写的方法论。
2. 让阶段划分由证据和依赖驱动，而不是由字母顺序驱动。
3. 让 `02` 接收被实跑打实的规则，`03` 接收影响系统边界的变化，`05` 接收阶段执行。

## 阶段生成法

1. 每一阶段只允许有 `1` 个主闸口。
2. 每一阶段只允许有 `1` 个主 blocker。
3. 每一阶段只允许有 `1` 个主验收对象。
4. 每一阶段只允许有 `1` 个最高回写层级。
5. 每一阶段只允许有 `1` 种主资产接入深度。
6. 只要其中任一项发生变化，就应重新判定是否要切新阶段。
7. 阶段名不先定死，阶段数也不先定死。

## IIIA 既有锚点

1. `IIIA` 保留，作为现有最小闭环验真锚点。
2. `IIIA` 负责把标准输入、路由、写作合同、首轮试运行、评分/裁决、第一批回写跑通。
3. `IIIA` 不负责预设后续阶段数，不负责全链冻结，不负责深资产全面接入。
4. `IIIA` 跑完后，后续阶段从剩余 blocker 和验收对象反推。

## 后续阶段怎么反推

1. 如果主 blocker 仍在回执、冻结、路由、合同，就说明还要先做上游收紧段。
2. 如果首稿、门禁、返修已经稳定，才有资格进入扩样或稳健段。
3. 如果冲突已经上升到节点职责或主链边界，才进入升层裁决段。
4. 后续阶段名称和数量不预设，只按证据生成。

## 本层与其他层的关系

1. `02` 层：接收被验证过的节点规则。
2. `03` 层：接收影响系统边界的变化。
3. `05` 层：承接阶段执行包和试运行证据。
4. `01` 层：承接经验裁决，但不代替本层做阶段划分。
5. 先改 `04`，再收 `05`。

## 环境层固定原则

1. III 期执行环境的总契约只写在 `04_III期执行环境契约.md`。
2. 阶段包只允许补本阶段额外前置，不允许重写环境总契约。
3. 只要环境问题同时涉及后端生成链、评分链、CLI 链，就必须先拆链再判断，不能混成一句“LLM 配置错了”。
