# III期阶段验收总表

状态：工作版 / 阶段验收总表
日期：2026-04-10

这份表现在只做两件事：
1. 把 `IIIA` 先钉成可执行的硬验收口径。
2. 给后续阶段提供一个通用闸门模板，不预设固定字母阶段，也不预设固定段数。

## IIIA 硬验收

| 维度 | 当前硬验收口径 |
| --- | --- |
| 阶段对象 | `IIIA`：最小闭环验真阶段，保留既有锚点，不等同于全局收口 |
| 最小样本量 | 当前工作口径：`10` 题主实验集；若后续阶段对象变化，按对象在 `04` 层补齐，不在这里瞎定 |
| 覆盖面 | 至少覆盖 `回执 -> 冻结 -> 路由 -> 合同 -> 生成 -> 机器评分 -> 主执行裁决 -> 单题全貌档案 -> 回写判定` 的最小闭环 |
| 通过分 | 当前工作口径：标准输出目标 `85` 分；若阶段对象或评分口径变化，按对象补齐，不提前冻结 |
| 硬失败项 | 1. 回执/冻结/路由/合同任一关键输入缺失；2. 单题全貌档案缺稿子本体或过程锚点；3. 机器评分与主执行裁决无法对齐；4. 回写越权到未裁决层级；5. 阶段裁决没有直接证据锚点 |
| 回退条件 | 先回 `05/IIIA` 补执行与证据；必要时回 `02` 收紧规则；若发现阶段边界本身有问题，再回 `04` 重新切阶段 |
| 连续无提升停机条件 | 待按 `IIIA` 真实对象与跑批结果补齐；目前不在此处假装冻结 |
| 当前验收结论 | 这阶段只在“跑通最小闭环并形成可回写结论”时才算过，不以“想清楚了”代替“跑出来了” |

## 说明

1. 这张表不负责定义整个 III 期怎么切段，只负责把每一段的验收门槛说硬。
2. `IIIA` 现在按工作口径先执行，后续如果对象、资产深度或回写层级变化，再回到 `04` 层重推阶段边界。
3. 后续阶段不要先写成固定字母列，再倒推验收；应该先由对象、依赖、资产接入深度和回写层级反推，再把模板补齐。
