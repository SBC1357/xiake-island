## 执行顺序

1. 先确认本阶段要跑的题目和输入。
2. 再冻结唯一 runner、唯一输出根、唯一评分口径。
3. 再冻结允许先冒烟的 case 集与 blocked 目录形态。
4. 再确认旧 `IIIA期试验` 里哪些内容迁入准备清单，哪些只留历史。
5. 再把结果文件和裁决文件全部降成未激活模板。
6. 最后只保留一个可启动前准备入口。

## 建设计划

1. 先抓高杠杆准备件，不平均铺开。
2. 先做会直接决定能否开跑的入口、输入、输出、停闸约束。
3. 先把可启动前准备壳搭起来，再考虑真正实跑。
4. 当前默认 runner 为 `scripts/run_ii_benchmark.ps1`，当前默认 smoke case 为 `lecanemab_patient`，直到有同轮直接证据推翻。
