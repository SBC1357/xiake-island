## 一、目录职责

本目录只做三件事：

1. 承接白盒编排器多轮施工方案
2. 承接每一轮白盒实验的施工入口与审计入口
3. 把每轮暴露出来的稳定问题回指到 `02/03` 两层

本目录不做：

1. 代替 `02_节点模块规则明细层` 冻结长期规则
2. 代替 `03_完整架构层` 冻结系统总图
3. 承接仓外 runtime 原始运行产物

## 二、真相源纪律

1. `07_白盒施工包/00_入口索引.md` 只负责：
   - 当前白盒施工包总入口
   - 当前活动轮次指针
   - 各轮施工包目录索引
2. 每一轮具体执行口径、范围、顺序、验收，只认该轮子目录下的总方案。
3. 任何已经稳定到足以升层的节点规则或系统边界，不在施工包里另写一版长期口径，而是回写 `02/03`。

## 三、当前活动轮次

当前活动轮次：`01_第一轮白盒完整可实验版`

定位：

1. 目标不是玩具 MVP，而是可完整跑 `IIIA` 三题的第一轮白盒版本
2. 白盒主链覆盖：`用户需求/题目 -> 稿型 -> 候选主题 -> 扩证成链 -> 正式主题确认 -> 写作合同 -> 取材 -> 出稿 -> 评分回指`
3. 本轮执行采用 `B'`：架构按终局 `C` 设计，评分先借旧评分器作适配器与对照组

## 四、目录约定

1. 每一轮一个下级目录，目录名按 `NN_轮次名称` 命名。
2. 每轮只保留少量强文档，不在施工包里铺碎文档。
3. 每轮总方案必须显式写清：
   - 本轮目标
   - 本轮边界
   - 本轮状态机与对象
   - 本轮实验题
   - 本轮工件与全貌回写要求
   - 本轮验收门与人工审计入口

## 五、当前目录结构


## 六、与 02/03 的关系

1. 施工包负责把“这一轮要怎么建、怎么跑、怎么验”写清楚。
2. `02_节点模块规则明细层` 负责把已经成熟的节点规则冻住。
3. `03_完整架构层` 负责把已经成熟的主链、节点边界和职责分工冻住。
