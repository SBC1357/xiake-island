# 第一轮白盒完整可实验版施工总方案

状态：工作版 / 待审计  
日期：2026-04-11  
用途：定义第一轮白盒编排器的完整施工范围、状态机、对象、工件和 IIIA 实验闭环  
边界：本方案不是长期架构总图；长期稳定规则仍回写 `02_节点模块规则明细层` 与 `03_完整架构层`

## 证据摘要

1. 已直接核对 `III期临床/02_节点模块规则明细层` 与 `03_完整架构层`，确认当前已经有一轮足够成熟的白盒状态机、节点边界和长期升层口径。
2. 已直接核对 `docs/多Agent藏经阁实验/侠客岛白盒编排器/00_启动入口.md` 与现有 `run_ii_whitebox_*.ps1`，确认仓内已有白盒实验骨架，但尚未形成 IIIA 第一轮完整可实验版。
3. 已直接核对当前线程已确认的 IIIA 稿型偏航事实：旧链当前主要病灶在 `planning/writing` 越权判稿型、判主轴、判 must-cover，而不是下游 draft 单点失灵。

## 一、本轮目标

第一轮白盒的目标只有一个：

`直接建设一条可完整跑 IIIA 三题的白盒主链，并把问题收敛到可从全貌包反追的具体节点。`

这里的“完整”指：

1. 白盒不只从合同层开始，而是覆盖完整上游：
   - `用户需求/题目`
   - `稿型判定`
   - `候选主题`
   - `扩证成链`
   - `正式主题确认`
   - `主题锁策略`
   - `写作合同`
   - `取材`
   - `出稿`
   - `评分回指`
2. 白盒默认自动推进，不把编辑拖进高频同步过闸。
3. 本轮看稿方式以 `全貌.md` 为主，通过形成过程和最终结果反追问题层。

## 二、本轮执行策略

本轮按 `目标选 C，执行选 B'` 落地。

含义：

1. **目标按 C 设计**
   - 白盒对象、状态机、工件、节点边界，一开始就按终局完整架构来设计。
2. **执行按 B' 落地**
   - 白盒主链负责到 `出稿 + 全貌回写 + 问题回指`
   - 评分先借旧评分器作适配器与对照组
   - 白盒同时保留新评分器未来要接入的位置

这意味着：

1. 本轮不再依赖旧 `planning` 决定稿型、主轴和合同
2. 本轮暂不要求旧评分器退场
3. 旧评分器在本轮的角色是：
   - 适配器
   - 对照组
   - 新评分器未来升级的反向证据源

## 三、本轮边界

### 3.1 覆盖范围

只覆盖 `IIIA` 三题：

1. `patient`
2. `mechanism`
3. `news`

### 3.2 不做的事

本轮不做：

1. 直接替换旧 runtime 全部执行逻辑
2. 一次写死所有客户、所有稿型的终局规则
3. 让 AI 代替编辑裁决“这是不是一篇真好稿子”

### 3.3 需要保留的人工职责

人工只留在这些位置：

1. 审看 `全貌.md` 与成稿
2. 判断推荐主线是不是优于备选主线
3. 判断当前评分、AI 味判断和稿件好坏是否一致
4. 判断哪些问题有资格升格为长期规则

## 四、本轮白盒状态机

第一轮白盒主链收成 `4` 个主状态：

### 4.1 任务定界

职责：

1. 接用户需求/题目
2. 锁稿型
3. 锁交付物、受众、篇幅
4. 判断当前是：
   - 已有正式命题
   - 只有候选主题
   - 证据不足需扩证

输出对象：

1. `task_brief_card`
2. `genre_routing_card`
3. `composition_target_card`

### 4.2 主题探索

职责：

1. 组织现有证据
2. 扩证
3. 形成逻辑链 / 故事链
4. 给出：
   - 推荐主线
   - 备选主线

输出对象：

1. `topic_candidate_packet`
2. `evidence_expansion_log`
3. `recommended_vs_alternative_card`

### 4.3 主题确认

职责：

1. 形成正式主题
2. 明确当前任务的 `theme_lock_policy`
3. 判断当前是否达到成文闭环

输出对象：

1. `topic_confirmation_card`
2. `theme_lock_policy_card`
3. `composition_closure_check`

### 4.4 正式写作

职责：

1. 生成大纲
2. 过闸
3. 编译写作合同
4. 取材
5. 出稿
6. 接评分适配器
7. 落全貌与问题回指

输出对象：

1. `outline_gate_card`
2. `writing_contract`
3. `materials_packet`
4. `draft_bundle`
5. `score_adapter_result`
6. `review_backtrace`
7. `case_fullview`

## 五、本轮核心对象

第一轮必须先把下面对象白盒化，不再让旧 `planning/writing` 隐式代判：

1. `task_brief_card`
   - 任务原文、交付物、平台、受众、目标篇幅、当前窗口
2. `genre_routing_card`
   - 一级稿型、主二级稿型、旁支候选、当前判定依据
3. `sample_contract`
   - 主样稿母本、借用层级、禁止误学项、样稿只学什么
4. `topic_candidate_packet`
   - 候选主题、主线、支撑点、当前缺口、下一轮扩证方向
5. `topic_confirmation_card`
   - 正式主题、正式主问题、正式落点、不可漂移点
6. `theme_lock_policy_card`
   - `strict / bounded / open`
7. `outline_gate_card`
   - 大纲、主承重段、段落任务、过闸结果
8. `writing_contract`
   - 正式主题、稿型、样稿合同、段落任务、篇幅上限、禁止动作
9. `materials_packet`
   - 正文取材包、supplement 取材包、关键事实位、回指锚点
10. `review_backtrace`
   - 问题属于哪一层、证据锚点是什么、下一轮该改哪一层

## 七、本轮全貌包要求

第一轮不做“只看最终稿”。  
每题必须产一份白盒全貌包，至少包括：

1. 当前任务理解
2. 当前稿型判定
3. 推荐主线 / 备选主线
4. 正式主题与锁策略
5. 大纲与写作合同
6. 取材包
7. prompt 全文
8. 当前稿件全文
9. 评分适配器结果
10. review backtrace
11. 本轮真实 blocker
12. 下一轮应改哪一层

## 八、本轮问题回指分类法

第一轮白盒发现的问题，统一回指到下面 `6` 类，不再只说“prompt 不行”：

1. `任务定界错`
   - 需求理解、交付物、受众、篇幅、窗口识别错误
2. `稿型路由错`
   - 一级 / 二级稿型、样稿母本、样稿学习边界错误
3. `主题探索错`
   - 候选主题、扩证方向、推荐主线 / 备选主线错误
4. `主题确认错`
   - 正式主题、主题锁策略、成文闭环判断错误
5. `合同与取材错`
   - 大纲、合同、must-cover、materials 组装错误
6. `表达与审查错`
   - prompt、draft、评分适配器或问题回指错误

## 九、本轮施工顺序

第一轮按下面顺序施工：

1. 先搭白盒对象与目录骨架
2. 再把 `任务定界 -> 主题探索 -> 主题确认 -> 正式写作` 四状态串起来
3. 再接 `IIIA` 三题
4. 再落全貌包
5. 再用旧评分器适配结果
6. 再按全貌暴露的问题回指节点
7. 再把稳定问题回写 `02/03`

## 十、本轮验收门

本轮不以“分数全过”为验收。  
本轮先以“白盒主链是否真实成形”验收。

最低验收条件：

1. `IIIA` 三题都能从白盒入口完整跑到：
   - `writing_contract`
   - `materials_packet`
   - `generated.txt`
   - `score_adapter_result`
   - `全貌.md`
2. 三题都能在 `全貌.md` 中直接看出：
   - 稿型怎么判的
   - 主线怎么来的
   - 哪一层开始出问题
3. 至少能把问题稳定回指到本方案第八节的 `6` 类之一
4. 至少完成一轮 `02/03` 回写，而不是只停留在 runtime 结论
