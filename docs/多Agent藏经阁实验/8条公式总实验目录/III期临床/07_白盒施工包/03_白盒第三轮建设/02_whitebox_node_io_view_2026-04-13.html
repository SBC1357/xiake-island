这页的目标不是给程序员看 schema，而是帮你快速判断 4 件事：主链有没有跑偏、每层到底交付什么、当前样例到底跑出了什么、如果不对先去改哪里。

给谁看 这是给验收这套白盒系统的人看的，不是给写代码的人看的。
你应该先看什么 先看主链图，再看“后审原始裁定”和“本轮最终停点”的区别，最后顺着节点卡片看每层交付和改动入口。
这页应该怎么用
  • 先看主链图，判断链路有没有跑偏。
  • 再看“当前样例”，判断这次到底卡在哪一层、为什么没过。
  • 最后看节点卡，判断这层交付件和边界是不是对的；真要动手改，再去“去哪改”区找文件入口。

先看主链，不要先看字段

真正要审的是交付关系。冻结架构已经定死：第一层冻结意图和证据，合同层编译执行规格，大纲层编译骨架，成稿节点负责首稿 / 整改，后审回原始数据核，然后只允许固定回流。

后审后的固定去向

  • pass：通过。
  • requirement_rework：回第一层。
  • whole_revise：回成稿节点整改模式。
  • local_patch：进局改 loop，再回后审。

两个容易搞错的边界

  • 前审：并行，只判断“按当前结构化证据够不够开写”。
  • 补充证据：独立侧链，默认不进主链，不得污染主证据包 JSON。

这次真实样例现在到了哪

样例任务
  • 要写什么：把仑卡奈单抗 48 个月临床数据翻成患者 / 家属能懂的科普解读。
  • 核心问题：这 4 年数据，对患者到底意味着抢回多少时间、保住多少日常能力。
  • 要求结构：患者痛点 → 核心获益翻译 → 亚组理解 → 长期价值 / 风险 → 选择路径。
样例当前状态
  • 第一层：direct，因为目标、约束、材料和验收标准都已清楚。
  • 前审：pass，说明当前结构化证据足够开写。
  • 后审原始裁定：needs_local_patch → local_patch。
  • 局改 loop：已跑满 3 轮，仍未收口。
  • 本轮最终停点：whole_revise_exhausted。
  • 原因：后审认为问题本来只该局改，但局改 3 轮后仍没收口，所以这次整轮运行最终没有通过。
这次样例最关键的一句判断 要分两层看：后审原始裁定是 local_patch，这说明主链、模块承重和回流方向基本对了；但这次整轮运行最后停在 whole_revise_exhausted，说明局改 loop 没能把问题收掉。

逐层看板

每张卡只默认回答 5 个问题：这层干嘛、这层收什么、这层交什么、这次样例实际是什么、如果不对先去改哪里。原始字段名全部收在折叠区。

1. 取材节点 materials把候选材料整理成正式材料入口
这层干嘛
它不理解任务，只负责把候选材料整理干净，给第一层提供正式材料入口。
这层收什么
候选题和候选材料池。关键输入形式是 CandidateId / CandidatePool / input_rules。
这层交什么
materials_manifest.json、materials_digest.md、materials_compiled.txt。
如果不对先去改
脚本：run_ii_whitebox_materials.ps1。
展开看这层原始字段 
candidate_id, candidate_name, status, exit_code, run_directory,
extractable_count, total_items, compiled_text, coverage_audit, items
2. 第一层 layer1先分流，再冻结意图包 / 证据包 / 样稿边界
这层干嘛
把“用户到底要写什么”和“现在手上的材料到底能当什么证据”冻结下来。它不是写稿层。
这层收什么
用户原话、候选上下文、materials_manifest.json、materials_compiled.txt。
这层交什么
给合同层：intent_pack.json + evidence_pack.json。
给成稿层：evidence_pack.md。
同时落盘：route_decision.json、sample_boundary.json。
这次样例实际是什么
这次冻出来的是一篇“仑卡奈单抗 48 个月数据”的患者向解读；核心问题是“这 4 年到底为我抢回了什么”。分流是 direct。样稿只允许借结构 / 语气 / 风格，不允许借事实 / 数字 / 结论 / 标题。
如果你觉得这层不对，先去改哪里
  • 分流逻辑：prompts/layer1_main.md
  • 直通冻结：prompts/layer1_direct.md
  • SP 澄清：prompts/layer1_sp.md
  • 意图包结构：prompts/intent_extraction.md
  • 证据包：prompts/evidence_preprocessing.md
  • 样稿边界：prompts/sample_boundary.md
  • 脚本：run_whitebox_layer1.ps1
这一层你真正该盯什么
  • 意图包有没有把用户真实目标保住，而不是缩水摘要。
  • 证据包 JSON 有没有把来源、边界、冲突、缺口说清。
  • 证据包 md 是不是预处理材料，不是粗糙摘要。
  • 是否把 gold / reference_answer 排除在生产链外。

这层交什么
