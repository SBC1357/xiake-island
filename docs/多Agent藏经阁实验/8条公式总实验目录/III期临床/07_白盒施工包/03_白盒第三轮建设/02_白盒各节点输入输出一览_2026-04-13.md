## 这份文档现在解决什么

这份文档不是给你背英文节点名，也不是给你看一张密表。  
它只解决 4 件事：

1. 白盒主链现在到底怎么跑。
2. 每一层到底吃什么、吐什么。
3. 你如果要调提示词，应该改哪个 prompt 文件。
4. 你如果要调模型 / profile / base_url / protocol，应该看哪个配置文件。

## 先看总流程，不要先看细节

主链现在就是这一条：

`取材 -> 第一层 -> 写作合同 -> 观点层 -> 大纲 -> 成稿首稿 -> 后审 -> 四流`

把它展开成人话，就是：

1. 先把原始材料整理出来。
2. 第一层先冻结"到底要写什么"和"现在有哪些证据"。
3. 写作合同把要求编译成正式执行规格。
4. 观点层先把文章的主张、策略口径、论证方向和医学视角定住（当前与写作合同串行，设计目标可并行）。
5. 大纲层把合同和观点卡拆成可执行骨架。
6. 成稿节点按骨架写首稿。
7. 后审拿成稿回原始材料核事实、边界和路线。
8. 后审根据结果，只能往 4 个方向之一走：
   - `pass`
   - `requirement_rework`
   - `whole_revise`
   - `local_patch`

## 后审后三流，到底各回哪

### 1. `pass`

意思：  
稿子通过了，主链停。

### 2. `requirement_rework`

意思：  
错不在后面，是前面任务理解、边界、证据方向就错了。

回哪：  
回 `第一层 -> 写作合同 -> 大纲 -> 成稿首稿 -> 后审`

### 3. `whole_revise`

意思：  
前面主链没错，但整篇稿整体写歪了。

回哪：  
回成稿节点的整改模式，也就是 `draft(revise)`。  
不是新开一个顶层，不是另起整改世界。

### 4. `local_patch`

意思：  
整篇不用重写，只修局部。

回哪：  
进 `local_patch_loop`，也就是：

`局改 -> 局部验收 -> 再回后审`

## 0. 取材层 `materials`

### 这一层是干嘛的

把候选题目录里的原始材料，整理成：

- 一份材料清单
- 一份摘要
- 一份合并文本

### 真正执行脚本

- `docs/多Agent藏经阁实验/侠客岛白盒编排器/run_ii_whitebox_materials.ps1`

### prompt 在哪

- 这一层没有独立 prompt 文件。

### profile / 模型配置在哪

- 这一层不是独立 LLM 节点，没有单独 profile。

### 主要输入

- `CandidateId`
- `CandidatePool`
- 候选题自己的 `input_rules`

### 主要输出

- `materials/materials_manifest.json`
- `materials/materials_digest.md`
- `materials/materials_compiled.txt`

### 最重要边界

- 它不负责理解任务
- 它不负责判断证据够不够
- 它不负责写合同、大纲、成稿

## 1. 第一层 `layer1`

### 这一层是干嘛的

先冻结两件事：

1. 这次到底要写什么
2. 这次手里到底有哪些证据

### 真正执行脚本

- `docs/多Agent藏经阁实验/侠客岛白盒编排器/run_whitebox_layer1.ps1`

### prompt 在哪

这一层不是一个 prompt，而是一组 prompt：

- 主分流：
  - `prompts/layer1_main.md`
- 直通：
  - `prompts/layer1_direct.md`
- SP：
  - `prompts/layer1_sp.md`
- 三并发模块：
  - `prompts/intent_extraction.md`
  - `prompts/evidence_preprocessing.md`
  - `prompts/sample_boundary.md`

### profile / 模型配置在哪

看：

- `config/whitebox_llm_profiles.json`

这一层对应的 profile 有：

- `layer1_main`
- `layer1_direct`
- `layer1_sp`
- `intent_extraction`
- `evidence_preprocessing`
- `sample_boundary`

### 主要输入

- `user_input`
- 候选题上下文
- `materials/materials_manifest.json`
- `materials/materials_compiled.txt`

### 主要输出

主产物只看 3 个：

1. `layer1/intent_pack.json`
2. `layer1/evidence_pack.json`
3. `layer1/evidence_pack.md`

- `layer1/freeze_status.json`

### 最重要边界

- 先判断 `direct / sp`
- `gold / reference_answer` 不得进生产主链
- 没冻结完成，不得进入写作合同层

### profile / 模型配置在哪

- `config/whitebox_llm_profiles.json`
- 对应 profile：
  - `local_patch`
  - `local_patch_audit`

### 主要输入

协议主输入是：

- `draft/draft.txt`
- `post_review/revise_card.json`
- `layer1/evidence_pack.md`

但当前脚本真实还会补读：

- `outline/outline.json`
- `layer1/intent_pack.json`
- `writing_contract/writing_contract.json`
- `post_review/post_review_result.json`

### 主要输出

- `draft/draft.txt`
- `local_patch/patch_log.json`

### 最重要边界

- 协议主输入是“当前稿 + 整改卡 + 证据 md”
- 但脚本为了不丢上下文，补读了大纲和后审结果
- 它内部其实是两段：`local_patch -> local_patch_audit`
