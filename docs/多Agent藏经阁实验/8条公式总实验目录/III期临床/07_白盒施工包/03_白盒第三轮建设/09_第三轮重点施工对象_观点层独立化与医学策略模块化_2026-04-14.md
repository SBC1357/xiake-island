# 第三轮重点施工对象：观点层独立化与医学策略模块化

- 状态：已落地 / 观点层已接入主链（串行验证中）
- 说明：本文为第三轮施工背景说明；当前冻结主链以 `../00_入口索引.md` 三、当前冻结主链 为唯一真相源。当前执行态、阻塞和下一步以 `11_白盒第三轮长线程任务包_2026-04-14.md` 状态卡为准。

## 为什么要独立出“观点层”

现在这套白盒链路里，写作合同层负责把意图包和证据包编成正式合同，大纲层负责把合同和证据编成可执行骨架，成稿层负责按骨架落成文章。  
但“这篇文章到底先站在哪个观点上说话、该走哪条医学论证路线、重点强调什么策略”这件事，不能全压在写作合同层或大纲层里，否则它们会一边编规则、一边替文章先定调，容易把路线、策略和成稿动作混在一起。

所以本轮要把“观点层”单独拎出来。它的作用不是重写合同，也不是直接写正文，而是先把文章的主张、策略口径、论证方向和医学视角单独定住，再交给后面的大纲层继续编排。写作合同层在设计上仍然是并行层，不是观点层的下游；当前实现和验证口径以 `11_白盒第三轮长线程任务包_2026-04-14.md` 为准。

## 各层分别干什么

- 观点层：先定“这篇文章站哪边、怎么说、先讲什么后讲什么”，把主张和策略先冻结住，形成一份可复用的观点卡。
- 写作合同层：把第一层给出的意图和证据编成正式写作合同，明确能写什么、不能写什么、合格标准是什么。
- 大纲层：把写作合同和观点卡一起拆成板块骨架，确定每个板块写什么、怎么承重、怎么过渡。
- 成稿层：按大纲和证据把文章写成完整中文稿，负责真正落字。

## 建议主链（已落地，当前真相源见 `../00_入口索引.md`）

下方为设计时建议，观点层已接入主链（当前串行验证中）。实际执行链路以入口索引为准。

建议主链目标先按这个顺序走：

第一层 -> （并行）写作合同层 + 观点层 -> 大纲层 -> 成稿层 -> 后审

这个意思很直白：第一层先把任务冻结清楚，然后写作合同层和观点层可以并行跑，前者管规则和边界，后者管主张和策略。等这两块都稳了，再进大纲层和成稿层。

## 观点层的最小输入 / 输出

观点层不要吃太多东西，最小够用就行。

- 最小输入：第一层冻结后的意图包、证据包，外加当前任务的主题和受众。
- 最小输出：一份观点卡，说明这篇文章要站的观点、要用的医学策略、主要论证顺序、需要避开的误区。

用人话讲，就是先回答“这篇文章要替谁说话、从哪个角度说、靠什么逻辑推进”，然后把这个结果主要交给大纲层继续展开。

## 并行成立的前提

并行不是谁想并就能并，前提是第一层已经把这些东西冻结住了：

- 任务到底要写什么
- 受众是谁
- 证据边界在哪里
- 有没有样稿边界或禁区

如果这些还没冻结，观点层和写作合同层就不能各说各话，否则一个定路线、一个定规则，很容易互相打架。

## 回流原则

- 该回观点层的情况：任务主张不对、医学策略走偏、论证路线定错、文章整体站位需要重选。
- 只回成稿层的情况：主张和路线都对，只是句子不顺、段落别扭、个别证据位没落稳、中文表达需要局部修一下。

一句话说清楚：观点错了，回观点层；只是写坏了，回成稿层。
