## 定位

观点层不是新的顶层、不是写作合同的子模块、不是大纲层的前置。它是一个独立模块，在第一层冻结之后、大纲层之前，协议目标上与写作合同层**并行**；当前脚本先按串行接入验证。

它只做一件事：先把文章要站的观点、医学策略方向、主张优先级和论证路线冻结住，形成一份观点卡。

## 下游消费关系

- **大纲层**：在现有输入基础上，额外吃 `viewpoint_card.json`。大纲层用它来确定板块排列顺序和承重策略。
- **写作合同层**：不吃观点卡。写作合同管规则和边界，不管主张和策略。
- **成稿层**：不直接吃观点卡。观点卡的影响通过大纲层传递到成稿层。
- **后审层**：不直接吃观点卡。后审回原始数据核事实，观点偏移归到"需求返修"流。

## 回流规则

| 后审判定 | 是否回观点层 | 说明 |
|----------|-------------|------|
| 主张错误 / 策略走偏 | 是 | 属于"需求返修"，回第一层时观点层也要重跑 |
| 论证顺序别扭但方向对 | 否 | 属于大纲/成稿问题 |
| 局部句子不通 | 否 | 属于成稿/局改问题 |

回观点层的唯一途径是通过"需求返修"回第一层，第一层重新冻结后，观点层和写作合同层一起重跑。不存在"只回观点层不回第一层"的路径。

## 并行成立前提

以下条件全部满足后，写作合同层和观点层才能并行：

1. 第一层已冻结意图包 JSON
2. 第一层已冻结证据包 JSON
3. 意图包中的任务目标、受众、禁区字段均非空

如果第一层输出不完整，两者都不能启动。
