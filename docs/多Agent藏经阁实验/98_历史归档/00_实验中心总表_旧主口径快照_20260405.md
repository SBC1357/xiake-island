## 固定执行流程

每一轮实验都按下面这 3 步走，不靠临时记忆：

1. 三个中心执行当前实验方案，按统一规范落成正式文档
2. 三个中心互相阅读对方的实验结果，各自补一份“汇总补充结论”
3. Codex 做最终审核，并明确：
   - 是否推进下一阶段实验
   - 还是继续扩展当前阶段
   - 还是判定本轮失败并回退到 I 期

这条流程的全周期定义见：

- [藏经阁临床试验总纲_20260403.md:98](../00_实验中心/藏经阁临床试验总纲_20260403.md:98)

## Codex 提醒责任

实验中心默认把下面这 3 件事当成 Codex 的固定提醒职责：

1. 提醒三中心先各自落正式结果，再进入互读
2. 提醒互读后必须补“汇总补充结论”，不能直接跳总审
3. 提醒总审必须明确写出：
   - 升期
   - 扩期
   - 回退
4. 提醒 `01_强留痕重跑记录.md` 必须按强留痕模板保留“假设变更链”和“废案保留”，不能只交结论摘要
5. 任何发给各中心执行的文档，必须同时写明：
   - 交付文件名
   - 精确提交路径
   - 如为多中心任务，需先预建各中心子目录
