## 一、这件事到底是什么

这不是普通的“文章拆解任务”，也不是“模仿某个作者写一篇稿子”。

我们真正要判定的是：

**藏经阁未来应该用什么通用储存模式，来保存好文章中可迁移、可重组、可跨域复用的写作本体。**

这里有两个系统：

- **藏经阁**：长期知识、规则、风格、样本与资产库，负责“什么值得长期存”
- **侠客岛**：本地编排与运行系统，负责“这次任务调用什么、怎么组合、怎么留痕”

所以这轮研究的核心不是先改侠客岛，而是先回答：

**藏经阁到底应该存什么。**

## 二、为什么会一路推翻旧拆法

新 Agent 必须知道，我们不是一开始就站在“动作 / 构件 / 规则 / 拓扑”这条路上的。  
这条路是被一路逼出来的。

### 阶段 1：最开始是“分层直觉”

最开始的直觉是：好文章像油画，一层一层抹出来。  
于是容易自然地把藏经阁想成 `L1 / L2 / L3 / L4` 这样的内容层。

问题很快暴露：

- 同一类东西会出现在不同层
- 有些东西像规则，不像内容
- 有些东西像组合方式，不像“更高一层内容”
- “冰点样板”“戴老板样板”这种整包东西太大，没法精确迁移

所以：

**单纯分层不够。**

### 阶段 2：后来试过二维坐标

后来一度尝试把它想成二维：

- 一边是“颗粒度”
- 一边是“成稿层 / 组织层”

这个思路比纯分层前进了一步，但很快又卡住。  
因为所谓“颗粒度”本身就混了不同种类的东西：

- 信息
- 判断
- 关系
- 推进
- 修辞

这些不是一条尺度上的大小关系，而是不同类。

所以：

**“颗粒度轴”这个说法不稳。**

### 阶段 3：再往下逼，发现规则不是外挂，而是主体

继续往前想时，一个关键转折出现了：

门禁、正向组合、负向组合，不应该被扔在主表外面。  
它们本身就是结构的一部分。

于是逻辑变成：

- 不只是“有什么对象”
- 还要知道“这些对象怎么能拼、怎么不能拼、何时该触发、失败时怎么降级”

这一步很关键，因为它让系统从“整理素材”转向了“定义可执行本体”。

### 阶段 4：再继续逼，发现整包样板不该做底层主存储

像“冰点样板”“戴老板样板”这种东西，一开始很容易被当成大对象。  
但越拆越发现，它们其实混了太多性质：

- 信息偏好
- 判断习惯
- 关系接法
- 推进拓扑
- 修辞习惯
- 风味约束

所以：

- 它们不适合做底层主对象
- 但它们仍然有价值
- 更合适的角色是：**上层聚合视图**

这条后来也被独立 Agent 的实验进一步加固了。

## 三、双锚点样稿把问题逼到了哪里

这轮最早的两个锚点样稿是：

1. EGFR 深度稿
2. 戴老板编辑的小米稿

这两篇的作用不是“代表全世界”，而是：

- 一篇偏医疗深度解释
- 一篇偏商业人物与公司浮沉

两篇差异足够大，但都足够强。  
如果它们能被拆出共同的、可迁移的结构，说明我们触到了真正有用的东西。

基于这两篇，我们先得到过一批中间对象：

- 题眼
- 起手器
- 判断链
- 信息篮子
- 解释桥
- 风味约束
- 门槛与降级

这些仍然有价值，但后来被进一步压缩和改写了。

## 四、独立 Agent 实验把哪些东西钉实了

后来做了多 Agent 独立实验。  
这一步很重要，因为它避免我们只是在自己说服自己。

### I 期的最小结论线

I 期并没有证明“通用模式已成立”，  
但已经把方向筛到一个比较窄的范围：

1. **整包样板不适合做底层主存储**
2. **底层方向明显偏向：动作 + 构件 + 规则**
3. **作者风味更可能是上层组合映射，而不是整包底层件**
4. **当前方向值得继续走，但还远未完备**

### IB 期到目前为止最有价值的新增

基于目前已经交上来的 `GPT-5.4` 和 `GLM-5` 结果，有几条特别值得带走：

#### 1. 拓扑层越来越像必须单列

`GPT-5.4` 最有力地逼出了这件事。

它发现，光有“动作 / 构件 / 规则”还不够，  
因为强个人风格稿、人物稿、半佛、张佳玮这类样本，会在“推进顺序 / 壳 / 级联方式”上明显失真。

所以当前很强的活结论是：

**拓扑层很可能必须入底座。**

#### 2. 人物真相钻头已经够硬

人物稿不是把人生经历码平。  
人物稿更像围着一个执念、伤口、矛盾或骨相反复回钻，直到回答“这个人到底是什么”。

这个对象现在已经不再只是猜想，  
而是足够继续保留在候选层。

#### 3. 人格值得保留，但还没完全证成

`Opus/Sonnet 4.6` 在 I 期最有价值的增量是：

**作者风味可能不是底层件，而是动作、构件、规则、拓扑的组合偏好。**

这件事被它叫成“人格效应”。  
这个方向目前仍然值得保留，但因为 `Opus/Sonnet 4.6` 的 IB 期还在继续，所以还没到最后收口。

#### 4. 对照组已经说明两条旧路都不够好

到目前为止，实验对照组已经反复说明：

- **整包样板法** 不够好
- **纯碎片法** 也不够好

也就是说，未来的通用模式大概率既不能是“大整包模板”，也不能只是“碎片堆积”。

## 五、当前最稳的工作模型

如果新 session 只允许记住一句话，那就是这句：

**藏经阁底层目前最可信的候选方向，是：动作 + 构件 + 规则 + 可能需要单列的拓扑；作者风味更像上层人格组合映射；整包样板降级成聚合视图。**

再拆开一点说：

### 目前大概率应当存的

- 动作
- 构件
- 规则
- 拓扑
- 来源锚点
- 适用范围
- 失败降级方式
- 上层聚合视图

### 目前大概率不应当直接存成底层主对象的

- 整包作者样板
- 具体比喻文本
- 表面口气
- 只能事后解释、不能指导重组的“伪规则”

## 六、哪些事已经成立，哪些还不能说成立
