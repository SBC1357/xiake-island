## 一、这份文档给谁看

这份文档是给两类人看的：

1. 你
2. 后面接手这件事的协作者

它不是给模型自言自语用的，也不是过程聊天记录。  
它只负责回答 4 件事：

1. 这件事到底要证明什么
2. 现在已经做到哪一步
3. 后面每一期分别要验证什么
4. 什么情况下才能进入下一期

## 二、这件事到底在研究什么

这不是普通的“拆文章”任务。  
也不是“模仿某作者风格”的任务。

我们真正要证明的是：

**藏经阁是否存在一套通用储存模式，能够把好文章里的可迁移本体拆出来，并长期支持复用、重组、生成和失败诊断。**

如果最后证明成立，藏经阁应该能做到：

1. 解释一篇文章为什么成立，而不是只说“像谁”
2. 把文章拆成可复用对象，而不是整包样板
3. 用这些对象跨域重组出新稿
4. 在失败时指出问题出在对象、规则、拓扑还是映射
5. 被侠客岛运行时稳定调用

## 三、目前已经拿到的稳结论

到现在为止，可以相对稳地保留 7 条判断：

1. 整包样板不适合作为底层主存储。
2. 当前最可信的候选底座是：`动作 + 构件 + 规则 + 拓扑`。
3. 人格更像上层组合映射，不该和底层四层平级。
4. 条件性对象 / 稿型子型不应被误写成新层级，应进入单独的适用边界表。
5. 人物向样本已经逼出了稳定的人物稿子型，不能继续当边角例子处理。
6. 阿基米德当前已有待入库 raw sample pool，但还不能写成“已正式入库”。
7. 现有结果足以继续往前推，但还不足以宣布“正式 schema 已成立”。

## 四、整个 I~III 期到底在验证什么

整个 I 到 III 期，不是在做“文章赏析项目管理”。  
它验证的是：

**写作系统新架构能否成立。**

这里的新架构，先落在藏经阁，后反推侠客岛。

- **I 期总体目标**
  - 明确一个尽可能合理、后续不会被整体推翻、只会被微调的藏经阁储藏方案
- **II 期总体目标**
  - 把这个方案补成一个足够通用的藏经阁储藏规范
- **III 期总体目标**
  - 在足够大的样本量下，把调取规则也明确出来，并反向推动侠客岛重建

所以整个试验链的方向是：

**先确认“存什么”，再确认“怎么通用地存”，最后确认“怎么被系统稳定调取”。**

## 五、全周期基础规则

这套试验从一开始就按下面的规则运行：

1. 每一轮实验都可能对当前架构做两种事：
   - 补强
   - 推翻
2. 如果核心架构被推翻：
   - 当前轮判失败
   - 项目回到 I 期重新圈定方案
3. 如果只是补强：
   - 允许修改当前架构
   - 继续留在当前主期，扩展同一期实验
4. 扩展命名规则：
   - I 期历史上已经用了 `IB`
   - 从 II 期开始，扩展轮次统一按 `IIA / IIB / IIC ...` 命名
5. 任何升期结论都不能只靠单一路结果：
   - 必须先有三中心正式结果
   - 再有三中心互读后的补充结论
   - 最后由 Codex 给出是否升期、扩期或回退的总审判断
6. “强留痕”统一指结构化变更链：
   - 要保留推理过程
   - 但不用原始聊天流充当正式结果
   - 正式结果至少要写清“旧判断、触发证据、新判断、影响范围、废案保留”
7. 实验的第一收敛方向默认不是“抢先宣布正确答案”，而是先系统找出：
   - 哪些旧判断不成立
   - 为什么不成立
   - 被哪些样本和证据击穿
8. 口径卫生与概念防污染，和仓库防污染同优先级：
   - 禁止为了图快临场打造新概念，污染已有口径
   - 每一个新解释概念、层级、模板、对象名，必须先检索仓库里有没有同类型旧概念
   - 未完成“旧概念检索 + 映射关系说明 + 证据验证”前，禁止把新说法推进成正式口径
   - 新加入的 session、模型、Agent，必须先读当前状态和既有口径，再继续施工
   - 未经验证禁止口径漂移；发现漂移，按污染处理：先止血、再回退、后重建

被推翻的旧判断，和最后留下来的新判断一样，都是正式实验产出。

## 六、单轮实验组织方式

每一轮实验都按“三中心并发 + Codex 总审”的结构运行。

三个实验中心是：

1. `Opus/Sonnet 4.6`
2. `GPT-5.4`
3. `GLM-5`

Codex 不替代三中心做独立实验。  
Codex 的职责是：

1. 维护实验总纲、阶段方案和目录规范
2. 收三中心正式结果
3. 审核三中心互读补充结论
4. 给出升期、扩期或回退判断

每一轮固定流程如下：

1. 三个中心执行同一轮实验方案，按统一规范落成文档
2. 三个中心互相阅读对方的实验结果，各自补一份“汇总补充结论”
3. Codex 做最终审核，并明确：
   - 是否推进下一阶段实验
   - 还是继续扩展当前阶段
   - 还是判定本轮失败并回退到 I 期

## 七、标准实验方法库

下面这些不是零散招数，而是全周期都可以调用的标准实验方法：

1. **互相拆解**
   - 三中心对同一篇样本独立拆解，比较是否能收敛到同一骨架
2. **互相重组**
   - 用一个中心提出的结构，去重组另一个中心重点解释过的样本
3. **第三种稿件重组**
   - 拿前面没见过的新稿型做重组，检验是否真的跨域
4. **带标准答案的命题作文**
   - 给定题目和参考答案，检验当前架构能否稳定逼近“正确解”
5. **无标准答案的命题作文**
   - 给定题目但不设标准答案，结果由人工审核，检验当前架构在开放任务里的可用性

这些方法不是一一对应具体阶段，  
而是不同阶段各自挑最合适的组合来打。

## 八、整条试验链怎么分

### I 期：储藏方案圈定期

I 期要解决的不是“通不通用”，而是：

**藏经阁到底应该怎么存。**

I 期主要方法：

1. 双锚点样本拆解
2. 互相拆解
3. 互相重组
4. 第三种稿件重组

I 期过关标准：

1. 已经提出一套明确的储藏方案
2. 新样本进来后，主要是边界微调，而不是整套方案被整体推翻
3. 三中心对主骨架开始收敛
4. Codex 判断这套方案已经够资格进入 II 期通用性验证

当前判断：

**I 期总体目标已经完成。**

### IB 期：I 期补硬与升期确认

`IB` 是 I 期历史上已经实际跑出的补硬轮。  
它的职责不是改变 I 期目标，而是补硬 I 期最脆的地方，确认是否真的够资格进入 II 期。

IB 期主要补的就是：

1. 拓扑层是否必须单列
2. 人格应不应该定位为上层映射
3. 人物向对象是否稳定存在
4. 当前方案是否明显优于对照组

当前判断：

**IB 已经拿到结果，但还没有形成统一、可审计、可冻结的升期版本。**

### II 期：储藏规范通用性验证期

II 期的目标不是再证明“有没有方案”，  
而是：

**把现有方案补成一个足够通用的储藏规范。**

II 期统一采用扩展轮次命名：

- `IIA`
- `IIB`
- `IIC`
- `IID`
- 必要时继续往后扩

II 期的总过关标准是：

1. 新样本进入后，只会补规则和边界，不会反复推翻核心骨架
2. 互相拆解、互相重组、第三种稿件重组能稳定通过
3. 带标准答案的命题作文能逼近正确结构
4. 无标准答案的命题作文能在人工审核下保持可用

#### IIA：方案锁定与补强期

IIA 的任务是：

**把现有 I + IB 的结果收成一个可信的候选方案 `v1`。**

这一步重点不是扩张规模，  
而是先把候选方案、对象分级、规则表、阶段终点评估和强留痕重跑收齐。

IIA 收口阶段新增一条明确要求：

1. 统一术语为 `人格`
2. 单列一张 `条件性对象表`
3. 这张表回答“哪些对象只在特定条件下成立、哪些已经窄到稿型子型”
4. 这件事属于结构整理，不以扩大样本量为前提

#### IIB：通用性扩样与方法扩展期

IIB 的建议重点是：

1. 扩第三种稿件样本
2. 补带标准答案的命题作文
3. 扩更强的对照组
4. 对 IIA `条件性对象表` 中单样本、弱证据或跨中心命名未统一的条目做扩样复核

如果 IIA 之后发现问题只是“还不够广”，而不是“主骨架错了”，  
那就进 IIB，而不是回 I 期。

#### IIC：闭卷生成与抗过拟合验证期

IIC 的建议重点是：

1. 去作者标签、去原标题、去原文顺序后，验证系统能否仅凭处理后的素材包写出高质量作品
2. 在加噪音的条件下，验证系统能否区分主承重信息与次要信息
3. 在给定少量限制条件时，验证系统能否逼近原稿的结构家族与气质家族
4. 顺手裁决 `IIB` 遗留的边界：`问题现场群像稿` 与 `双人物共享核心轴`

#### IID：开放任务与人工审核期

IID 的建议重点是：

1. 跑无标准答案的命题作文
2. 引入人工审核
3. 判断当前规范在开放任务里是否仍然稳定

### III 期：调取规则确认与系统反推期

III 期的目标是：

**在足够多的样本量下，把调取规则也明确出来，并反向推动侠客岛重建。**

III 期主要验证：

1. 什么任务该调什么对象、规则、拓扑、映射
2. 调取规则是否稳定、可追责、可留痕
3. 运行时 trace 能不能稳定回指藏经阁对象
4. 这些规则能不能反向约束侠客岛的重建

III 期过关标准：

1. 藏经阁不只是“会存”，而且“会被稳定调取”
2. 调取规则不再频繁改口
3. 足够多样本下，架构不再被高频推翻
4. 侠客岛重建已经能被这套调取规则反向驱动

只有走到这里，才有资格说：

**写作系统新架构被阶段性证明成立。**

## 九、现在到底在哪个阶段

当前最合理的统一口径是：

- **I 期：总体完成**
- **IB：结果已出，但升期版未统一冻结**
- **IIA：主体执行完成**
- **IIB：主体执行完成，但未作为完全闭环通过**
- **当前实际工作：IIC**

之所以不继续把工作卡在 `IIB` 的补样细节里，  
是因为当前最需要的不是继续局部补量，而是直接验证：

**这套系统脱离作者线索后，是否仍然真的会写。**

换句话说：

**现在不是继续空转争论，而是进入 `IIC 闭卷生成与抗过拟合验证期`。**

## 十、当前阶段最该做什么

既然当前阶段是 IIC，  
那最该做的就是下面 5 件事：

1. 冻结闭卷素材包
2. 把出题方与答题方彻底分离
3. 在加噪条件下跑优稿测试
4. 在受限条件下跑相似度测试
5. 判断两条 `IIB` 遗留边界在闭卷条件下是否收束

这里的重点不是继续认作者、认刊物、认套路，  
而是逼系统只用“可迁移结构”写作。

如果一篇样本主要贡献的是“击穿旧判断”，  
那它已经是有效结果，不需要等到稳定新答案完全长出来，才算有价值。

当前阶段的成功，不是“继续补几个边界说明”，  
而是：

**把现有成果送进闭卷环境，检验它是不是抗过拟合的真结构。**

## 十二、最短版结论

如果只看一分钟，只记下面这 6 句：

1. 整个 I 到 III 期，是在确认写作系统新架构，而不只是拆文章。  
2. I 期负责圈定藏经阁储藏方案；II 期负责把它补成通用规范；III 期负责把调取规则也定出来。  
3. 每一轮实验都可能补强，也可能推翻；如果核心架构被推翻，就回到 I 期。  
4. 每一轮都由三个中心并发实验、互读补充，再由 Codex 总审决定升期、扩期或回退。  
5. 当前实际阶段是 `IIC 闭卷生成与抗过拟合验证期`。  
6. 现在最重要的不是继续争名字，而是把现有结果收成 `v1`，再决定是进 `IIB` 还是继续扩 `IIA`。
