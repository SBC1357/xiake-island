## 一、这份文档解决什么问题

这份文档不负责交代我们聊了多久、改了多少版。  
它只回答一件事：

**基于当前两篇锚点样稿，我们已经能比较稳定地知道什么，应该把这些结论怎样转成藏经阁建设逻辑。**

它不是最终真理。  
但它应该足够稳，能作为后面实验和系统设计的共同起点。

## 二、当前最重要的总判断

### 1. 藏经阁不能继续只按“整包样板”来想

如果继续把“冰点样板”“戴老板样板”“某作者整套风味”当成底层主存储，后面会反复撞到三个问题：

1. 样板太大，换域时不能稳定迁移
2. 样板里混了不同性质的东西，无法精确复用
3. 一旦样板和内容拓扑不匹配，整包就会失真

所以，整包样板最多只能是上层聚合视图，不能再是底层主存储。

### 2. 当前最稳的中心对象，不是“层”，而是三类东西

到目前为止，最值得保留的判断不是某个固定 `L1~L4`，而是：

- **动作**：作者在推进文章时实际做了什么
- **构件**：动作所操作的最小内容对象
- **规则**：动作怎样操作构件、构件怎样组合、什么时候不能组合

这三类对象一起，才更像“可迁移的写作本体”。

### 3. 风格不是单独一层，很多所谓“风格”其实是组织方法

目前最值得记住的一条纠偏是：

**级联 / 平行不应该先当成作者风格标签。**

它们更像内容拓扑：

- 如果内容之间有强依赖，就更接近级联
- 如果几条线是同级佐证关系，就更接近平行

这说明很多过去会被误叫成“风格”的东西，实际是内容组织方式。

## 三、两篇锚点样稿已经共同指向的可迁移对象

下面这些不是终版 schema，  
但已经是当前两篇样稿共同指向、且有跨域价值的东西。

### 1. 动作

这是文章推进时反复出现的高价值动作。

当前最稳的几类有：

- **压力起手**
  - 先让读者感到旧状态不可持续
  - EGFR 用“旧分类失效”起手
  - 小米用“2016 低谷现场”起手

- **关键问题钉子**
  - 全文必须围着一个高价值问题打
  - 不是讲知识全貌，而是逼一个核心判断

- **历史回望链**
  - 不是背景介绍，而是证明“今天的问题是怎么长出来的”

- **解释桥**
  - 把抽象机制、复杂战略翻译成能被抓住的判断画面

- **比喻承重**
  - 比喻不是装饰，而是解释本身
  - 但具体比喻不可迁移，只能重造

- **细节买权**
  - 大判断必须用具体细节买到发言权

- **失败收编**
  - 失败不是污点，而是逼出系统性答案的节点

- **边界后拔高**
  - 先承认边界，再做价值收束

### 2. 构件

这是动作所操作的内容对象。

当前最稳的几类有：

- 一条硬事实
- 一个判断
- 一组关系
- 一个场景
- 一段引述
- 一条边界声明

这里要特别注意：

这些构件还不一定是最终最小件。  
它们只是当前最稳的中间层对象。

### 3. 规则

这是最容易被低估、但最有系统价值的一层。

当前已经比较稳的几条规则有：

- **判断必须由事实购买**
- **判断越大，前面需要的细节成本越高**
- **承重比喻必须前置于被解释对象**
- **内容拓扑决定级联还是平行**
- **失败必须出场，但处理力度由说服路径决定**
- **收尾不能高于正文证据支撑的最大判断**

这批规则的意义在于：

它们不只可以解释原文，  
还可以约束新稿生成和失败复盘。

## 四、哪些东西可迁移，哪些东西不可直接迁移

### 1. 可迁移的

- 动作
- 构件定义
- 组合规则
- 触发条件
- 失败后的降级方式
- 某一类文章的内容拓扑判断

### 2. 不可直接迁移的

- 具体比喻本身
- 具体场景本身
- 某篇文章的具体历史素材
- 某篇文章的专有引述
- 整包样板

最关键的一条是：

**可迁移的不是“遗迹”“MIX”“乔峰”，而是这些对象在原文中承担的结构作用。**

## 五、这对藏经阁建设意味着什么

### 1. 藏经阁应该优先存什么

当前最稳的方向是，藏经阁优先存下面 5 类：

1. **母本来源**
   - 原文章本身
   - 原文章是证据，不是垃圾

2. **切片后的对象**
   - 动作实例
   - 构件实例
   - 规则实例
   - 以及它们的来源锚点

3. **映射关系**
   - 哪类动作常操作哪类构件
   - 哪类规则约束哪类组合
   - 哪种内容拓扑下应选级联还是平行

4. **适用范围与门槛**
   - 什么场景能用
   - 什么场景不能硬套
   - 不满足条件时如何降级

5. **聚合视图**
   - “冰点式”“戴老板式”“辰公子式”
   - 这些可以保留，但应作为上层可阅读视图，而不是底层主存储

### 2. 藏经阁不应该再优先存什么

不应再把下面这些当底层主对象：

- 整包作者风味文件
- 具体比喻集合
- 把级联 / 平行写成作者风格标签
- 只能解释原文、不能指导新稿的“事后规则”

## 六、这对侠客岛意味着什么

如果藏经阁未来这样存，侠客岛运行时应该承担的是：

- 这次任务调用了哪些对象
- 这次触发了哪些规则
- 选了哪种内容拓扑
- 哪些地方因为证据不够而被降级
- 最终的 trace 如何回溯到对象、规则、来源文章

一句话：

- **藏经阁负责“存本体”**
- **侠客岛负责“跑这次”**

## 七、当前最值得警惕的四个误判

### 1. 把整包样板误当底层件

这会让系统永远停在“模仿”。

### 2. 把具体比喻误当可复用资产

真正该存的是“比喻承重”的动作与相关规则，不是比喻文本本身。

### 3. 把级联 / 平行误写成风格

它们更可能是内容拓扑选择。

### 4. 把规则写成事后解释

如果规则不能用于：

- 拟合新文
- 生成新稿
- 解释失败

那它还不是系统规则。

## 八、当前仍未被证明的部分

下面这些目前仍是活假设，不能写死：

1. 动作 / 构件 / 规则 是否已经足够完备
2. 当前构件是否已经足够接近最小可存单位
3. 这套结构能否跨更多作者、更多题材稳定存在
4. 是否还需要补更高层的映射对象
5. 阿基米德、人物稿、半佛仙人、张佳伟等样本进来后，会不会逼出新的大类
