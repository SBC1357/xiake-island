# 侠客岛白盒编排器启动入口

## 用途

本目录用于先建设、验证、优化“白盒编排逻辑”，暂不直接修改侠客岛黑箱编排本体。

`run_ii_whitebox_phase.ps1` 和 `run_ii_whitebox_resume.ps1` 都只是机器执行入口，不等于规则阅读入口。
任何新 session 要接手本目录工作，先读仓库根目录 `AGENTS.md`；本文件只补 `AGENTS.md` 之外、本任务独有的必读文件。

## 当前顶层禁令

1. 自 `2026-04-06` 起，如无明确证据证明 `raw_source_analysis.md` 这种级别的原始资料解读不能支持写出高质量文章，默认禁用 `G0` 卡、`g0_evidence_card.json` 或其他同类复杂 `JSON` 结构作为项目需求、实验前置或启动阻塞项。
2. 当前默认主路径改为：Agent 直接读取原始 `PDF / PPT / 图表 / inputs`，并落 `raw_source_analysis.md` 作为当前可读、可审、可继续压缩的前置输入。
3. 只有在正式文件和 [00_总留痕.md](../8条公式总实验目录/03_过程强留痕/00_总留痕.md) 中出现明确反证，证明 `raw_source_analysis.md` 级别素材仍不足以支持高质量写作时，才允许重新启用 `G0` 卡类复杂结构。
4. 即使后续重新启用，`G0` 卡也只能是从 `raw_source_analysis.md` 或原始资料直读结果压缩出的派生产物，不得再反过来卡住当前实验或真实写作任务。

### 0.5 `10题三段验证` 主结构

1. `IID` 顶层执行结构固定为 `10题三段验证`；旧的 `3题最小基线 + A臂 / B臂` 不再作为正式主结构。
2. 第 `1` 题固定为盲测：
   - 不看 `gold`
   - 不看参考样稿
   - 不吃答案形约束
   - 只验证当前系统对象在单题上是否能给出可回指结果
3. 第 `2~7` 题固定为“阶段性放开条件”段：
   - 第 `2~3` 题：只验证无样稿优化能否扩展，不放样稿变量
   - 第 `4~5` 题：样稿路径 / 来源 / 选择人 / 可见阶段 / 防泄露边界全部冻结后，才允许把样稿变量作为条件变量放开
   - 第 `6~7` 题：锁定前面已证实有效的提升项，只验证这些提升是否已经沉成系统级能力
   - 历史上的 `B臂` 若保留，只能指第 `4~5` 题样稿条件变量阶段，不再是 `IID` 顶层结构
4. 最后 `3` 题固定为真实用户颗粒度指令段：
   - 只接真实用户指令
   - 不得回退成预清洗样题或白盒自造题面
   - 只验证前两段中已拿到直接证据的能力是否能迁移
5. 当前静态题池只承接前 `7` 题；最后 `3` 题在收到真实用户指令时再逐题冻结输入边界、`raw_source_analysis.md` 与运行留痕。
6. 每题开跑前都必须先补出合格的 `raw_source_analysis.md`：
   - 必须基于题目原始材料、`inputs/`、原始 `PDF/PPT/图表` 等原始资料
   - 不得读取 `gold/`、参考样稿、成稿 `docx` 或答案形摘要
   - 若 `raw_source_analysis.md` 只是流程审计、结论判定或空泛摘要，该题只允许记为 `无效实验`

### 1. 第一批系统对象与验收

1. `P1 / writing_contract`
   - 必修字段：题型路线、必写结构、核心事实位、必须前置判断、风险边界
   - 当前已知来源：`genre / required_structure / core_data_points / must_frontload`
   - 验收：每题合同与 `draft_prompt` 都能显式看到这些字段；缺任一字段按未完成处理
2. `P2 / input_contract`
   - 必修字段：必保留数字、必保留比较位、必保留 figure / table 值、每项来源锚点
   - 验收：进入 draft 前，输入层必须把这些值结构化交给成稿层；只有原始 `PDF/PPT/图表` 文本碎片不算通过
3. `P3 / orchestration_contract`
   - 必修动作：把 `input_contract` 与 `writing_contract` 的硬字段全程透传；关键事实缺失时只能拦截或标 `blocked`
   - 验收：缺字段、缺来源、缺关键值时，不得放行成功判定
4. `P4 / materials_coverage`
   - 必修动作：补齐原始 `PDF/PPT/图表` 关键值的抽取与回填；具体缺口由前置工程逐题确认
   - 验收：该类关键值能稳定进入输入层并回指来源；只拿到局部值、近似值或碎片文本时，按未收口处理

### 2. 原始资料读取约束与非 blocker 说明

1. 原始资料高效阅读工具（`PDF/PPT/图表`）统一视为提效工具，不是当前 `IID` blocker。
2. 当前只看该题原始资料是否已经被读成合格的 `raw_source_analysis.md`；能直接支持后续写作就继续，不能则把该题记为 `无效实验`。
3. 不要求等统一阅读工具全部完工才开始 `10` 题验证；但任何题目都不得拿模糊值、近似值或无锚点结论冒充有效实验。
4. 若后续补出统一工具，它服务的是效率与稳定性，不直接替代实验裁决。
5. 统一口径：工具不是 blocker，事实缺口才影响题目有效性。

### 3. 三段验证中的盲审与条件变量规则

1. 审核允许看：
   - 题目简报与材料

3. 样稿变量只允许出现在第 `4~5` 题条件变量组；第 `1` 题与第 `2~3` 题以及第 `6~7` 题一律不可见。

   - 只作风格锚点
   - 只作结构锚点
   - 只作返修锚点
   - 不得作为事实抄写源或 `gold` 替身

   - 核心数字、比较位、图表值是否齐全
   - 结构承重是否完整
   - 风险边界与结尾判断是否成立
   - 关键结论是否可回指来源
6. 一票否决项起始版：
   - 错题型、错路线
   - 漏关键数字、关键比较位或关键图表值
   - 主判断未前置或与材料不符
   - 出现无来源支撑的结论
   - 题面要求存在竞品 / 对照 / 风险边界，但稿面缺失
7. 盲审必回产物：
   - `通过 / 不通过`
   - 命中的裁决项
   - 触发的一票否决项
   - 显性回指
   - 对应返修方向

## 当前白盒目录包含什么

- `run_ii_whitebox_orchestrator.ps1`：单题白盒合同生成
- `run_ii_whitebox_materials.ps1`：按题目边界抽取允许材料并生成取材摘要

- `run_pdf_raw_index.ps1`：对题目目录做 `raw-only` 原始 PDF 索引，保留 noisy inputs，不读 `gold/样稿/docx`
- `run_ii_whitebox_phase.ps1`：整批白盒阶段执行
- `run_ii_whitebox_resume.ps1`：在既有阶段目录中续跑，不重做已完成步骤
- `whitebox_common.ps1`：公共函数、模型选择、错误分型
- `config/ii_candidate_pool.json`：题池、输入边界、参考答案、D类伪承重点
