# 防爆指令

**状态**：Active  
**性质**：仓库级长线程续跑 / 接棒防爆规则  
**适用**：低模型长任务、分阶段攻坚任务、可能被限速或上下文打断的任务  
**优先级说明**：如与 [AGENTS.md](AGENTS.md) 或 [仓库卫生最高禁令.md](仓库卫生最高禁令.md) 冲突，以它们为先；其余长线程场景本文件直接生效

## 顶部速读卡

- 这份文件解决什么：防止长线程、低模型、多阶段任务在中断、换人、限速或续跑时，只靠聊天记忆续命，导致方向跑偏、状态断档、后手接不上。
- 本轮先记住这一条：没写进状态卡、台账和最近验证锚点的知识，默认都不算交接完成。
- 命中这些场景必须深读正文：要长线程续跑；要接棒别人的阶段任务；要切模型、切 session、暂停后恢复；要在阻塞点停下或准备换人。
- 深读顺序：先看“1. 最小交接四锚必须同时存在”“2. 限速 / 崩溃 / 中断时先封板，再换人”“3. 低模型必须走‘小步过闸’”，再看“接棒顺序”“最小交接口径”“明确禁止”。

## 这份文件是为了解决什么

这份文件只解决一类问题：

- 模型降级到 low / mini / spark 后，一次改太多层，方向跑偏
- 线程跑了很久后被限速、上下文崩溃或中断
- 前一个 Agent 只在聊天里解释过，后一个 Agent 接不上，不知道现在卡在哪一步

所以这份文件的目标不是“让模型更聪明”，而是让任务**不靠聊天记忆也能续跑**。

## 核心原则

1. 每次只收一个阶段、一个主假设。
2. 代码一落盘，留痕要立刻跟上。
3. 没写进状态卡和台账的知识，默认不算交接完成。
4. 后续 Agent 只准读文档与直接验证结果，不准靠聊天记录脑补当前状态。
5. 阶段不过闸，不准扩大修改面。

## 强制执行件

### 1. 最小交接四锚必须同时存在

当前任务如果有阶段化长任务文档，任何阶段开始前、阶段验收后、发生阻塞时、模型切换前、线程结束前，都必须同时维护好下面四类交接锚点：

1. `当前接棒状态卡`
至少包含：
- `phase`
- `status`
- `current_hypothesis`
- `changed_files`
- `last_verified`
- `last_failed`
- `latest_scores`
- `blocker`
- `next_first_step`
- `updated_at`

2. `追加式修改流水`
实际代码写入后，必须立刻把前因、目标、改动、结果、验证追加到当前任务文档文末台账。

3. `下一棒第一步`
每次停下时，都必须明确下一位 Agent 接手后的第一条命令或第一步检查；`next_first_step` 必须写成可执行动作。

4. `最近直接验证锚点`
每个阶段至少要留下最近一条直接验证锚点，优先是原始命令与结果、评分 JSON 路径、生成稿路径、trace 路径或原始报错。

不允许：

- 只在聊天里汇报
- 只在终端里解释
- 先连改几轮，最后一次性回忆补账
- 用“稍后统一整理”替代即时记录

四锚缺一，按未收口处理。

### 2. 限速 / 崩溃 / 中断时先封板，再换人

遇到模型限速、上下文崩溃、线程被中断时，禁止直接把窗口一关了事。

必须先做 4 件事：

1. 把当前阶段状态改成 `gated` 或 `blocked`
2. 把最近一次 `last_verified` 和 `last_failed` 写清
3. 把当前产物路径、报错路径、分数路径写清
4. 把 `next_first_step` 写成下一位 Agent 开工第一步

这 4 件事没做完，不算可交接。

### 3. 低模型必须走“小步过闸”

如果当前执行模型不是高能力大上下文模型，就必须额外遵守：

1. 一次只让它处理当前阶段的唯一主假设
2. 不允许同时改 runner、planning、writing、evidence 四层
3. 15 分钟内连续两次方向性错误，立刻停止当前阶段深改
4. 没过当前阶段闸门，不准让它读下一阶段目标

### 4. 口头知识一律不算数

下面这些都不能替代交接文档：

- 聊天消息
- 终端里的自然语言解释
- “我刚看过，应该是这个原因”
- “上一位 Agent 应该已经修了”

如果某个关键判断只存在于口头或聊天里，没有进状态卡、台账或直接验证锚点，后续 Agent 一律按“未证明”处理。

## 接棒顺序

后续 Agent 接棒时，顺序固定为：

1. [仓库卫生最高禁令.md](仓库卫生最高禁令.md)
2. [防钻空子.md](防钻空子.md)
3. 本文件
4. 当前阶段任务文档顶部“当前接棒状态卡”
5. 当前阶段任务文档文末最近几条台账
6. 最近一条直接验证结果对应的产物或报错

没按这个顺序读完，不准开始下一轮改动。

## 最小交接口径

每次准备停下或换 Agent 时，至少要让后手能直接回答：

1. 现在处于哪个阶段
2. 这阶段唯一主假设是什么
3. 最近一次直接验证通过了什么
4. 最近一次直接验证失败了什么
5. 下一步第一条命令或动作是什么

只要这 5 个问题答不出来，就说明还没交接好。

## 明确禁止

1. 长线程跑到一半，只在聊天里解释状态，不更新文档。
2. 代码已改，但没立刻写台账。
3. 阶段已切换，但状态卡还停留在上一阶段。
4. 模型被限速后直接换 Agent，不先封板。
5. 把“上一位 Agent 说过”当成证据。
